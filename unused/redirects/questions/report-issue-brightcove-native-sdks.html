Überblick

Ist bei der Entwicklung mit den Brightcove Native Player SDKs etwas schief gelaufen oder ist unerwartet aufgetreten? Lassen Sie uns wissen, was passiert ist, damit Sie wieder auf die Spur kommen können.

Um den Meldeprozess zu beschleunigen, gehen Sie vor dem Einreichen eines Fehlers wie folgt vor:

  1. Überprüfen Sie, ob dies ein bekanntes Problem ist
  2. Isolieren Sie das Problem auf die Brightcove SDKs

Sobald Sie festgestellt haben, dass Ihr Problem kein bekanntes Problem ist und von den Brightcove-SDKs isoliert ist, können Sie einen Fehler übermitteln. Bitte folgen Sie diesen Schritten:

  1. Geben Sie fallspezifische Informationen an
  2. Fügen Sie zusätzliche hilfreiche Informationen hinzu
  3. Senden Sie Ihren Fehler

Überprüfen Sie, ob das Problem bekannt ist

Überprüfen Sie zunächst, ob Ihr Problem im Entwicklerforum gemeldet wurde:

  • Brightcove Player SDKs Entwicklerforum

Überprüfen Sie dann, ob Ihr Fehler ein bekanntes Problem ist, das in der Entwicklerdokumentation angegeben ist:

  • Bekannte Probleme mit nativem SDK

Isolieren Sie das Problem

Versuchen Sie, das Problem in einer der bereitgestellten Beispielanwendungen zu reproduzieren. Diese Apps bieten eine Basisimplementierung und können Ihnen dabei helfen, herauszufinden, ob das Problem mit dem SDK oder einem externen Faktor zusammenhängt.

Testen Sie mit einer der folgenden Beispiel-Apps:

  • Brightcove SDK für iOS-Beispiel-Apps
  • Brightcove SDK für Android-Beispiel-Apps

Informationen bereitstellen

Um fallspezifische Informationen bereitzustellen, sammeln Sie Folgendes:

  • Fehlerverhalten, das Sie erleben / das erwartete Verhalten
  • Gerätespezifische Details (Marke, Modellversion)
  • Verwendete SDK-Version
  • Absturzprotokolle (logcat für Android, Konsolen-/Geräteabsturzprotokolle)
  • Schritte zum Reproduzieren

Weitere Informationen

Fügen Sie die folgenden Details ein, wenn Sie können:

  • Code-Auszug
  • Beispiel-App mit Fehler
  • Was Sie versucht haben, das Problem zu lösen

Senden Sie Ihren Fehler

Verwenden Sie Folgendes, um Ihren Fehler beim Support-Team einzugeben:

  • Erstellen Sie ein allgemeines Support-Ticket
