Verwenden des SSAI-Plugins

Mit dem SSAI-Plugin können Sie Inhalte mit Dynamic Delivery und serverseitiger Werbung abspielen.

Es gibt zwei Optionen für die Wiedergabe von Inhalten mit serverseitigen Anzeigen:

  1. Geben Sie eine URL für SSAI-Inhalte an
  2. Bereitstellen eines Videos mit VMAP

In jedem Fall fordert das SSAI-Plugin die VMAP-Quelle an, parst sie und fügt sie automatisch zu VideoView hinzu.

Geben Sie eine URL für SSAI-Inhalte an

Rufen Sie mit einer SSAI-Asset-URL die folgende Methode auf:

SSAIComponent.processVideo(String url)

Hier ist ein Beispiel:

String ssaiUrl = "https://onceux.host.com/path/to/content.once"
SSAIComponent ssaiComponent = new SSAIComponent(context, brightcoveVideoView);
ssaiComponent.processVideo(ssaiUrl);

Bereitstellen eines Videos mit VMAP

Für diese Option stellen Sie ein Video mit einer Quelle bereit, die eine VMAP-Eigenschaft mit der SSAI-Asset-URL enthält. Gehen Sie wie folgt vor, um das SSAI-Plugin zu verwenden:

  1. Um VideoCloud-SSAI-Inhalte anzufordern, muss Ihr Konto zunächst für die Verwendung von Dynamic Delivery und SSAI konfiguriert werden. Einzelheiten finden Sie im Übersicht über die Video Cloud-SSAI dokumentieren.
  2. Als Nächstes müssen Sie ein Ad Configuration-Objekt (adConfig) erstellen. Einzelheiten finden Sie im Serverseitige Anzeigeneinstellungen konfigurieren dokumentieren. Beachten Sie, dass das Brightcove Native SDK für Android selbst keine Funktionen zum Erstellen oder Verwalten von adConfigs enthält.
  3. Verwenden Sie die adConfig's id Eigenschaft als URL-Parameter (ad_config_id ) zum Methodenaufruf Catalog hinzugefügt: 
    findVideoById

    oder

    findVideoByReferenceId
  4. Nachdem Sie das Video-Objekt mit der findVideo-Methode abgerufen haben, rufen Sie die folgende Methode im SSAI-Plugin auf: 
    SSAIComponent.processVideo(Video ssaiVideo)

Hier ist ein Beispiel:

Catalog catalog = new Catalog(eventEmitter, myAccountId, myPolicyKey);
SSAIComponent plugin = new SSAIComponent(appContext, getBrightcoveVideoView());

// Set the ad_config_id URL parameter
HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
        .addQueryParameter("ad_config_id", myAdConfig)
        .build();

// Find the Video by ID
catalog.findVideoByID(myVideoId, httpRequestConfig, new VideoListener() {
        @Override
        public void onVideo(Video video) {

        // Process the Video
        plugin.processVideo(video);
    }
});

Mit Companion-Anzeigen arbeiten

Wenn Ihre Anzeigen Companions enthalten, können Sie diese ganz einfach in Ihrer App schalten, indem Sie eine ViewGroup um das Companion-Bild zu halten, indem Sie Folgendes anrufen:

SSAIComponent.addCompanionContainer(ViewGroup companionContainer)

Hier ist ein Beispiel. Fügen Sie zunächst das FrameLayout zu Ihrem Aktivitäts- oder Fragment-Layout hinzu:

// my_activity.xml
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
                xmlns:app="http://schemas.android.com/apk/res-auto"
                android:layout_width="match_parent"
                android:layout_height="match_parent">
 ...
  <FrameLayout
        android:id="@+id/ad_frame"
        android:layout_width="300dp"
        android:layout_height="250dp"
        android:visibility="invisible" />
…
</RelativeLayout>

Dann benutze das FrameLayout um es an das Plugin zu übergeben:

ViewGroup viewGroup = findViewById(R.id.ad_frame);
plugin.addCompanionContainer(viewGroup);

// process your Video
plugin.processVideo(video);

Umgang mit Ereignissen

Bei SSAI-spezifischen Veranstaltungen können Sie diese über die EventEmitter. Diese Veranstaltungen finden Sie im SSAIEventType Klasse.

Die START_AD_SEGMENT und END_AD_SEGMENT Ereignisse wurden ersetzt durch EventType.AD_BREAK_STARTED und EventType.AD_BREAK_COMPLETED bzw. Einzelheiten finden Sie unter Anhang B und Anhang C.

Um relative und absolute Positionen zu erhalten, rufen Sie die com.brightcove.ssai.AdPod mit dem SSAIEvent.AD_POD Schlüssel. Hier ist ein Beispiel:

private void registerEventHandlers() {
  EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
  eventEmitter.on(EventType.AD_BREAK_STARTED, new EventListener() {
    @Override
    public void processEvent(Event event) {
      Object adPodObject = event.getProperties().get(SSAIEvent.AD_POD);
      if (adPodObject instanceof AdPod) {
        AdPod adPod = (AdPod) adPodObject;
        adPod.getRelativeStartPosition();
        adPod.getAbsoluteStartPosition();
        adPod.getAbsoluteEndPosition();
      }
    }
  });
}

Umgang mit Fehlern

Alle Fehler werden dem Entwickler mit dem EventType.ERROR Veranstaltung, wie hier gezeigt:

eventEmitter.on(EventType.ERROR, new EventListener() {
  @Override
  public void processEvent(Event event) {
    String errorMessage = event.getProperty(Event.ERROR_MESSAGE, String.class);
    Throwable error = event.getProperty(Event.ERROR, Throwable.class);
    Log.w(TAG, String.format("Received the error message: %s with error: %s", errorMessage, error));
  }
});

IAB Android-Bibliothek

Das SSAI-Plugin verwendet eine interne Bibliothek, die für das Parsen des VMAP-Objekts verantwortlich ist. Wenn Ihre App direkt Objekte aus dieser Bibliothek verwendet (package com.brightcove.iab), überprüfen Sie die Änderungen in der neuesten Version.

    • com.brightcove.iab.ssai.BrightcoveSSAI.getContentLengthUs():long
    • com.brightcove.iab.ssai.BrightcoveSSAI.getPayloadLengthUs():long
    • com.brightcove.iab.vmap.AdBreak.getTimeOffsetUs():long

    • com.brightcove.iab.vast.Linear.getSkipOffsetUs():long
    • com.brightcove.iab.vast.NonLinear.getMinSuggestedDurationUs():long
    • com.brightcove.iab.vast.Icon.getDurationUs():long
    • com.brightcove.iab.vast.Icon.getOffsetUs():long
