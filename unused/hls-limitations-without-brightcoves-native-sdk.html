Überblick

Das Brightcove Player SDK für Android behebt die meisten Probleme bei der HLS-Videowiedergabe auf Android-Geräten. Einzelheiten zu dieser Lösung finden Sie im HLS-Wiedergabe mit dem Player SDK für Android dokumentieren.

Wenn Sie Androids verwenden müssen MediaPlayer bei der HLS-Wiedergabe können Sie auf einige Einschränkungen stoßen. Weitere Informationen zu diesen Einschränkungen finden Sie im folgenden Abschnitt.

Bekannte Einschränkungen

Wenn Sie verwenden HLS-Wiedergabe mit dem Player SDK für Android , gelten die Einschränkungen in diesem Abschnitt nicht.

Die Probleme in diesem Abschnitt treten nur auf, wenn Sie Android verwenden MediaPlayer für HLS-Wiedergabe ohne Brightcove-Lösung. Hier sind die Einschränkungen:

  • Android 4.x
    • HLS-Streams suchen aufgrund eines Betriebssystemfehlers nur bis zum Anfang von Segmenten.

      Daher werden Mid-Roll-Anzeigen nur mit Nicht-HLS-Streams oder mit dem Brightcove Player SDK (Android 4.1 oder höher) unterstützt.

    • HLS-Streams melden aufgrund eines Betriebssystemfehlers nicht durchgängig die korrekte Videodauer.

      Das Brightcove Player SDK (Android 4.1 oder höher) weist diese Einschränkung nicht auf.

    • Timer, Zähler, Schaltflächen-Overlay "Weitere Informationen" und einige AdEvent-Ereignisse fehlen bei Google IMA-Midrolls aufgrund eines gemeldeten Fehlers im Google IMA SDK.

    • Auf einigen Samsung-Geräten sind Probleme mit Widevine-Inhalten aufgetreten. Die Widevine-Teams empfehlen ein Upgrade auf die neueste Android-Version.
  • Android 4.4.2
    • Die Wiedergabe wird gestartet, nach einigen Sekunden angehalten und dann von vorne wiedergegeben.

FAQ

Wie kann ich HLS auf Android zum Laufen bringen?

Um adaptives Streaming auf Android bereitstellen zu können, bietet das Player SDK von Brightcove HLS-Wiedergabefunktionen für Android 4.1 und höher.

Was ist mit HLSe?

HLSe hat die gleichen Einschränkungen und Empfehlungen wie oben.
