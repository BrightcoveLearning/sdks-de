    Schritt für Schritt: Einfache TV-Video-App mit Swift

    In diesem Thema erfahren Sie, wie Sie mit dem Brightcove Player SDK for tvOS eine einfache Video-Wiedergabe-App erstellen, die auf einem Apple TV oder dem Xcode-Simulator abgespielt werden kann. Sie werden es mit der Programmiersprache Swift erstellen.

    Übersicht

    In diesem Beispiel wird ein Video wiedergegeben, das von der Brightcove Playback API zurückgegeben wurde, der neuesten und empfohlenen API, um Inhalte aus Ihrer Video Cloud-Bibliothek abzurufen.

    In diesem Beispiel werden Videos aus einem internen Array von Videodateien wiedergegeben.

    Um die Playback-API verwenden zu können, benötigen Sie einen Richtlinienschlüssel. Wenn Sie mit Richtlinienschlüsseln nicht vertraut sind, lesen Sie die Richtlinien-API-Übersicht.

    Erstellen Sie das Projekt

    Richten Sie ein Projekt in Xcode ein. Fügen Sie dann das SDK zusammen mit allen Abhängigkeiten zum Projekt hinzu.

    Richten Sie das Xcode-Projekt ein

    Erstellen Sie ein neues Xcode-Projekt für die App.

    1. Beginnen Sie mit der Erstellung eines neuen tvOS-Projekts in Xcode. Wählen Sie Neues Xcode-Projekterstellen aus.

    2. Wählen Sie eine tvOS Single View-Anwendung für die Vorlage und wählen Sie Weiteraus.

    3. Legen Sie die Projektinformationen wie folgt fest:

      • Bezeichnung des Produkts: Einfaches Video-Wiedergeben
      • Mannschaft: keine

        Das Feld Team ist optional. Wenn Sie Ihre App im App Store verteilen möchten, müssen Sie über das Apple Developer Program ein Team auswählen, zu dem Sie gehören. Der Teamname ist für das Signieren von Code erforderlich. In diesem Beispiel wählen wir keine aus.

      • Name der Organisation: Ihr Firmenname
      • Organisations-ID: com.beispiel-company
        Dies macht den Produktnamen für den App Store einzigartig.
      • Sprache: Swift

    4. Wählen Sie den Speicherort aus, an dem Sie Ihr Projekt speichern und erstellen möchten.

    5. Schließen Sie nun das Projekt (ja, schließen Sie es — das ist wichtig!)

    Fügen Sie das SDK und seine Abhängigkeiten zu Ihrem Projekt hinzu

    Die einfachste Möglichkeit, das SDK und seine Abhängigkeiten zu Ihrem Projekt hinzuzufügen, ist die Verwendung von CocoaPods.

    CocoaPods ist ein Abhängigkeitsmanager, der Ihrem Projekt Bibliotheken hinzufügt. Dies ist nicht erforderlich, erleichtert aber die Installation. Informationen zur Installation von CocoaPods finden Sie in den Anweisungen auf der CocoaPods Website.

    • CocoaPods Version 1.0 oder neuer ist erforderlich.
    • Durch die Verwendung von CocoaPods (dem pod Tool) in einem bestehenden Xcode-Projekt wird die Projektdatei geändert. Vielleicht möchten Sie vorher ein Backup erstellen.
    Um den einfach zu erstellen und zu aktualisieren Podfile, können Sie die folgenden Schritte in der CocoaPods Appausführen.

    1. Erstellen Sie in Ihrem Projektordner eine Nur-Text-Datei namens Podfile (keine Dateierweiterung).

    2. Fügen Sie der Podfile mit einem Texteditor die folgenden Codezeilen hinzu und speichern Sie sie. Dieser Code macht folgendes:

      • 1. Zeile: Definiert die GitHub-Quelle.
      • 3. Zeile: Legt fest, dass Pods statt statischer Bibliotheken Frameworks verwenden.
      • Zeile 5: Definiert die Version der tvOS-Plattform.

      • Zeilen 7-9: Installieren Sie das Brightcove Native Player SDK.

        1. Holen Sie sich die aktuelle Native SDK-Version aus dem Brightcove Native SDK for iOS Overview Dokument.
        2. Ersetzen Sie den Wert '6.3.11' durch die aktuelle SDK-Versionsnummer.

        3. Weitere Informationen zum Verwalten von Abhängigkeiten finden Sie im Artikel So verwalten Sie Abhängigkeiten mit CocoaPods .

      source 'https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks!

platform :tvos, '11.0' # (or whatever version you want)

target 'Simple-Video-Playback' do
	pod 'Brightcove-Player-Core/dynamic', '6.3.11'
end
      Hinweise:
      • Die target Eigenschaft wird für CocoaPods Version 1.0 und höher verwendet.
      • Sie müssen den Brightcove Native SDK-Kern v6.3+ zur Unterstützung der UI-Steuerelemente ansprechen.

    3. Öffnen Sie eine Terminalsitzung und navigieren Sie zu Ihrem Xcode-Projektordner für Simple-Video-Playure

    4. Geben Sie in der Terminalsitzung den Befehl ein

      pod install

      und drücke die Eingabetaste , um es auszuführen. Sie sollten eine Reihe von Nachrichten im Terminal sehen, die darauf hinweisen, dass das Brightcove Player SDK Ihrem Projekt hinzugefügt wurde.

    5. Beachten Sie die letzte Zeile, was wichtig ist - ab diesem Zeitpunkt müssen Sie die Datei Simple-Video-Playback.XCWorkspace in Xcode öffnen, nicht die Simple-Video-Playback.xcodeproj Datei. Der pod install Befehl erstellt diese .xcworkspace Datei für Ihr Projekt.

    Wenn Sie CocoaPods nicht verwenden möchten, können Sie die neueste Version des SDK herunterladen und sie manuell zu Ihrem Projekt hinzufügen.

    Verwalten der Mediensicherheit

    App Transport Security (ATS) setzt sichere Verbindungen zwischen Ihrer App und Webdiensten durch. Mit der Veröffentlichung des iOS 9 SDK durch Apple wurde eine neue Funktion namens App Transport Security (ATS) hinzugefügt.

    1. Eine der folgenden Situationen trifft auf Ihre App zu:

      • Standardmäßig verwendet das Brightcove Native SDK für iOS eine Quellenauswahlrichtlinie, um HTTPS über HTTP-Quellen zu wählen, damit Sie Ihre Apps mit aktiviertem ATS erstellen können.

        Das war's, du bist bereit, deine App zu starten.


      • Wenn Sie HTTP-Quellen verwenden oder andere HTTP-Anrufe in Ihrer App haben, kann folgende Fehlermeldung auftreten:

        App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

        Dies bedeutet, dass ATS aktiviert ist, Ihr System jedoch nicht so konfiguriert ist, dass es die ATS-Anforderungen erfüllt. Informationen zur Behebung dieser Situation finden Sie im Dokument Working with App Transport Security (ATS) .
