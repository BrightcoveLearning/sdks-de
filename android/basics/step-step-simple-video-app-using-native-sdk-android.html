Erstellen Sie ein Projekt

Erstellen Sie ein Projekt in Android Studio und verknüpfen Sie es dann mit Gradle mit dem Player-SDK.

Erstellen Sie ein Projekt in Android Studio

  1. Öffnen Sie Android-Studio.
    • Wenn Sie Android Studio zum ersten Mal öffnen, wählen Sie Starten Sie ein neues Android Studio-Projekt.
    • Wenn Sie die Menüs sehen, wählen Sie Datei > Neu > Neues Projekt.
  2. In dem Wählen Sie Ihr Projekt Fenster, wählen Leere Aktivität und klicke Nächste.

  3. Lieferwerte für die Name , Paketnamen , Sprache , und Sicherer Ort. In diesem Schnellstart werden die unten gezeigten Werte verwendet:

  4. Klicken Beenden.

  5. Android Studio funktioniert eine Weile und zeigt schließlich den Anfangszustand des Projekts an.

Verwenden Sie Gradle, um eine Verknüpfung zum Brightcove Player SDK für Android herzustellen

Da Gradle in Android Studio integriert ist, verwenden Sie es, um das Native SDK für Android zu Ihrem Projekt hinzuzufügen.

  1. Erweitere die Gradle-Skripte Abschnitt und öffnen Sie den zweiten build.gradle Datei, die mit dem aktuellen Modul in Ihrem PlayVideos-Projekt verknüpft ist.

  2. In dem build.gradle Datei, gehen Sie wie folgt vor:

    1. In dem android Abschnitt, fügen Sie Folgendes hinzu compileOptions Wert:
      Für ExoPlayer v2.9.0+ müssen Sie die Java 8-Unterstützung aktivieren. Fügen Sie dazu Folgendes zu den android Abschnitt Ihrer App build.gradle Datei. Einzelheiten finden Sie unter ExoPlayer von Android dokumentieren.
      android {
    compileSdkVersion 28
    defaultConfig {
        applicationId "com.brightcove.playvideos1"
        minSdkVersion 21
        targetSdkVersion 28
        versionCode 1
        versionName "1.0"
        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
    }
    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
    compileOptions {
        targetCompatibility JavaVersion.VERSION_1_8
    }
}
    2. Unter dem android Abschnitt, fügen Sie ein repositories Abschnitt für das Brightcove Maven-Repository. 
      repositories {
      maven {
          url 'https://repo.brightcove.com/releases'
      }
    }
    3. In dem dependencies fügen Sie die neueste Version des Native SDK für Android hinzu. Ersetzen Sie den Wert 6.10.0 unten mit der neuesten SDK-Version, die Sie im Überblick: Brightcove Natives SDK für Android dokumentieren. 
      dependencies {
  implementation fileTree(dir: 'libs', include: ['*.jar'])
  implementation 'com.android.support:appcompat-v7:26.1.0'
  implementation 'com.android.support.constraint:constraint-layout:1.0.2'
  testImplementation 'junit:junit:4.12'
  androidTestImplementation 'com.android.support.test:runner:1.0.1'
  androidTestImplementation 'com.android.support.test.espresso:espresso-core:3.0.1'
  implementation "com.brightcove.player:exoplayer2:6.10.0"
}
     

    Verwenden einer dynamischen Abhängigkeitsversion mit dem '+' Charakter wird in der Gradle-Community abgeraten. Die Verwendung der dynamischen Versionsverwaltung birgt ein erhebliches Risiko für Ihren Build-Prozess, da neuere APIs möglicherweise unerwartet mit dem Quellcode Ihrer App inkompatibel werden.

    Anmerkungen:
    • Brightcove empfiehlt, immer die neueste Version mit einer bestimmten Versionszeichenfolge zu verwenden. Überprüfen Sie beispielsweise die build.gradle und gradle.eigenschaften Dateien in der Android-Player-Beispiele Projekt.
    • Die ExoPlayer-Abhängigkeit enthält das Native SDK für Android, sodass Sie das SDK nicht separat einschließen müssen.
    • Googles ExoPlayer ist ein Open-Source-Mediaplayer auf Anwendungsebene für Android. Beachten Sie, dass wir das ExoPlayer 2-Framework verwenden. Einzelheiten finden Sie im Migration zum ExoPlayer 2-Framework dokumentieren.
  3. Synchronisieren Sie das Projekt, um die Änderungen der Gradle-Datei zu übernehmen.

Codieren Sie die Anwendung

Als Nächstes schreiben Sie den Code, um die App zu gestalten, die Videoliste zu erstellen und die Videos abzuspielen

Definieren Sie das Layout der App

Auch mit der leeren Aktivität erhalten Sie ein einfaches TextView Layout. Ersetzen Sie dies durch die BrightcoveExoPlayerVideoView , die die Ansicht für den Player definiert.

  1. Öffne die Datei app/res/layout/activity_main.xml.

  2. Klicken Text unten auf der Registerkarte der Datei, um die XML-Rohdaten anzuzeigen.

  3. Entfernen Sie das vorhandene TextView Element und fügen Sie a . hinzu BrightcoveExoPlayerVideoView Abschnitt, sodass das resultierende XML wie folgt aussieht. Hinweis für die spätere Verwendung id der Ansicht ist benannt brightcove_video_view.

    <?xml version="1.0" encoding="utf-8"?>
    <android.support.constraint.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:app="http://schemas.android.com/apk/res-auto"
        xmlns:tools="http://schemas.android.com/tools"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        tools:context="com.brightcove.playvideos.MainActivity">

    <com.brightcove.player.view.BrightcoveExoPlayerVideoView
        android:id="@+id/brightcove_video_view"
        android:layout_width="match_parent"
        android:layout_height="280dp"
        android:layout_gravity="center_horizontal|top"/>
    </android.support.constraint.ConstraintLayout>

Holen Sie sich ein Video und spielen Sie es ab

In diesem Abschnitt verwenden Sie die Catalog -Klasse, um ein einzelnes Video vom Video Cloud-Server abzurufen und dann abzuspielen.

Die com.brightcove.player.edge.Catalog -Klasse bietet asynchrone Methoden zum Abrufen von Videos und Wiedergabelisten aus dem Brightcove-Wiedergabe-API. Dies ist die neueste und empfohlene API zum Abrufen von Inhalten aus Ihrer Video Cloud-Bibliothek.

Nicht benötigten Code entfernen

  1. Ein Teil des Codes aus der vorherigen App wird nicht mehr benötigt. In dem onCreate() -Methode entfernen Sie den gesamten Code, nachdem Sie die Instanz instanziiert haben brightcoveVideoView .

  2. Bestätigen Sie Ihre onCreate() Methode sieht wie folgt aus:

    public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);
  }
}

Abrufen eines Videos aus dem Katalog

  1. Erfassen Sie in Video Cloud Studio die folgenden Informationen:
    • Konto-ID
    • Video-ID
    • Richtlinienschlüssel
     
    Anmerkungen:
    • Um auf die Playback-API zuzugreifen, benötigen Sie einen Richtlinienschlüssel. Wenn Sie mit Richtlinienschlüsseln nicht vertraut sind, lesen Sie die Übersicht über die Richtlinien-API.
    • Da die nativen SDKs keinen Ursprungsheader senden, können Sie keinen domänenbeschränkten Richtlinienschlüssel verwenden. Wenn Sie dies versuchen, erhalten Sie ein 403 Forbidden Antwort, wenn Sie versuchen, Inhalte aus Ihrer Video Cloud-Bibliothek abzurufen.

  2. Definieren Sie Ihre benutzerdefinierten Werte in Ihrem Projekt. Öffne das res/values/strings.xml Datei und aktualisieren Sie sie wie folgt:

    <?xml version="1.0" encoding="utf-8"?>
  <resources>

      <!-- Application name -->
      <string name="app_name">PlayVideos</string>

      <!-- A sample Brightcove Edge Account ID -->
      <string name="account">your account id</string>

      <!-- A sample Brightcove Edge Policy Key -->
      <string name="policy">your policy key</string>

      <!-- A sample Brightcove Video ID -->
      <string name="videoId">your video id</string>

  </resources>

  3. Gehe zurück zum MainActivity.java Datei und fügen Sie den folgenden Code hinzu:

    • Holen Sie sich den Event-Emitter aus dem SDK

    • Erstellen Sie eine Instanz von com.brightcove.player.edge.Catalog , indem Sie Ihre definierten Werte für Konto-ID und Richtlinienschlüssel aus dem vorherigen Schritt verwenden.

    // Get the event emitter from the SDK and create a catalog request to fetch a video from the
// Brightcove Edge service, given a video id, an account id and a policy key.
EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));

  4. Verwenden Sie die Kataloge findVideoByID() Methode mit deiner Video-ID und a VideoListener für den Rückruf.

    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

  // Add the video found to the queue with add().
  // Start playback of the video with start().
  @Override
    public void onVideo(Video video) {
    }

  @Override
    public void onError(String s) {
    }
});
    Die VideoListener identifiziert Objekte, die erwarten, ein Video-Objekt von einem asynchronen Vorgang zu empfangen. Bei einer erfolgreichen Operation, ein onVideo() Methode aufgerufen wird. Bei einem Fehler in der Bedienung wird ein onError() Methode aufgerufen wird. Android Studio hat die Signaturen der beiden benötigten Methoden eingefügt.

  5. In dem onVideo() Methode, füge das Video hinzu zu brightcoveVideoView , dann starten Sie das Video.

    // Start playback of the video with start().
  @Override
    public void onVideo(Video video) {
      brightcoveVideoView.add(video);
      brightcoveVideoView.start();
    }

  6. In dem onError() -Methode wirft die Fehlerzeichenfolge.

    @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }

  7. Der komplette Code für deine MainActivity Klasse sollte ungefähr so aussehen:

    public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

      // Add the video found to the queue with add().
      // Start playback of the video with start().
      @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}
  8. Führen Sie die App aus, um die Videowiedergabe zu bestätigen.

Eine Playlist abrufen und abspielen

Die com.brightcove.player.edge.Catalog -Klasse bietet asynchrone Methoden zum Abrufen von Videos und Wiedergabelisten aus dem Brightcove-Wiedergabe-API. Dies ist die neueste und empfohlene API zum Abrufen von Inhalten aus Ihrer Video Cloud-Bibliothek.

Nicht benötigten Code entfernen

Abrufen einer Playlist aus dem Katalog

  1. Die vorhandene Kataloginstanz funktioniert zum Abrufen einer Playlist, also keine Änderungen notwendig zu diesen Codezeilen:

Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
  2. In Video Cloud Studio Medien Modul, wählen Sie eine Playlist aus und kopieren Sie die Playlist-ID.

  3. Verwenden Sie die Kataloge findPlaylistByID() Methode mit deiner Playlist-ID und a PlaylistListener für den Rückruf.

    catalog.findPlaylistByID("2764931905001", new PlaylistListener() {
  @Override
  public void onPlaylist(Playlist playlist) {
  }

  @Override
  public void onError(String s) {
  }
});
    Die PlaylistListener identifiziert Objekte, die erwarten, ein Playlist-Objekt von einem asynchronen Vorgang zu erhalten. Bei einer erfolgreichen Operation, ein onPlaylist() Methode aufgerufen wird. Bei einem Fehler in der Bedienung wird ein onError() Methode aufgerufen wird. Android Studio hat die Signaturen der beiden benötigten Methoden eingefügt.

  4. In dem onPlaylist() Methode, rufen Sie die Videos aus der Wiedergabeliste ab, fügen Sie alle Videos zu . hinzu brightcoveVideoView , dann starten Sie das erste Video.

    @Override
    public void onPlaylist(Playlist playlist) {
      brightcoveVideoView.addAll(playlist.getVideos());
      brightcoveVideoView.start();
    }

  5. In dem onError() -Methode wirft die Fehlerzeichenfolge.

     @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }
