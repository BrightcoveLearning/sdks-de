    Übersicht: Brightcove Native SDK für Android

    Willkommen bei der Android SDK-Entwicklerdokumentation. Hier finden Sie die Informationen, die Sie für die Entwicklung mit dem Android SDK benötigen.

    Aufbau der Dokumentation

    Bei Brightcove bemühen wir uns, die Dokumentation strukturiert, sauber und intuitiv zu halten. Daher nutzen wir Github ReadMes für Details zu all unseren Plugins und Integrationen. Zu Ihrer Bequemlichkeit haben wir diese Plugins unten aufgelistet und verknüpft.

    Hinweise:

    • Das native SDK für Android von Brightcove unterstützt jetzt Android Jetpack (AndroidX) -Bibliotheken. Weitere Informationen finden Sie im Abschnitt SDK herunterladen

    Unterstützte Versionen

    Das Native Player SDK für Android bietet Unterstützung für die folgenden Android-Versionen:

    Aktive Unterstützung

    Brightcove bietet aktive Unterstützung für das neueste Android-SDK unter folgenden Android-Betriebssystemversionen:

    • 10.0
    • 9.0
    • 8,0, 8.1
    • 7.0, 7.1
    • 6.0

    Passive Unterstützung

    Brightcove bietet passive Unterstützung für die folgenden Android-Versionen:

    • 5.0, 5.1

    Support-Level verstehen

    Die Unterstützung ist in Bezug auf die [Hauptversion]definiert.[ Nebenversion] eines Betriebssystems (zum Beispiel: Android 8.1). Um die neuesten technologischen Verbesserungen von Google nutzen zu können, bieten wir zwei Supportstufen an:

    • Active (Aktiv): Brightcove wird Fehler auf diesen Betriebssystemversionen testen und beheben.
    • Passiv: Brightcove führt unter diesen Betriebssystemversionen Tests aus, Bugs werden allerdings nur fallabhängig behoben.

    Brightcove leistet im Allgemeinen aktiven Support für die Betriebssystemversionen, die von 80 % der Benutzer verwendet werden. Brightcove leistet passiven Support für alle anderen Versionen, die von mindestens 5 % der Benutzer verwendet werden.

    Verbundenes Fernsehen/Top Boxen einstellen

    Das Native Player SDK für Android bietet Unterstützung für die folgenden Android-basierten Set-Top-Boxen:

    • Android TV
    • Fire TV (FireOS 5.0+ Geräte)

    Zu den von uns getesteten Set-Top-Box-Geräten gehören:

    • Nexus-Spieler
    • Nvidia-Schild

    Das Brightcove Native SDK für Android-Sample-Apps funktioniert auf Android TV und Fire TV. Dies bedeutet, dass Ihre TV-basierten Apps Werbung, Content-Schutz (DRM), Live-Wiedergabe, benutzerdefinierte UI-Steuerelemente, Brightcove Analytics und QoE-Metriken enthalten können.

    In der Beispiel-App Styled Controls finden Sie Details zum Styling der UI-Steuerelemente in Ihrer TV-App.

    Erste Schritte

    Bevor Sie mit der Entwicklung mit dem Android SDK beginnen, sollten Sie einige Konzepte kennenlernen:

    1. Medien

      Die aktuelle Version des Native Player SDK unterstützt das Abspielen von Inhalten von Ihrem Video Cloud-Konto über die Playback-API oder als Remote-Assets mit Internet-zugänglichen URLs.

    2. Analytics

      Mit dem Native Player SDK erhalten Video Cloud-Kunden automatisch robuste Analysedaten, die in Video Cloud gemeldet werden.

    3. Gradle

      Gradle ist die Grundlage des Android SDK-Buildsystems und ist in Android Studio integriert.

    4. Maven-Repo

      Der Brightcove Native Player für Android Maven-Artefakte (Jar-Dateien) ist jetzt im Brightcove Maven Repo verfügbar - https://repo.brightcove.com/releases

      Einzelheiten zum Hinzufügen des Maven-Repos zu Ihrem Android-Projekt finden Sie im Dokument Build an App using the Native SDK for Android .
    5. Überblick über Android SDK

      Um Ihnen das Verständnis des SDK zu erleichtern, erfahren Sie, wie kritische Komponenten interagieren, um mit der Komplexität der Steuerung des nativen Videoplayers umzugehen.

    6. Erstellen Sie eine App mit dem Native SDK für Android

      Erstellen Sie Ihre erste App mit dem Brightcove Player SDK für Android, das ein Video aus Ihrem eigenen Backend-System abspielt. Video Cloud-Kunden rufen zusätzlich ein Video vom Brightcove Catalog-Service ab und geben es ab.

    7. Entwicklerforum für SDK

      Begleiten Sie uns im SDK-Entwicklerforum , in dem Fragen zu SDK von Brightcove Engineers und der Community beantwortet werden. Suche nach zuvor beantworteten Fragen oder poste eine neue. Als öffentliche Gruppe können Sie Zugriff erhalten, indem Sie Ihre E-Mail-Adresse angeben.

    8. Video-on-Demand-Schulungen

      Zeigen Sie eine Reihe von Schulungsvideos an, die Sie durch die Schritte zum Erstellen einer App in Android Studio führen, die ein Video und eine Wiedergabeliste aus Ihrer Video Cloud-Bibliothek abspielt.

    Hinweise

    Die folgenden Hinweise beziehen sich auf die neueste Version des Native SDK:

    • Beginnend mit Version 6.10.0 verwendet das Native SDK für Android von Brightcove die Jetpack-Bibliotheken (androidx.*) anstelle der ursprünglichen Support-Bibliotheken. Diese Änderung ist für Exoplayer Version 2.10.0 und höher erforderlich. Um Jetpack nutzen zu können, müssen Ihre Apps die folgenden Anforderungen erfüllen:

      • Migrieren Sie zu Jetpack (AndroidX) -Abhängigkeiten
      • Gradle Tools für Android 3.2.0
      • Gradle Version 4.6

      Weitere Informationen zur Migration auf JetPack finden Sie im offiziellen Leitfaden zur Android-Migration.

    • Die vom Brightcove Native SDK für Android unterstützte Mindest-SDK-Version für Android ist jetzt 19.
    • Für ExoPlayer v2.9.0+ müssen Sie die Unterstützung von Java 8 aktivieren. Fügen Sie dazu dem android Abschnitt der build.gradle-Datei Ihrer App Folgendes hinzu. Einzelheiten finden Sie im ExoPlayer-Dokument von Android . 
      android {
    compileOptions {
      targetCompatibility JavaVersion.VERSION_1_8
    }
  }

      Dies überschreibt die Standard-Java-Version basierend auf dem compileSdkVersion Projektwert.

    Die Unterstützung für ExoPlayer 1 wurde aus dem Native SDK für Android entfernt. Wenn Sie dies noch nicht getan haben, migrieren Sie bitte zum ExoPlayer 2 Framework.

    Core-SDK-Installation

    Die Installation des Brightcove Core SDK ist sehr einfach. Hierfür gibt es zwei Möglichkeiten:

    • Verwenden des Gradle-Build-Systems
    • Manuelles Herunterladen des SDK

    Verwenden des Gradle-Build-Systems

    Der einfachste Weg, das Brightcove SDK und seine Abhängigkeiten hinzuzufügen, besteht darin, das Gradle-Build-System in Android Studio zu verwenden.

    Die Verwendung einer dynamischen Abhängigkeitsversion mit dem Zeichen „+“ wird in der Gradle-Community empfohlen. Die Verwendung von dynamischer Versionierung bringt ein erhebliches Risiko für Ihren Build-Prozess mit sich, da neuere APIs möglicherweise unerwartet mit dem Quellcode Ihrer App nicht kompatibel sind.

    Brightcove empfiehlt, immer die neueste Version mit einer bestimmten Versionszeichenfolge zu verwenden. Sehen Sie sich beispielsweise die build.gradle- und gradle.properties-Dateien im Android-Player-Samples-Projekt an.

    Weitere Informationen zur Verwendung von Gradle finden Sie in der Dokumentation Add Build Dependencies von Android.

    Manuelles Herunterladen des SDK

    Sie können das Brightcove SDK auch wie folgt manuell herunterladen:

    1. Laden Sie einfach das SDK herunter und entpacken Sie es. Sie finden eine Reihe von Ordnern.
    2. Im SDK-Ordner finden Sie das Kern-SDK mit dem Namen android-sdk-x.x.x.aar , wobei x.x.x die neueste SDK-Versionsnummer ist.
    3. Fügen Sie die Datei android-sdk-x.x.x.aar zusammen mit allen Plugin-Dateien, die Sie benötigen, Ihrer Bibliothek hinzu.

    Werbung bei IMA

    Die Platzierung von VAST Ads wird normalerweise durch ein VMAP-Dokument festgelegt. VAST beschreibt eine Anzeige, während VMAP die Platzierung von Anzeigen in einem Video beschreibt. Einzelheiten zum Code finden Sie in den folgenden Abstattungsdaten:

    • adRuleSimasampleApp
    • adRulesimaWideVineModularSampleApp

    • basicimasampleApp

      Für das Basic IMA Sample wird empfohlen, dass Sie den BrightcoveVideoView in ändern BrightcoveExoPlayerVideoView , damit Sie das ExoPlayer-Framework verwenden.

     

    Bei der Arbeit mit IMA-Anzeigen empfiehlt es sich, zu überprüfen, ob Ihre Anzeigen außerhalb des SDK funktionieren, bevor Sie sie implementieren. Die folgenden Links sind hierfür hilfreich:

    • Google VAST Werbeantwort-Validator
    • Leitfaden für die Wiedergabeliste für mehrere Digitalvideos (VMAP)
    • Technisches Forum Google IMA

    Begleitwerbung

    Begleitwerbung und Click-Through-Werbung werden auf Android TV nicht unterstützt. Da es auf Android TV keine Unterstützung für Webbrowser gibt, gibt es kein Ziel für eine Clickthrough-URL.

    Mit ProGuard arbeiten

    Mit ProGuard können Sie sicherere Apps erstellen, wodurch es schwieriger wird, sie zurückzuentwickeln. ProGuard ist in das Android-Build-System integriert, sodass Sie es aktivieren oder ohne es erstellen möchten.

    Wenn Sie einen Fehler erhalten, wenn ProGuard in Gradle aktiviert ist, fügen Sie die folgenden Ausschlussregeln hinzu:

      -dontwarn com.google.**
  -dontwarn android.media.**
  -keep class android.media.** { *; }
  -keep class com.google.** { *; }
  -keep interface com.google.** { *; }
  -keep class com.google.ads.interactivemedia.** { *; }
  -keep interface com.google.ads.interactivemedia.** { *; }

    Um eine vollständige Liste der ProGuard-Regeln des Brightcove Native SDK zu erhalten, laden Sie das aktuelle Native SDK für Androidherunter und entpacken Sie es. Öffnen Sie den SDK-Ordner und sehen Sie sich die proguard.txt Datei an.

    Wiedergabe-Verhalten

    Möglicherweise bemerken Sie einige der folgenden Wiedergabeverhalten auf bestimmten Android-Geräten:

    Video-Pause

    Die Smart Pause-Funktion von Android auf Samsung-Geräten pausiert ein Video, wenn der Benutzer vom Gerätebildschirm wegschaut. Sie können diese Funktion deaktivieren, wenn Sie nicht möchten, dass Videos auf Smart Pause-fähigen Geräten pausiert werden.
