    SSAI-Plugin für das native SDK für Android

    In diesem Thema erfahren Sie, wie Sie das SSAI (Server-Side Ad Insertion) -Plugin mit dem Brightcove Native SDK für Android verwenden. Es wird auch beschrieben, wie Sie von der Verwendung des OnceUX-Plugins migrieren können.

    Verwenden des SSAI-Plugins

    Mit dem SSAI-Plugin können Sie Inhalte mit Dynamic Delivery und serverseitiger Werbung wiedergeben.

    Es gibt zwei Möglichkeiten, Inhalte mit serverseitigen Anzeigen wiedergeben zu können:

    1. Geben Sie eine URL für SSAI-Inhalte
    2. Bereitstellen eines Videos mit VMAP

    In beiden Fällen fordert das SSAI-Plugin die VMAP-Quelle an, analysiert sie und fügt sie automatisch dem VideoView hinzu.

    Geben Sie eine URL für SSAI-Inhalte

    Rufen Sie bei einer SSAI-Asset-URL die folgende Methode auf:

    SSAIComponent.processVideo(String url)

    Beispiel:

    String ssaiUrl = "https://onceux.host.com/path/to/content.once"
SSAIComponent ssaiComponent = new SSAIComponent(context, brightcoveVideoView);
ssaiComponent.processVideo(ssaiUrl);

    Bereitstellen eines Videos mit VMAP

    Für diese Option stellen Sie ein Video mit einer Quelle zur Verfügung, die eine VMAP-Eigenschaft mit der SSAI-Asset-URL enthält. Gehen Sie folgendermaßen vor, um das SSAI-Plugin zu verwenden:

    1. Um VideoCloud SSAI-Inhalte anfordern zu können, muss Ihr Konto zuerst für die Verwendung von Dynamic Delivery und SSAI konfiguriert werden. Einzelheiten finden Sie im Video Cloud SSAI-Übersichtsdokument .
    2. Als Nächstes müssen Sie ein Ad Configuration (AdConfig) -Objekt erstellen. Weitere Informationen finden Sie im Dokument Konfigurieren von serverseitigen Anzeigeneinstellungen . Bitte beachten Sie, dass das Brightcove Native SDK für Android selbst keine Funktionen zum Erstellen oder Verwalten von AdConfigs enthält.
    3. Verwenden Sie die id Eigenschaft der AdConfig als URL-Parameter (ad_config_id ), der dem Methodenaufruf Catalog hinzugefügt wurde: 
      findVideoById

      oder

      findVideoByReferenceId
    4. Nachdem Sie das Video-Objekt mit der FindVideo-Methode abgerufen haben, rufen Sie die folgende Methode im SSAI-Plugin auf: 
      SSAIComponent.processVideo(Video ssaiVideo)

    Beispiel:

    Catalog catalog = new Catalog(eventEmitter, myAccountId, myPolicyKey);
SSAIComponent plugin = new SSAIComponent(appContext, getBrightcoveVideoView());

// Set the ad_config_id URL parameter
HttpRequestConfig httpRequestConfig = new HttpRequestConfig.Builder()
        .addQueryParameter("ad_config_id", myAdConfig)
        .build();

// Find the Video by ID
catalog.findVideoByID(myVideoId, httpRequestConfig, new VideoListener() {
        @Override
        public void onVideo(Video video) {

        // Process the Video
        plugin.processVideo(video);
    }
});

    Arbeiten mit Begleitanzeigen

    Wenn Ihre Anzeigen über Gefährten verfügen, können Sie diese einfach in Ihrer App anzeigen, indem Sie ein ViewGroup Companion-Image bereitstellen, indem Sie Folgendes anrufen:

    SSAIComponent.addCompanionContainer(ViewGroup companionContainer)

    Hier ist ein Beispiel. Fügen Sie zuerst das FrameLayout zu Ihrer Aktivität oder Ihrem Fragment-Layout hinzu:

    // my_activity.xml
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
                xmlns:app="http://schemas.android.com/apk/res-auto"
                android:layout_width="match_parent"
                android:layout_height="match_parent">
 ...
  <FrameLayout
        android:id="@+id/ad_frame"
        android:layout_width="300dp"
        android:layout_height="250dp"
        android:visibility="invisible" />
…
</RelativeLayout>

    Verwenden Sie das dann, FrameLayout um es an das Plugin zu übergeben:

    ViewGroup viewGroup = findViewById(R.id.ad_frame);
plugin.addCompanionContainer(viewGroup);

// process your Video
plugin.processVideo(video);

    Umgang mit Ereignissen

    Bei SSAI-spezifischen Ereignissen können Sie diese mit dem anhören EventEmitter. Diese Ereignisse finden Sie in der SSAIEventType Klasse.

    Das START_AD_SEGMENT und END_AD_SEGMENT Ereignisse wurden ersetzt durch EventType.AD_BREAK_STARTED und EventType.AD_BREAK_COMPLETED beziehungsweise. Einzelheiten finden Sie in Anhang B und Anhang C.

    Um relative und absolute Positionen zu erhalten, erhalte den com.brightcove.ssai.AdPod mit dem SSAIEvent.AD_POD Schlüssel. Beispiel:

    private void registerEventHandlers() {
  EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
  eventEmitter.on(EventType.AD_BREAK_STARTED, new EventListener() {
    @Override
    public void processEvent(Event event) {
      Object adPodObject = event.getProperties().get(SSAIEvent.AD_POD);
      if (adPodObject instanceof AdPod) {
        AdPod adPod = (AdPod) adPodObject;
        adPod.getRelativeStartPosition();
        adPod.getAbsoluteStartPosition();
        adPod.getAbsoluteEndPosition();
      }
    }
  });
}

    Umgang mit Fehlern

    Alle Fehler werden dem Entwickler bei der Verwendung des EventType.ERROR Ereignisses zukommen, wie hier gezeigt:

    eventEmitter.on(EventType.ERROR, new EventListener() {
  @Override
  public void processEvent(Event event) {
    String errorMessage = event.getProperty(Event.ERROR_MESSAGE, String.class);
    Throwable error = event.getProperty(Event.ERROR, Throwable.class);
    Log.w(TAG, String.format("Received the error message: %s with error: %s", errorMessage, error));
  }
});

    Migrieren vom OnceUx Plugin

    Wenn Sie das OnceUX-Plugin bereits zum Abspielen Ihrer SSAI-Videos in Ihrer App verwenden, müssen Sie nur ein paar Schritte ausführen, um zum neuen SSAI-Plugin zu migrieren.

    1. Ersetzen Sie in der build.gradle-Datei Ihrer App 
      implementation 'com.brightcove.player:android-onceux-plugin:6.5.1'

      mit den folgenden: (benutze die neueste Plugin-Version)

      implementation 'com.brightcove.player:android-ssai-plugin:6.7.0'
    2. Ersetze die OnceUxComponent Klasse durch die SSAIComponent Klasse.
    3. Ersetzen OnceUxEventType.NO_AD_DATA_URL durch EventType.ERROR.
    4. Wenn Ihr Code eine der Konstanten verwendet, die in der OnceUxEventType Klasse gefunden wurden, ersetzen Sie diese durch SSAIEventType und SSAIEvent Konstanten gemäß ihrer Funktion:
      • Die Konstanten, die verwendet werden, um auf ein Ereignis zu hören, werden jetzt in der SSAIEventType Klasse gefunden. Beispiel: 
        SSAIEventtType.AD_DATA_READY
      • Die Konstanten, die als Schlüssel für Event-Eigenschaften verwendet werden, sind jetzt in der SSaaEvent-Klasse zu finden. Beispiel: 
        SSAIEvent.SSAI_AD
    5. Die SSAI- und OnceUx-Plugins teilen sich eine interne Bibliothek, die für die Analyse des VMAP-Objekts verantwortlich ist. Wenn Ihre App direkt Objekte aus dieser Bibliothek verwendet (package com.brightcove.iab), gibt es Änderungen in der neuen Version, die Sie möglicherweise nützlich finden.

    6. Weitere Informationen finden Sie in Anhang A.

      • Anhang C

    Diese Tabelle zeigt die Änderungen in der internen IAB Android Library 1.5.0:

      • com.brightcove.iab.ssai.BrightcoveSSAI.getPayloadLengthUs():long
      • com.brightcove.iab.vmap.AdBreak.getTimeOffsetUs():long
      • com.brightcove.iab.vmap.AdBreak.getDurationUs():long
      • com.brightcove.iab.vast.Linear.getDurationUs():long
      • com.brightcove.iab.vast.Linear.getSkipOffsetUs():long
      • com.brightcove.iab.vast.NonLinear.getMinSuggestedDurationUs():long
      • com.brightcove.iab.vast.Icon.getDurationUs():long
      • com.brightcove.iab.vast.Icon.getOffsetUs():long
      • com.brightcove.iab.vast.Tracking.getOffsetUs():long
    • Hinzugefügt AdBreak.POST_ROLL_TIME_OFFSET_US , um den Wert eines Postroll-AdBreak bei der Verwendung anzugeben AdBreak.getTimeOffsetUs().
    • Die BrightcoveSSAI Eigenschaftsgetter nach der Namenskonvention für Camelcase wurden hinzugefügt.
    • Veraltet ist die BrightcoveSSAI Eigenschaft veraltet, die nicht der camelcase-Namenskonvention folgt.
