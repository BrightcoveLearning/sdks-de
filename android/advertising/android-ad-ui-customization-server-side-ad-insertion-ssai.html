    Anpassung der Android-Anzeigenbenutzung mit serverseitiger Anzeigeneinfügung (SSAI)

    In diesem Thema erfahren Sie, wie Sie die während der serverseitigen Anzeigenwiedergabe angezeigten Benutzeroberflächenelemente mithilfe des SSAI-Plugins für das Brightcove Native SDK für Android verwalten und anpassen.

    Übersicht

    Wenn Sie mit serverseitigen Anzeigeneinfügungen (SSAI) arbeiten, können Sie mit dem Brightcove Native SDK für Android die Elemente der Anzeigenbenutzung anpassen. Weitere Einzelheiten zu SSAI finden Sie in den folgenden Themen:

    • Implementierung von serverseitigen Anzeigen mit den Native Player SDKs
    • Das SSAI-Plugin für das native SDK für Android

    Die Werbe-Benutzeroberfläche besteht aus zwei Hauptkomponenten:

    • Anzeigenüberlagerung (A)
    • Admedia-Controller (B)

    Ad-Overlay

    Das Ad Overlay besteht aus folgenden Komponenten:

    • Countdown für Gesamtdauer des Ad Pods (A)
    • Erfahren Sie mehr View (B)
    • Anzeigenansicht überspringen (C)
    Die Schaltfläche „Anzeige überspringen“ wird nur bei serverseitigem VOD-Ad-Einfügen (SSAI) unterstützt. Es wird derzeit nicht mit Live SSAI unterstützt.

    Admedia-Controller

    Der Ad Media Controller besteht aus diesen Komponenten von links nach rechts:

    • Play-Taste (A)
    • Anzeigennummer-Countdown (B)
    • Countdown für die Dauer einer Anzeige (C)
    • Vollbild-Taste (D)

    Verwalten von Ad-UI-Komponenten

    Informationen zum Aktivieren oder Deaktivieren von Ad-UI-Komponenten in der Ansicht finden Sie unter:

    • Ändern des VAST Dokuments
    • Festlegen der Anzeigenkonfiguration

    Ändern des VAST Dokuments

    Die Ansichten „ Weitere Informationen “ und „ Anzeige überspringen “ können basierend auf der Anzeigendefinition im VEISTen-Dokument angezeigt oder ausgeblendet werden.

    Die Ansicht „ Mehr erfahren “ wird nur angezeigt, wenn das Linear Creative eine ClickThrough URL enthält, wie hier gezeigt:

     <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear>
            ...
            <VideoClicks>
              <ClickThrough id="clickthrough">https://www.brightcove.com/en/</ClickThrough>
              <ClickTracking id="...”>...</ClickTracking>
            </VideoClicks>
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

    Die Ansicht „ Anzeige überspringen “ wird nur angezeigt, wenn im Linear Creative ein gültiger skipoffset Wert vorhanden ist:

     <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear skipoffset="00:00:03">
            ...
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

    Festlegen der Anzeigenkonfiguration

    Wenn Sie Player Enhancements for Live SSAI verwenden, können Sie Komponenten innerhalb Ihrer Anzeigenkonfiguration aktivieren oder deaktivieren. application_ad_configuration Das client_options Objekt ermöglicht es Ihnen, Ad UI Komponenten ein- oder auszuschalten.

    Die Änderung der Anzeigenbenutzeroberfläche mit der Anzeigenkonfiguration gilt nur für Live SSAI.
    {
  "application_ad_configuration": {
	"ad_configuration_description": "$YOUR_DESCRIPTION",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "$YOUR_AD_SERVER",
     "ad_configuration_client_sdk_enabled": true,
     "client_options": {
        "show_ad_break_remaining_time": true / false,
        "show_ad_remaining_time": true / false,
        "show_number_of_remaining_ads": true / false,
        "client_only_tracking": true / false
     }
  },
  "application_description": "$YOUR_DESCRIPTION"
}
    Wenn Sie client_options Komponenten verwenden, müssen Sie auf ad_configuration_client_sdk_enabled festgelegt haben true.

    Verwenden Sie das client_options-Objekt, um die folgenden Komponenten zu aktivieren/deaktivieren:

    • Show_ad_break_remaining_time: Der Countdown für die Gesamtdauer des Anzeigen
      • Beispiel: Ihr Video wird in 15 Sekunden fortgesetzt
    • Show_ad_remaining_time: Der Countdown für die Dauer einer Anzeige
      • Beispiel: ( 0:15)
    • Show_number_of_remaining_ads: Der Countdown für Anzeigennummern
      • Beispiel: Anzeige 1 von 3, Ad 2 von 3 usw.

    Anpassen des Ad-Overlays

    Sie können die Ansichten und den mit dem Ad Overlay verknüpften Standardtext anpassen.

    Ändern der Ansicht

    Um das Ad Overlay anzupassen, können Sie Ihre Version der Layoutdatei ssai_ad_overlay.xml im Res-Ordner Ihrer App ersetzen. Diese Datei enthält mehrere verschiedene unabhängige Layouts, wie hier gezeigt:

    Sie könnten beispielsweise die Daueransicht des Ad Pod überschreiben, indem Sie die Datei ssai_ad_pod_duration_countdown_view.xml aktualisieren. Sie können auch die gesamte Ad Overlay View überschreiben, indem Sie die Datei ssai_ad_overlay.xml ersetzen.

    Diese Layoutstruktur ermöglicht es Ihnen, bestimmte Komponenten zu überschreiben und gleichzeitig die Abwärtskompatibilität mit älteren SSAI-Plugin-Versionen beizubehalten.

    Erkunden Sie die Ad Overlay-Dateien mit Android Studio. Ab Version 6.9.0 des Native SDK für Android sieht die ssai_ad_overlay.xml des SSAI-Plugins folgendermaßen aus:

    <?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:background="@color/white">

   <include layout="@layout/ssai_ad_countdown_and_learn_more_overlay"/>
   <include layout="@layout/ssai_ad_skip_overlay"/>
</FrameLayout>

    Wenn Sie eine Komponenten-XML-Datei ersetzen, müssen Sie eine TextView mit der angegebenen ursprünglichen Ansichts-ID verwenden, um unbeabsichtigte Ergebnisse zu vermeiden. Einzelheiten finden Sie in der folgenden Tabelle

    Bezeichnung der Komponente Komponenten-ID Komponenten-Ansicht
    Ansicht der Dauer von Ad Po text_ad_countdown TextView
    Erfahren Sie mehr Ansicht text_ad_learn_more TextView
    Anzeigenansicht überspringen text_ad_skip TextView

    Ändern des Standardtextes

    Alle Ad Overlay-Komponenten sind TextViewsin der Datei string.xml Ihrer App definiert. Um den Anzeigetext zu ändern, überschreiben Sie ihn, indem Sie ein erstellen string oder plurals Element mit derselben Zeichenfolgen-ID, die vom Native SDK für Android verwendet wird.

    Die Daueransicht des Ad Pods verwendet die folgenden Zeichenketten:

    <plurals name="ssai_message_ad_break_duration_countdown">
   <item quantity="one">Your video will resume in\n%1$d second</item>
   <item quantity="other">Your video will resume in\n%1$d seconds</item>
</plurals>
<string name="ad_info_now_text">Your video will resume now</string>
<string name="ad_buffering_text">Your ad is buffering …</string>

    Die Ansicht „ Mehr erfahren “ verwendet die folgende Zeichenfolge:

    <string name="ssai_message_learn_more">Learn More >> </string>

    Die Ansicht „ Anzeige überspringen “ verwendet die folgenden Zeichenfolgen:

    <plurals name="you_can_skip_text">
   <item quantity="one">You can skip this ad in\n%d second</item>
   <item quantity="other">You can skip this ad in\n%d seconds</item>
</plurals>
<string name="skip_text">Skip Ad</string>

    Anpassen des Ad Media Controllers

    Sie können die Ansichten und den Standardtext anpassen, der mit dem Ad Media Controller verknüpft ist.

    Ändern der Ansicht

    Um den Ad Media Controller anzupassen, können Sie Ihre Version der Layoutdatei default_ssai_ad_media_controller.xml im Ordner „ res “ Ihrer App ersetzen.

    Dies überschreibt das Standardlayout, das im SSAI-Plugin verwendet wird. Jedes Mal, wenn eine Anzeige erkannt wird, wechselt das SSAI-Plugin zu diesem Layout und kehrt dann zum Standard-Controller zurück, wenn die Anzeige beendet ist.

    Wenn Sie mit Ihrem eigenen Layout arbeiten, verwenden Sie denselben Ansichtstyp und Ansichts-IDs wie ursprünglich angegeben, um unbeabsichtigte Ergebnisse zu vermeiden. Erkunden Sie das Layout von Ad Media Controller mit Android Studio. Für die SSAI-Plugin-Version 6.9.0 sieht die default_ssai_ad_media_controller.xml wie folgt aus:

    <com.brightcove.player.mediacontroller.BrightcoveControlBar
   xmlns:android="http://schemas.android.com/apk/res/android"
   xmlns:tools="http://schemas.android.com/tools"
   android:id="@+id/brightcove_control_bar"
   style="@style/SSAIAdControlBar"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:layout_gravity="bottom"
   android:background="@color/bmc_background"
   android:orientation="vertical"
   android:padding="8dp">

   <LinearLayout
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:gravity="bottom"
       android:orientation="horizontal">

       <Button style="@style/BorderlessButton"
           android:id="@id/play"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:text="@string/brightcove_controls_play"
           android:visibility="gone"
           tools:visibility="visible"/>

       <include layout="@layout/ssai_ad_number_countdown_view"/>

       <include layout="@layout/ssai_single_ad_duration_countdown_view"/>

       <View
           android:id="@id/two_line_spacer"
           android:layout_width="0dp"
           android:layout_height="0dp"
           android:layout_weight="1"
           android:visibility="gone"
           tools:visibility="visible"/>

       <Button style="@style/BorderlessButton"
           android:id="@id/full_screen"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:padding="4dp"
           android:text="@string/brightcove_controls_enter_full_screen"
           android:visibility="gone"
           tools:visibility="visible"/>
   </LinearLayout>

</com.brightcove.player.mediacontroller.BrightcoveControlBar>

    Die SSAI Ad Media Controller-Komponentenansichtstypen und View-IDs werden hier beschrieben:

    Bezeichnung der Komponente Komponenten-ID Komponenten-Ansicht
    Play-Button play Schaltfläche
    Schaltfläche „Vollbild“ full_screen Schaltfläche
    Countdown für Anzeigenum text_ad_number_countdown TextView
    Countdown für die Dauer einer Anzeige text_single_ad_duration_countdown TextView

    Sehen Sie sich die Beispiel-App für angepasste Steuerungen an, in der der Standard-Medien-Controller für das Hinzufügen neuer Schaltflächen angepasst ist. Sie können den gleichen Ansatz anwenden, um Ihren SSAI-Ad Media Controller anzupassen.

    Ändern des Standardtextes

    Alle Ad Media Controller-Komponenten sind TextViewsin der Datei string.xml Ihrer App definiert. Um den Anzeigetext zu ändern, überschreiben Sie ihn, indem Sie ein erstellen string oder plurals Element mit derselben Zeichenfolgen-ID, die vom Native SDK für Android verwendet wird.

    Die Countdown-Ansicht für Anzeigennummern verwendet die folgende Zeichenfolge:

    <string name="ssai_message_ad_number_countdown">Ad %1$d of %2$d</string>

    Die Countdown-Ansicht für die Dauer einer Anzeige verwendet die folgende Zeichenfolge:

    <string name="ssai_message_ad_duration_countdown">(%1$s)</string>

    Zusammenfassung

    Sie haben einige Optionen, um die SSAI-UI-Elemente anzupassen. Der Grad der Komplexität hängt davon ab, was Sie erreichen möchten. Von geringer bis hoher Komplexität können Sie:

    • Konfigurieren/verbergen Sie bestimmte Komponenten durch Ihre VAST Konfiguration
    • Anzeigen/Ausblenden von Komponenten mit dem client_options Objekt in Ihrer Anzeigenkonfiguration (derzeit nur Live SSAI)
    • Ändern Sie den angezeigten String-Wert
    • Überschreiben des UI-Layouts

    Bitte teilen Sie Ihre Ideen oder Rückmeldungen im Brightcove Native Player SDKs Forummit.
