    Brightcove Native SDK für Android - Versionsinformationen

    In diesem Thema finden Sie die neuesten Informationen zu Änderungen und Verbesserungen des Brightcove Native SDK für Android.

    Um der Mailingliste beizutreten und Versionsankündigungen für die Native SDKs zu erhalten, melden Sie sich hieran.

    Aktuelle Version

    Version 6.16.0

    14. Okt 2020

    Cast Plugin-Änderungen:

    • Das Cast-Plugin wurde überarbeitet und unterstützt jetzt den Brightcove Cast Receiver v2.0.

    • Weitere Informationen und Codebeispiele finden Sie in der Brightcove Android Native SDK-Dokumentation und in den Beispiel-Apps des Brightcove Android Native SDK.

    Frühere Versionen

    Version 6.15.1

    22. September 2020

    SDK-Änderungen:

    • Das Brightcove Native SDK für Android unterstützt jetzt Arabisch (RTL) Sprache in Player-Steuerelementen und UI-Labels.

    • Hinweis: Während der Text der arabischen Sprache (RTL) unterstützt wird, werden derzeit nur Layouts der Player-Steuerung von links nach rechts (LTR) unterstützt.

    • Es wurde ein Problem behoben, durch das bestimmte video_view und video_engagement Analytics-Ereignisparameter nicht westlich-arabische Ziffern für die folgenden Parameter enthalten konnten:

      • video_view:
        • start_time_ms
        • load_time_ms
      • video_engagement:
        • range
        • video_seconds_viewed

    Cast Plugin-Änderungen:

    • Das Cast-Plugin unterstützt jetzt arabischen (RTL) Text in Cast-Steuerelementen und UI-Labels.

    • Bitte beachten Sie, dass sich die Unterstützung der arabischen Sprache nicht auf die Displays (verstrichene Zeit und Dauer des Videos) im erweiterten Cast-Controller erstreckt.

    • Die Anzeige dieser Komponenten wird von der Google Cast-API gesteuert.

    Änderungen des Offline Playback Plugins

    • Das Offline Playback-Plugin unterstützt jetzt arabischen (RTL) -Text in UI-Labels.

    Änderungen am Pulse Plugin:

    • Das Pulse-Plugin unterstützt jetzt arabischen (RTL) Text in Anzeigensteuerelementen und UI-Labels.

    SSAI-Plugin-Änderungen:

    • Das SSAI-Plugin unterstützt jetzt arabischen (RTL) Text in Anzeigensteuerelementen und UI-Labels.

    Version 6.15.0

    25 August 2020

    SDK-Änderungen:

    • Zusätzliche Nullprüfung auf Beschriftung URIs implementiert

    ExoPlayer-Änderungen:

    • Es wurde eine Regression in der ExoPlayerVideoDisplayComponent setPeakBitrate() Methode behoben, bei der die angegebene Bitrate ignoriert und stattdessen die höchste Bitrate ausgewählt wurde.

    Änderungen des IMA-Plugins:

    • Das Google IMA SDK wurde auf Version 3.19.4 aktualisiert. Einige der Änderungen umfassen:

      • SdkFactory.createAdDisplayContainer() ist veraltet
      • AdDisplayContainer.setPlayer(VideoAdPlayer) ist veraltet
      • AdDisplayContainer.setAdContainer(ViewGroup) ist veraltet

      • AdsRequest.setAdDisplayContainer(AdDisplayContainer) wird entfernt

        Die vollständige Liste der Änderungen finden Sie im Versionsverlauf des Google IMA Android SDKs.

    • Der benutzt GoogleIMAComponent jetzt einen Baumeister für seinen Bau.

    • Das IMA-Plugin erstellt automatisch das AdDisplayContainer Verwendung der BrightcoveVideoView standardmäßig.

      Sie können eine der AdDisplayContainerFactory zur Verfügung stellen GoogleIMAComponent.Builder , um Ihre eigenen bereitzustellen AdDisplayContainer.

    • Die folgenden Methoden waren in der GoogleIMAVideoAdPlayer Klasse veraltet:
      • playAd(). Use playAd(AdMediaInfo) statt
      • loadAd(String). Use loadAd(AdMediaInfo, AdPodInfo) statt
      • stopAd(). Use stopAd(AdMediaInfo) statt
      • resumeAd(). Use playAd(AdMediaInfo) statt

      • pauseAd(). Use pauseAd(AdMediaInfo) statt

    • Es wurde ein Problem behoben, bei dem das IMA-Plugin die AD_PAUSED/AD_RESUME Ereignisse beim Aufruf GoogleIMAVideoAdPlayer.pauseAd bzw. GoogleIMAVideoAdPlayer.resumeAd nicht ausgab.

      Weitere Informationen finden Sie in der Brightcove Android SDK-Übersicht auf der Brightcove Support-Website.

    Änderungen am VMAP Plugin:

    • Das veraltete VMAP-Plugin wurde aus dem Brightcove Native SDK für Android entfernt. Für die Bereitstellung von VMAP-Anzeigen verwenden Sie bitte weiterhin das IMA-Plugin.

    OnceUx Plugin-Änderungen:

    • Das veraltete OnceUX-Plugin wurde aus dem Brightcove Native SDK für Android entfernt. Bitte benutzen Sie stattdessen das SSAI-Plugin.

    Version 6.14.1

    07. August 2020

    SSAI-Plugin-Änderungen:

    • Behebt ein Problem, bei dem das CustomClick Element nicht analysiert werden konnte.

    • Behebt ein Problem beim Parsen des VAST Extension-Elements.

    Version 6.14.0

    08. Juli 2020

    SDK-Änderungen:

    • Verbesserte Integration mit Android TalkBack (und Samsung Voice Assistant).

    • Der VideoParser ignoriert jetzt jede DRM-Quelle, die keine Schlüsselsysteme von Widevine enthält.

    • Es wurde ein Problem behoben, bei dem der VideoParser eine Thumbnail-VTT-Quelle, die nicht die Bandbreiteneigenschaft enthält, nicht analysieren konnte.

    • Es wurde ein Problem auf Android-TV-Plattformen behoben, bei dem die Schaltfläche „Player-Optionen“ nicht angezeigt wurde, nachdem serverseitige Werbeblöcke zum Einfügen von Anzeigen abgeschlossen waren

    Änderungen des IMA-Plugins:

    • ExoPlayer wird jetzt für die Anzeigenwiedergabe anstelle von MediaPlayer verwendet.

    Version 6.13.3

    21. Mai 2020

    SDK-Änderungen:

    • Verbessertes Rendern von Ruby-Text in geschlossenen Untertiteln mit Sidecar. Eine ähnliche Verbesserung für manifestierte Bildunterschriften steht vor der Tür.

    Cast Plugin-Änderungen:

    • Es wurde ein Problem behoben, bei dem die Aktivität bei Verwendung des Chromecast-Plugins nicht wie erwartet veröffentlicht wurde.

    Änderungen des IMA-Plugins:

    • Unterstützung für Zielgruppensegment-Targeting (Ad) hinzugefügt, wobei Schlüssel/Wert-Paare zum cust_params-Parameter der Anzeigen-Tag-URL hinzugefügt werden.

    SSAI-Plugin-Änderungen:

    • Unterstützung für Zielgruppensegment-Targeting (Ad) hinzugefügt, wobei Schlüssel/Wert-Paare zur VMAP-Quell-URL hinzugefügt werden.

    • Es wurde ein Problem behoben, bei dem eine leere oder fehlerhafte URL in einem VAST Tracking-Element einen App-Absturz verursachen kann

    Version 6.13.2

    24. Apr 2020

    Änderungen der Offline-Wiedergabe:

    • Die FileDownloader-Abhängigkeit wurde auf v1.7.7 aktualisiert

      Dieses Upgrade behebt ein Problem, bei dem das Herunterladen, während sich die App im Hintergrund befindet, unerwartet angehalten werden kann.

    Änderungen des Thumbnail-Plugins:

    • Es wurde ein Ausrichtungsproblem zwischen dem Scrubber und der Miniaturvorschau auf Android TV behoben

    Version 6.13.1

    16. April 2020

    SDK-Änderungen:

    • Unterstützung für HEVC-Videos wurde hinzugefügt.

      Auf Geräten, die HEVC unterstützen (mit API-Level 21 und höher) ist die Wiedergabe von HEVC jetzt automatisch verfügbar. Technische Details finden Sie in der Dokumentation.

    Änderungen am Pulse Plugin:

    • Seek ohne Werbeunterstützung bei der Einstellung adsDisabled hinzugefügt VideoPlaybackController

    • Unterstützung für das Abspielen von Pulse Passback-Anzeigen hinzugefügt

    • Es wurde eine Nachricht hinzugefügt, die angibt, wann der Benutzer in einer Pause-Anzeige ist

    Änderungen des Thumbnail-Plugins:

    • Erhöhte Größe des in Tablets gezeigten Vorschaubilds

    • Erhöhte Geschwindigkeit beim Ziehen von Miniaturbildern auf Android TV

    SSAI-Plugin-Änderungen:

    • Es wurde ein Problem behoben, bei dem die Untertitel möglicherweise nicht mit dem Video synchronisiert sind

    Version 6.12.0

    5. Mrz 2020

    SDK-Änderungen:

    • Veraltet BrightcoveSurfaceView.isVideo360Supported(). Basierend auf aktuell unterstützten Versionen gibt es immer true zurück.

    Pulse Plugin:

    • Mit dieser Version implementiert das Brightcove Native SDK für Android das Pulse Advertising Plugin.

      Bitte beachten Sie, dass diese neue Funktion in begrenzter Verfügbarkeit ist. Wenden Sie sich an Ihren Account Manager, um weitere Informationen zu

    Version 6.11.0

    7. Februar 2020

    SDK-Änderungen:

    • Unterstützung für den Brightcove Playback Rights Management Service wurde hinzugefügt.

      Bitte beachten Sie, dass diese neue Funktion in begrenzter Verfügbarkeit ist.

    • Breaking Änderung: Das Playlist-Objekt ist jetzt unveränderlich.

    • Das Playlist-Objekt kann jetzt eine Mischung aus Videos mit Status Status.OK und Status.ERROR

    • Playlist.getErrorVideos(): Diese neue Methode gibt nur Videos mit dem Status Status.ERROR

    • Playlist.getAllVideosAndErrors(): Diese neue Methode gibt alle Videos mit dem Status Status.OK und der gleichen Reihenfolge zurück Status.ERROR , die von der Playback Rights Management Service-Antwort zurückgegeben wird

    • Hinweis: Playlist.getVideos() und Playlist.getCount() verhalten sich genauso wie frühere Versionen, das heißt, sie berücksichtigen nur die Videos mit Status Status.OK

    • hinzugefügt ErrorListener.onError(List) Rückruf.

    • Veralteter ErrorListener.onError(String) Rückruf.

    • Das Video-Objekt hat jetzt einen Statuscode, der entweder Status.OK oder sein kann Status.ERROR. Sie können den Status über abrufen Video.getStatus():Status.

    • Status.OK: Das Video hat alle erwarteten Eigenschaften und hat es nicht. Video.Fields.ERRORS

    • Status.ERROR: Das Video hat Video.Fields.ERRORS in seinen Eigenschaften in Form von List<CatalogError>.

    VMAP Plugin:

    • Das VMAP-Plugin ist veraltet. Für clientseitige Werbung mit VMAP und VASTING-Dokumenten verwenden Sie bitte das IMA-Plugin.

    Version 6.10.1

    17. Jan 2020

    ExoPlayer-Änderungen:

    • Ein Absturz beim Versuch, einen HLS Live-Stream abzuspielen, wurde behoben.

    Änderungen der Offline-Wiedergabe:

    • Es wurde ein Problem behoben, bei dem einige VTT-Untertiteldateien nicht erfolgreich heruntergeladen wurden.

    Cast-Plugin:

    • Neue CastMediaUtil.toMediaInfo Methoden ohne den unnötigen Android Context-Parameter hinzugefügt.

    • Veraltete CastMediaUtil.toMediaInfo Methoden mit dem unnötigen Android Context-Parameter.

    Version 6.10.0

    19 Dez 2019

    SDK-Änderungen:

    • Breaking Änderung: Das Brightcove Android Native SDK wird jetzt mit AndroidX (JetPack) -Bibliotheken erstellt. Weitere Informationen zu dieser Änderung und den erforderlichen Schritten zur Migration Ihres Anwendungscodes finden Sie auf den Android SDK-Supportseiten.

    • Es wurde ein Problem mit der backgroundOpacity Methode behoben BrightcoveCaptionStyle , bei dem diese Methode die Deckkraft des Hintergrunds für Beschriftungen nicht wie erwartet einstellte.

    • Es wurde ein Problem behoben, bei dem die Fullscreen-Funktionalität im nicht wie erwartet funktionierte BrightcovePlayerFragment.

    ExoPlayer-Änderungen:

    • Die ExoPlayer-Unterstützung im Brightcove Android Native SDK wurde auf 2.10.8 aktualisiert.

    • Es wurde ein Problem bei der Auswahl der Audiospur behoben, bei dem Wiedergaben mit höherer Bitrate nach dem Wechsel der Sprache nicht ausgewählt werden konnten.

    Änderungen des IMA-Plugins:

    • Fügt Unterstützung für Live-Inhalte mit kundenseitigen Pre-Roll-Anzeigen hinzu.

    SSAI-Plugin-Änderungen:

    • Es wurde ein Problem behoben, bei dem die Schaltfläche Untertitel während der Inhaltswiedergabe nicht konsistent angezeigt wurde. Bitte beachten Sie, dass dieses Problem derzeit nur für Mobilgeräte behoben ist. Eine Lösung für Android-TV-Geräte wird in einer kommenden Version folgen.

    Version 6.9.0

    22 Okt 2019

    SDK-Änderungen:

    • Das Android SDK zielt jetzt auf Android 10 (targetSdkVersion 29) ab und unterstützt es.

    • Behebt eine RuntimeException: „Canvas: Versuch, eine recycelte Bitmap zu verwenden“, wenn versucht wird, das Posterbild anzuzeigen.

    • Die folgenden Conveniency-Konstanten als HttpRequestConfig-Abfrageparametern wurden hinzugefügt:

      • HttpRequestConfig.KEY_DELIVERY_RULE_CONFIG_ID Wird verwendet, um eine ID für die Konfiguration von Lieferregeln hinzuzufügen
      • HttpRequestConfig.KEY_AD_CONFIG_ID Wird verwendet, um eine SSAI-Anzeigenkonfig-ID hinzuzufügen

    Änderungen an ExoPlayer v1:

    • Alle Unterstützung für ExoPlayer 1.x wurde aus dem Brightcove Native SDK für Android entfernt. Bitte stelle sicher, dass du den ExoPlayer v2 benutzt. Weitere Informationen zum Migrationsprozess finden Sie unter https://support.brightcove.com/migrating-exoplayer-2-framework

    SSAI-Plugin-Änderungen:

    • Live- und Live-DVR-Streams mit serverseitiger Anzeigeneinfügung (Live SSAI) werden jetzt unterstützt.

    • Breaking Änderung: Die Event.PLAYHEAD_POSITION Eigenschaft in beiden Ereignissen EventType.PROGRESS und enthält jetzt die absolute Abspielposition EventType.AD_PROGRESS , bei der es sich um dieselbe Abspielkopfposition handelt, die der zugrunde liegende Spieler gegeben hat.
      Bitte benutze Event.PROGRESS_BAR_PLAYHEAD_POSITION stattdessen, um die relative Abspielposition zu erhalten.

    • Die Event.PROGRESS_BAR_PLAYHEAD_POSITION Eigenschaft wurde EventType.PROGRESS sowohl für EventType.AD_PROGRESS Ereignisse eingeführt, die die relative Abspielposition enthalten ( Event.PLAYHEAD_POSITION wie auch in älteren Versionen).

    • Ad-Steuerelemente für Android TV hinzugefügt.

    • Es wurde ein Problem behoben, bei dem der Rückspulknopf nach aufeinanderfolgenden Klicks nicht ordnungsgemäß zurückgesetzt wird.

    Version 6.8.1

    31. Mai 2019

    ExoPlayer2 Änderungen:

    • ExoPlayer2 wurde aufgrund eines Problems mit HLS Multiple Audio Tracks in Version 2.9.6 auf Version 2.9.3 heruntergestuft.

    Änderungen der Offline-Wiedergabe:

    • Unterstützung für den Playback-Authentifizierungsdienst (PAS) hinzufügen.

    • Die folgenden Methoden werden dem hinzugefügt com.brightcove.player.edge.AbstractOfflineCatalog:

      • findOfflineVideoById(String, @NonNull HttpRequestConfig): Video
      • findOfflineVideoById(String, @NonNull HttpRequestConfig, @NonNull OfflineCallback<Video>): void
      • requestPurchaseLicense(@NonNull Video, @NonNull EventListener, @NonNull HttpRequestConfig): void
      • requestRentalLicense(@NonNull Video, @NonNull Date, long, @NonNull EventListener, @NonNull HttpRequestConfig): void

    • Die folgende Methode wird aus dem com.brightcove.player.edge.AbstractOfflineCatalog

      • renewLicense(@NonNull Video, @NonNull EventListener): void

    • Behebung eines Problems beim Laden von Posterbildern für heruntergeladene Wenn verfügbar, wird ein lokales Bild geladen.

    FreeWheel Plugin Änderungen:

    • Fügt Unterstützung für Live-Inhalte mit kundenseitigen Pre-Roll-Anzeigen hinzu

    SSAI-Plugin-Änderungen:

    • Behebt ein Problem, bei dem eine Inhaltsquelle direkt aus dem src Attribut in der Playback-API-Antwort anstelle des contenturi Attributs im VMAP geladen werden konnte.

    Version 6.8.0

    1. Mai 2019

    SDK-Änderungen:

    • Die vom Brightcove Native SDK für Android unterstützte Mindest-SDK-Version für Android ist jetzt 19.

    • Das Brightcove Native SDK für Android ist jetzt Java 8 konform. Bitte beachten Sie, dass Android-Apps, die mit dieser Version des Android SDK entwickelt wurden, möglicherweise die folgende Direktive zur build.gradle-Datei benötigen, um die Bytecode-Kompatibilität sicherzustellen:

      compileOptions{
targetCompatibility JavaVersion.VERSION_1_8
sourceCompatibility JavaVersion.VERSION_1_8
}

    ExoPlayer2 Änderungen:

    • ExoPlayer2 wurde auf Version 2.9.6 aktualisiert

    • Bitte beachten Sie, dass dieses Upgrade die folgenden API-Änderungen aufweist:

    • Breaking Change entfernt:constructor TrackSelectorHelper(@NonNull ExoPlayer player, @NonNull MappingTrackSelector trackSelector)

      Benutzen constructor TrackSelectorHelper(@NonNull ExoPlayer player, @NonNull DefaultTrackSelector trackSelector)

    • Breaking Change entfernt: setTrackSelector(@NonNull MappingTrackSelector trackSelector)

      Benutzen setTrackSelector(@NonNull DefaultTrackSelector trackSelector)

    • Entfernt getTrackSelector(): MappingTrackSelector

      Benutzen getTrackSelector(): DefaultTrackSelector

    FreeWheel Plugin Änderungen:

    • Das FreeWheel Plugin wurde aktualisiert, um FreeWheel AdManager Version 6.28.0 zu unterstützen.

    • Bitte beachten Sie, dass FreeWheel die AdManager-Bibliothek jetzt als Android-Archiv (AAR) verpackt.

    Änderungen des IMA-Plugins:

    • Das IMA-Plugin wurde aktualisiert, um die Google IMA SDK-Version 3.11.2 zu unterstützen.

    • Diese Version des Google IMA SDK unterstützt Open Measurement-Standards.

    • Weitere Informationen finden Sie in der Brightcove Android SDK-Dokumentation .

    • Es wurde ein Problem behoben, durch das Midroll-Anzeigen, die nach längerer Wiedergabe von Inhalten aufgerufen wurden, nicht abgespielt wurden.

    SSAI-Plugin-Änderungen:

    • Das SSAI-Plugin unterstützt jetzt die Inhaltsbereitstellung über den Brightcove Playback Authorization Service (PAS).

    • Bitte beachten Sie, dass PAS nur mit dem SSAI-Plugin funktioniert. PAS mit dem veralteten OnceUX Plugin wird nicht unterstützt.

    Version 6.7.0

    5. Apr 2019

    SSAI-Plugin:

    • Mit dieser Version implementiert das Brightcove Native SDK für Android das SSAI-Plugin. Weitere Informationen zu diesem neuen Plugin finden Sie in der Brightcove Android SDK-Übersicht auf der Brightcove Support-Website.

    OnceUx Plugin-Änderungen:

    • Mit dieser Version ist das OnceUx Plugin veraltet. Bitte benutzen Sie stattdessen das neue SSAI-Plugin. Weitere Informationen finden Sie in der Brightcove Android SDK-Übersicht auf der Brightcove Support-Website.

    Version 6.6.0

    29. Mrz 2019

    SDK-Änderungen:

    • Eine neue Klasse, HttpRequestConfig, wurde hinzugefügt. Diese Klasse unterstützt das Hinzufügen von Anforderungs-URL-Parametern und/oder Anforderungsheader, die von Brightcove-Diensten erforderlich sein können (z.
