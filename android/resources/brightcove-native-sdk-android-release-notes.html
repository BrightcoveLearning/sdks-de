Aktuelle Version

Version 6.16.3

1. Juli 2021

SDK-Änderungen:

  • Unterstützung für das Festlegen einer Beschriftungseigenschaft im hinzugefügt BrightcoveCaptionFormat -Klasse, um das Senden ausgewählter Untertitel- und Audiotrackinformationen an einen Cast-Empfänger zu unterstützen
  • Aktualisiert die VideoUtils -Klasse zum Hinzufügen einer Dienstprogrammmethode zum Entfernen der Sidecar-Beschriftungsquellen aus dem Video-Objekt
  • Es wurde ein Problem behoben, bei dem die video_dauer URL-Parameter war nicht in Anfragen an das Analytics-Back-End von Brightcove für die Berichterstellung enthalten

Änderungen am Cast-Plugin:

  • Unterstützung für das Festlegen einer URL zu einem benutzerdefinierten Player für die Cast-Empfänger-App hinzugefügt. Weitere Informationen finden Sie in der Android SDK-Supportdokumentation.

  • Unterstützung für das Senden ausgewählter Untertitel- und Audiotrackinformationen an einen Cast-Empfänger hinzugefügt

    Ein bekanntes Problem ohne Regression wird in einer kommenden Version behoben, bei dem die Untertitelauswahl im Manifest nicht automatisch auf dem Absender angezeigt wird, wenn die Cast-Sitzung während der Wiedergabe geschlossen wird.

    Eine Problemumgehung für dieses Problem besteht darin, dass die vor dem Öffnen der Cast-Sitzung getroffene Auswahl erneut ausgewählt und angezeigt werden kann, wenn die Sitzung auf dem Sendergerät erneut geöffnet wird.

Änderungen bei der Offline-Wiedergabe:

  • Unterstützung für das Festlegen von Eigenschaften im hinzugefügt AbstraktOfflineKatalog Klasse. Offline-Katalogeigenschaften können auf zwei Arten festgelegt werden:

    • Als ein Map<String, String> von Schlüssel/Wert-Paaren mit dem setProperties(Map<String, String>) Methode
    • Durch das getProperties().put(Schlüssel, Wert) Methode
  • Unterstützung für das Erstellen von benutzerdefinierten hinzugefügt Benachrichtigungskanäle in dem Download-Manager Klasse. Weitere Informationen finden Sie in der Android SDK-Supportdokumentation.

Version 6.16.2

10. Mai 2021

SDK-Änderungen:

  • Es wurde ein Problem behoben, bei dem Textspuren mit der Art Untertitel wurden nicht geparst oder angezeigt
  • Es wurde ein Problem behoben, bei dem Textspuren mit dem Position Attribut mit dem zusätzlichen Linie-links oder Linie-rechts Attribut wurde nicht geparst oder angezeigt

ExoPlayer 2-Änderungen:

  • Es wurde ein Problem behoben, bei dem ein falscher Index verwendet werden konnte, um die aktuell ausgewählte Audiosprache zu erhalten

Änderungen des Omniture-Plugins:

  • Es wurde ein Problem behoben, bei dem der Herzschlag fortgesetzt wurde, nachdem der Spieler zerstört wurde

Änderungen am SSAI-Plugin:

  • Parsing-Unterstützung für verschachtelte VAST- und VMAP-Erweiterungselemente hinzugefügt

    Erweiterungselemente werden unabhängig von der Verschachtelungstiefe nicht vom SSAI-Plugin ausgeführt, und daher stellt diese Unterstützung nur sicher, dass das Parsen von VAST und VMAP mit verschachtelten Erweiterungselementen beliebiger Tiefe nicht fehlschlägt.

  • Parsing-Unterstützung für VAST . hinzugefügt Anzeigenverifizierung Elemente

    Anzeigenverifizierung Elemente werden nicht vom SSAI-Plugin ausgeführt, und daher stellt diese Unterstützung nur sicher, dass das Parsen von VAST mit Anzeigenverifizierung Elemente scheitern nicht.

    Zusätzlich, Anzeigenverifizierung Elemente sind eine VAST 4.0-Funktion, und das Brightcove Android SDK unterstützt VAST 4.x noch nicht vollständig. Diese Parsing-Unterstützung sollte nicht allgemein als Unterstützung für VAST 4.x im Brightcove Native SDK für Android betrachtet werden.

Wenn sie direkt über den Builder aufgerufen werden, konfigurieren diese Methoden die AdsRenderingSettings und ImaSdk-Einstellungen Objekte, die standardmäßig im Builder erstellt werden.

    Wenn Sie sich entscheiden, benutzerdefinierte zu erstellen AdsRenderingSettings und ImaSdk-Einstellungen Google IMA SDK-Objekte zur Verwendung im IMA-Plug-in können Sie diese Objekte aufrufen' Methoden direkt:

    • AdsRenderingSettings.setLoadVideoTimeout(int) : void
    • ImaSdkSettings.setDebugMode(boolean) : void

SDK-Änderungen:
