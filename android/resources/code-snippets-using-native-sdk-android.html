    Anpassen von Steuerelementen für Android TV

    Sie können der Steuerungsleiste für Geräte oder Android TV Schaltflächen hinzufügen. Die folgenden Schritte führen Sie durch das Hinzufügen einer Vollbild-Taste zur Steuerungsleiste für Android TV:

    1. Öffnen Sie die Beispiel-App „ Benutzerdefinierte Steuerung “. Wir werden dies als Ausgangspunkt verwenden. Weitere Informationen zum Anpassen der Player-Steuerelemente finden Sie in der README-Datei der Beispiel-App.
    2. Öffnen Sie die Datei res/layout/land/my_media_controller.xml in Android Studio.
    3. Kopieren Sie das Button Element für die full_screen Schaltfläche. Der Code sollte ähnlich aussehen wie folgt: 
      <Button
style="@style/BorderlessButton"
android:id="@id/full_screen"
android:padding="4dp"
android:layout_gravity="center_vertical"
android:layout_height="wrap_content"
android:layout_width="wrap_content"
android:visibility="visible"
android:text="@string/brightcove_controls_enter_full_screen"/>
    4. Öffnen Sie die Datei res/layout/my_tv_media_controller.xml in Android Studio.
    5. Fügen Sie das kopierte Button Element in diese Layoutdatei ein. Die Position der Elemente bestimmt ihre Position in der Steuerungsleiste.
    6. Scrollen Sie zum Anfang der Datei res/layout/my_tv_media_controller.xml . Stellen Sie sicher, dass die style -Eigenschaft auf eingestellt ist @style/BrightcoveControlBar.TV.Custom. Der Code sollte so aussehen: 
      <?xml version="1.0" encoding="utf-8"?>
<com.brightcove.player.mediacontroller.BrightcoveControlBar
xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:bmc="http://schemas.android.com/apk/res-auto"
xmlns:tools="http://schemas.android.com/tools"
tools:ignore="Overdraw, InconsistentLayout"
android:id="@+id/brightcove_control_bar"
android:background="@color/bmc_background"
android:layout_width="match_parent"
android:layout_height="wrap_content"
android:padding="8dp"

android:orientation="horizontal"
android:layout_gravity="bottom"
    style="@style/BrightcoveControlBar.TV.Custom">
    7. Öffnen Sie die Datei res/values/styles.xml in Android Studio.
    8. Zum style Element mit a name Wert von BrightcoveControlBar.TV.Custom , füge ein hinzu item Element für die Vollbild-Schaltfläche und setzen Sie es auf true um es anzuzeigen. 
      <style name="BrightcoveControlBar.TV.Custom">
<item name="brightcove_vr_mode">false</item>
  <item name="brightcove_full_screen">true</item>
</style>
    9. Das ist es. Führen Sie die Beispielanwendung auf einem physischen oder virtuellen Android-Fernsehgerät aus. Sie sollten die Vollbild-Taste in der Steuerungsleiste sehen.

    Vollbildmodus

    Sie können den Vollbildmodus mithilfe von Code verwalten.

    • Hier ist der Code zum Einstellen des Vollbildmodus:

      brightcoveVideoView.getEventEmitter().emit(EventType.ENTER_FULL_SCREEN);

    • Hier ist der Code zum Beenden des Vollbildmodus:

      brightcoveVideoView.getEventEmitter().emit(EventType.EXIT_FULL_SCREEN);

    Dauer für Livestreams erhalten

    Um die Dauer für einen Livestream zu erhalten, können Sie das verwenden MAX_POSITION. Dies gibt Ihnen die größte suchbare Position für einen Live-Stream (dh die am weitesten entfernte, in der Sie die Bildlaufleiste ziehen können). Ihr Code sollte ähnlich wie folgt aussehen:

    brightcoveVideoView.getEventEmitter().on(EventType.PROGRESS, new EventListener() {
  @Override
  public void processEvent(Event event) {
  int duration = (int) event.properties.get(Event.MAX_POSITION);
  }
});

    Die ExoPlayer-Instanz abrufen

    Es kann vorkommen, dass Sie das ExoPlayer-Objekt benötigen, um Eigenschaften zu ändern, die nicht über das Brightcove Native SDK verfügbar gemacht werden. Eine Instanz dieses Objekts wird erstellt, nachdem das Video festgelegt wurde. Also musst du auf das DID_SET_VIDEO Event warten.

    Sie können die ExoPlayer-Instanz wie folgt abrufen:

    // Get the ExoPlayer instance
eventEmitter.on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    VideoDisplayComponent videoDisplayComponent = brightcoveVideoView.getVideoDisplay();
    if (videoDisplayComponent instanceof ExoPlayerVideoDisplayComponent) {
      // Get ExoPlayer
      ExoPlayer exoPlayer = ((ExoPlayerVideoDisplayComponent) videoDisplayComponent).getExoPlayer();
      if (exoPlayer instanceof SimpleExoPlayer) {
        // Get SimpleExoPlayer
        SimpleExoPlayer simpleExoPlayer = (SimpleExoPlayer) exoPlayer;
        Log.v(TAG, "onDID_SET_VIDEO: ExoPlayer = " + simpleExoPlayer);
      }
    }
  }
});

    Ein Beispiel für das Ändern der AudioAttribute Eigenschaft für den ExoPlayer finden Sie im Forumsbeitrag Setting AudioAttributes on Exoplayer Instance .

    Abrufen der URL für Vorschaubilder

    Vielleicht möchten Sie die Miniaturansichten oder Videostummel aus Ihrer Brightcove-Bibliothek verwenden. Verwenden Sie den folgenden Code, um die URL für Ihre Bilder zu erhalten:

    Für Video Cloud-Kunden, die den Edge-Katalog (com.brightcove.player.edge.catalog) verwenden, können Sie die Miniaturbild-URL wie folgt abrufen:

    String thumbnailUrl = video.getProperties().get("thumbnail").toString();

    Wenn Sie feststellen, dass das Miniaturbild zu klein ist, können Sie stattdessen das Videostillbild erhalten:

    String videoStillUrl = video.getProperties().get("stillImageUri").toString();

    Google-Analysen

    Wenn Sie den Brightcove Player und die Katalogklasse verwenden, werden Videoanalysen automatisch erfasst und in Ihrem Video Cloud Analytics-Modul angezeigt. Für zusätzliche Metriken können Sie Ihrer App Google Analytics hinzufügen.

    Um Google Analytics in Ihre App zu integrieren, führen Sie die folgenden Schritte aus:

    1. Fügen Sie Ihrem Projekt das Google Services-Plugin hinzu.
    2. Holen Sie sich eine Google-Konfigurationsdatei und fügen Sie sie Ihrem Projekt hinzu.
    3. Erweitern Sie die Anwendung und stellen Sie eine Hilfsmethode bereit, die Ihren Anwendungs-Tracker zurückgibt. Es sollte ähnlich aussehen wie die Google-Dienstleistungen AnalyticsApplication.
    4. Rufen Sie in einer separaten Klasse, die den BrightCovePlayer erweitert, die Shared Tracker-Instanz ab: 
      // Obtain the shared Tracker instance
AnalyticsApplication application = (AnalyticsApplication) getApplication();
tracker = application.getDefaultTracker();
    5. Überschreiben Sie geeignete Methoden, um Änderungen am Bildschirm zu protokollieren und/oder benutzerdefinierte Ereignisse zum Tracking zu senden.

    Ausführliche Schritte finden Sie im Dokument von Google zum Hinzufügen von Analytics zu Ihrer Android-App.

    Laden eines Remote-Video- und Posterbilds

    Brightcove Player verwenden nur Kunden Video-Assets von einem Remote-Server. Befolgen Sie diese Schritte, um ein Video und ein Posterbild zu laden, die sich beide auf einem Remote-Server befinden:

    1. Erstellen Sie in der onCreate() Methode nach dem Definieren der Videoansicht ein Videoobjekt aus Ihrem auf einem Remote-Server gehosteten Video. Stellen Sie DeliveryType den so ein, dass er dem Videotyp entspricht, den Sie haben.

      Video video = Video.createVideo("https://learning-services-media.brightcove.com/videos/hls/greatblueheron/greatblueheron.m3u8",
  DeliveryType.HLS);

    2. Laden Sie ein Remote-Image, das als Posterbild verwendet werden soll, bevor die Videowiedergabe beginnt.

      try {
  java.net.URI myposterImage = new java.net.URI("https://solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png");
  video.getProperties().put(Video.Fields.STILL_IMAGE_URI, myposterImage);
} catch (URISyntaxException e) {
  e.printStackTrace();
}

    3. Fügen Sie das Video zur Ansicht hinzu und starten Sie die Videowiedergabe.

      brightcoveVideoView.add(video);
brightcoveVideoView.start();

    4. Vergewissern Sie sich, dass Ihre onCreate() Methode wie folgt aussieht:

      @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    // Optional: For Brightcove Player customers to register their apps
    Analytics analytics = brightcoveVideoView.getAnalytics();
    analytics.setAccount("your account Id");

    // Define a video from a remote server
    Video video = Video.createVideo("https://learning-services-media.brightcove.com/videos/hls/greatblueheron/greatblueheron.m3u8", DeliveryType.HLS);

    // Load a remote poster image
    try {
        java.net.URI myposterImage = new java.net.URI("https://solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png");
        video.getProperties().put(Video.Fields.STILL_IMAGE_URI, myposterImage);
    } catch (URISyntaxException e) {
        e.printStackTrace();
    }

    // Add video to the view
    brightcoveVideoView.add(video);
    // Start video playback
    brightcoveVideoView.start();
  }

    Manuelles Hinzufügen von DRM-Inhalten

    Als Kunde von Brightcove Player möchten Sie möglicherweise DRM-geschützte Inhalte von Ihrem eigenen Server aus verwenden. Sie können DRM-Inhalte wie folgt manuell laden:

    import com.brightcove.player.display.WidevineMediaDrmCallback;

Video video = Video.createVideo("https://storage.googleapis.com/wvmedia/cenc/h264/tears/tears.mpd");
video.getProperties().put(WidevineMediaDrmCallback.DEFAULT_URL, "https://proxy.uat.widevine.com/proxy?video_id=&provider=widevine_test");
brightcoveVideoView.add(video);
brightcoveVideoView.start();
    Sie müssen DRM-Inhalte nicht manuell in Ihre Brightcove-Bibliothek laden.

    Methoden: synchron oder asynchron?

    Hier sind einige der asynchronen Methoden, die im Native Player SDK für Android zu finden sind:

    • Das start() , seekTo() und stopPlayback() Methoden sind asynchron, da sie ein Ereignis ausgeben, das von anderen Komponenten im System verarbeitet werden soll.
    • Die clear() Methode ist synchron in Bezug auf die Aktualisierung der Liste, ist jedoch asynchron in Bezug auf das Entladen des aktuellen Videos.

    Stummschalten/Stummschaltung

    Du kannst den Spieler programmatisch stummschalten oder die Stummschaltung aufheben. Brightcove empfiehlt, das BrightcoveExoplayerVideoView wann immer möglich zu verwenden.

    Sie müssen entweder auf das Ereignis DID_SET_VIDEO oder das DID_PLAY Ereignis warten, bevor Sie den Code ausführen, um den Spieler stummzuschalten. Andernfalls wird die Stummschaltung nicht wirksam. Der Grund, auf diese Ereignisse zu warten, ist, dass der Player beim Laden des Videos erstellt wird.

    brightcoveExoplayerVideoView

    Wenn Sie BrightcoveExoPlayerVideoView verwenden, können Sie den Player programmgesteuert stummschalten oder die Stummschaltung aufheben, indem Sie Folgendes tun:

    float volume = 100F;
...
brightcoveVideoView.getEventEmitter().on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    setMute(true);
    brightcoveVideoView.start();
  }
});
...
private void setMute(boolean mute) {
  volume = mute ? 0F : 100F;
  Map<string, Object> properties = new HashMap<>();
  properties.put(Event.VOLUME, volume);
  brightcoveVideoView.getEventEmitter().emit(EventType.SET_VOLUME, properties);
}

    brightcoveVideoView

    Brightcove empfiehlt die Verwendung von BrightcoveExoplayerVideoView.

    Wenn Sie das BrightCoveVideoView verwenden müssen, das den MediaPlayer von Android verwendet, können Sie den Player mit dem folgenden Code programmgesteuert stummschalten oder die Stummschaltung aufheben:

    float volume = 1F;
...
brightcoveVideoView.getEventEmitter().on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    setMute(true);
    brightcoveVideoView.start();
  }
});
...
private void setMute(boolean mute) {
  volume = mute ? 0F : 1F;
  Map<String, Object> properties = new HashMap<>();
  properties.put(Event.LEFT_VOLUME, volume);
  properties.put(Event.RIGHT_VOLUME, volume);
  brightcoveVideoView.getEventEmitter().emit(EventType.SET_VOLUME, properties);
}

    Offline Wiedergabe mit DRM

    Bei Videos, die für die Offline-Wiedergabe heruntergeladen wurden, kann der Downloadstatus auch im Benachrichtigungsbereich angezeigt werden. Der Titel der Benachrichtigung ist auf den Videotitel festgelegt. Die Benachrichtigung wird entfernt, wenn der Download pausiert oder abgebrochen wird.

    Benachrichtigungsstatus herunterladen
    Download-Status Text zum Inhalt der Benachrichtigung Symbol „Benachrichtigung“
    Downloadvorgang R.string.odrm_download_running - Dies ist standardmäßig „Download...“

    Es wird einen Fortschrittsbalken geben, der den abgeschlossenen Prozentsatz anzeigt.    		 Das animierte Symbol „Herunterladen“ zur Plattform-Standardeinstellung - android.R.drawable.stat_sys_download
    Erneut versuchen R.string.odrm_download_waiting_retry - das ist „Wartende Wiederholung...“ standardmäßig  
    fehlgeschlagen R.string.odrm_download_failed - das ist „Failed!"“ standardmäßig  
    Abgeschlossen R.string.odrm_download_complete - dies ist standardmäßig „Gespeichert“ Das statische Symbol „heruntergeladen“ des Plattform-Standardsymbols - android.R.drawable.stat_sys_download_done

    Für den Status „Wiederholen“ oder „Fehlgeschlagen“ wird der Benachrichtigungs-Subtext auf eine geeignete Ressource basierend auf dem Fehlertyp festgelegt. Hier ist die vollständige Liste:

    • R.string.odrm_error_none
    • R.string.odrm_error_cannot_resume
    • R.string.odrm_error_device_not_found
    • R.string.odrm_error_file_already_exists
    • R.string.odrm_error_file_error
    • R.string.odrm_error_http_data_error
    • R.string.odrm_error_insufficient_space
    • R.string.odrm_error_too_many_redirects
    • R.string.odrm_error_unhandled_http_code
    • R.string.odrm_error_unknown

    Paging mit der Playback-API

    Wenn Sie Ihre Video Cloud-Inhalte von der Playback-API abrufen, können Sie Paging für eine Playlist implementieren.

    Um eine Reihe von Videos in einer Wiedergabeliste zu blättern, verwenden Sie die folgenden Anforderungs-URL-Parameter:

    • limit - definiert die Anzahl der Videos, die von der Playback-API zurückgegeben werden sollen
    • offset - legt die Anzahl der Videos fest, die in einer Playlist aus der Playback-API übersprungen werden sollen

    Die Abfrageparameter werden als Map Objekt als Schlüssel-Wert-Paare an die Catalog -Methode übergeben. In diesem Beispiel werden 6 Videos zurückgegeben, beginnend mit dem 10. Video in der Playlist:

    Map<String, String> queryParameters = new HashMap<>();
queryParameters.put("limit", "6");
queryParameters.put("offset", "9");

Catalog catalog = new Catalog(eventEmitter, "myAccount", "myPolicy");
catalog.findPlaylistByID("myPlaylistId", null, queryParameters, myListener);

    Wiedergabe lokaler Videos

    Wenn Sie MP4-Videos abspielen möchten, die lokal gespeichert sind, sollten Sie sie im Application Asset-Ordner speichern.

    Uri video = Uri.parse("file:///android_asset/path/to/video.mp4");
brightcoveVideoView.add(Video.createVideo(video.toString()));
brightcoveVideoView.start();
    Sie erhalten möglicherweise einen Ausnahmefehler, wenn Sie versuchen, MP4-Videos abzuspielen, die im Anwendungsressourcenordner (/res/raw) gespeichert sind.

    Positionieren der Spielersteuerung auf der Unterseite

    Standardmäßig ist die Steuerungsleiste des Players direkt unter dem Video positioniert, wie hier gezeigt:

    Sie können die Steuerungsleiste des Players auch so positionieren, dass sie sich am unteren Rand der Videoansicht befindet. Da die Steuerelemente Teil des sind BrightcoveVideoView, deckt die Ansicht durch Festlegen der Höhe auf den gesamten Bildschirm ab. match_parent Wenn die Steuerelemente unten in der Ansicht angezeigt werden, bleibt das Video in der Mitte der Ansicht.

    Es gibt zwei Möglichkeiten, dies zu tun. Der erste Ansatz folgt:

    1. Öffnen Sie die Datei main/res/values/style.xml für Ihr Projekt.

    2. Fügen Sie den folgenden XML-Code hinzu:

      <style name="BrightcoveControlBar" parent="BrightcoveControlBarDefault">
  <item name="brightcove_align">false</item>
</style>

    Oder Sie können Code in der Datei MainActivity.java wie folgt verwenden:

    Sucht ohne Werbung

    Sie können die adsDisabled Eigenschaft für den verwenden VideoPlaybackController , um die Anzeigenwiedergabe zu deaktivieren, während Sie ein Video durchsuchen.

    Um diese Funktion zu nutzen, gehen Sie folgendermaßen vor:

    1. Holen Sie sich die VideoPlaybackController Instance aus dem BrightcoveVideoView.

      VideoPlaybackController playbackController = brightcoveVideoView.getPlaybackController();

    2. Deaktivieren Sie die Anzeigenwieder

      playbackController.setAdsDisabled(true);
    3. Suche nach der gewünschten Zeitposition im aktuellen Video.

    4. Setzen Sie das normale Anzeigenverhalten wieder auf

      playbackController.setAdsDisabled(false);

    Ihr Code sollte ungefähr so aussehen:

    final VideoPlaybackController playbackController = brightcoveVideoView.getPlaybackController();

  @Override
  public void processEvent(final Event event) {
    playbackController.setAdsDisabled(true);
    brightcoveVideoView.seekTo(10000);
  }
});

eventEmitter.on(EventType.DID_SEEK_TO, new EventListener() {
  @Override
  public void processEvent(final Event event) {
    playbackController.setAdsDisabled(false);
  }
});

    Einstellen der Puffergröße

    Sie können erwägen, die Pufferlänge zu erhöhen, um die Pufferung im Player zu vermeiden, wenn sich die Bereitstellung des nächsten Segments vom CDN verzögert. Möglicherweise können Sie jedoch nicht manuell etwas Besseres machen, als HLS es bereits tut.

    HLS wurde entwickelt, um sofort zu spielen und Qualität zu verlieren, wenn es nicht mithalten kann. Auf diese Weise muss kein Puffer vorgeladen werden. Wenn es nicht mithalten kann, wird es so viel wie möglich vom Video in bester Qualität geladen, um eine Unterbrechung zu verhindern.

    Beginnend mit dem Native SDK für Android v6.3.1 wurden dem SDK die folgenden Klassen für die Implementierung von ExoPlayer 2 hinzugefügt:

    • LoadControlConfig
    • AllocatorConfig

    Die in diesen Klassen festgelegten Werte werden verwendet, um das DefaultLoadControl ExoPlayer-Objekt zu erstellen, das zum Erstellen der ExoPlayer-Instanz verwendet wird. Entwickler können die Puffergröße wie folgt steuern:

    1. Das LoadControlConfig und AllocatorConfig Klassen folgen den Builder-Mustern. Die Werte, die Sie in beiden Klassen festlegen können, spiegeln die Werte wider, die Sie in ihren Konstruktoren festlegen können.

      AllocatorConfig allocatorConfig = new AllocatorConfig.Builder().build();
LoadControlConfig loadControlConfig = new LoadControlConfig.Builder()
.setAllocatorConfig(allocatorConfig)
.setMinBufferMs(X)
.setMaxBufferMs(Y)
.build();

    2. Werte, die nicht im Builder festgelegt sind, verwenden Standardwerte. Verwenden Sie die ExoPlayerVideoDisplayerComponent.setLoadControlConfig() Methode, um den festzulegen LoadControlConfig.

      brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);
...
ExoPlayerVideoDisplayComponent dc = (ExoPlayerVideoDisplayComponent) brightcoveVideoView.getVideoDisplay();
dc.setLoadControlConfig(loadControlConfig);

    Festlegen von Untertiteln und Themen

    Derzeit BrightcoveCaptionPropertiesActivity ist der in der Manifestdatei des Brightcove Player SDK festgelegt, sodass Entwickler es nicht in ihren Anwendungen angeben müssen:

    <?xml version="1.0" encoding="utf-8"?>
<manifest
xmlns:android="http://schemas.android.com/apk/res/android"
package="com.brightcove.player"
android:versionCode="1"
android:versionName="1.0">

<application>
<!-- If we don't register this Activity in the Manifest, apps using the SDK will crash when they try to access it. -->
<!-- During the app's build process, this manifest will be merged with the app-level one. -->
<activity android:name="com.brightcove.player.captioning.BrightcoveCaptionPropertiesActivity"/>
</application>
</manifest>

    Der BrightcoveCaptionPropertiesActivity erbt das Standardthema der Plattform, solange Sie Themen für Ihre App auf der <activity> Ebene im Manifest Ihrer App festlegen.

    Wenn Sie ein einzelnes App-Design auf der <application> Ebene festlegen, erbt der BrightcoveCaptionPropertiesActivity die Eigenschaften dieses Themas auf Anwendungsebene. Bei einigen Themen kann dies dazu führen, dass die Aktivität seltsam gestylt oder sogar unleserlich aussieht.

    In diesen Fällen sollten Sie das BrightcoveCaptionPropertiesActivity in Ihrem eigenen Manifest angeben und dort ein Thema anwenden, wie folgt:

    <activity
android:name="com.brightcove.player.captioning.BrightcoveCaptionPropertiesActivity"
android:theme="@style/MyCustomCaptionSettingsTheme"/>

    Video-Stillbilder anzeigen/ausblenden

    Das Standardverhalten beim Abrufen von Videos aus Ihrer Video Cloud-Bibliothek besteht darin, das Videostillbild anzuzeigen, bis die Wiedergabe beginnt.

    Zeigen Sie das Video-Stilbild

    Sie können das Video jederzeit noch anzeigen, indem Sie ein SET_VIDEO_STILL Ereignis mit einer VIDEO_STILL Eigenschaft aussenden, die auf den URI des Videostums festgelegt ist.

    Ausblenden des Video-Stillbilds

    Wenn Sie das Video automatisch abspielen, möchten Sie möglicherweise das Videostillstand ausblenden. Um dies zu tun, können Sie das SET_VIDEO_STILL Ereignis wie folgt abhören und das Standardverhalten verhindern:

    brightcoveVideoView.getEventEmitter().on(EventType.SET_VIDEO_STILL, new EventListener() {
  @Override
  public void processEvent(Event event) {
    event.preventDefault();
    event.stopPropagation();
  }
});

    Verwenden einer Playlist

    Wenn Sie mit einer Wiedergabeliste arbeiten, denken Sie daran, dass durch den Aufruf der clear() Methode alle Elemente der Wiedergabeliste entfernt werden. Daher kann es zu einer IndexOutOfBounds Ausnahme kommen, wenn Sie versuchen, zum ersten Video in der Wiedergabeliste zu springen.

    Stattdessen kannst du so etwas ausprobieren:

    brightcoveVideoView.pause();
brightcoveVideoView.stopPlayback();
brightcoveVideoView.setCurrentIndex(0);
