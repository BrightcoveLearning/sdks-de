    HEVC-Auswahl mit dem Native SDK für Android

    In diesem Thema erfahren Sie, wie das Brightcove Native SDK für Android HEVC-Quellen und -Wiedergaben auswählt.

    Übersicht

    High Efficiency Video Coding (HEVC), auch bekannt als H.265, ist ein Videokomprimierungsstandard, der die Größe von Videodateien reduziert und eine höhere Wiedergabequalität ermöglicht.

    Das Brightcove Native SDK für Android verwendet den folgenden HEVC-Auswahlprozess:

    1. HEVC-Quellenauswahl
    2. Auswahl der HEVC-Wiedergabe

    Hinweise:

    • Wird mit dem Brightcove Native SDK für Android 6.13.1+ unterstützt
    • Funktioniert mit Android-Geräten mit API Level 21 (Lollipop Version 5.0) und höher

    HEVC-Quellenauswahl

    Der HEVC-Quellauswahlprozess beginnt, nachdem Sie eine Anfrage für ein Video mit dem Brightcove gestellt haben Catalog.

    Abhängig von Ihrer Kontokonfiguration kann die Catalog Antwort mehrere Quellen enthalten, mit oder ohne HEVC-Wiedergaben. Beispiel:

    "profiles": "urn:mpeg:dash:profile:isoff-live:2011"

    In diesem Beispiel gibt es vier DASH-Quellen:

    • Eine Nicht-HEVC-HTTP-Quelle
    • Eine Nicht-HEVC-HTTPS-Quelle
    • Eine HEVC-HTTP-Quelle
    • Eine HEVC-HTTPS-Quelle

    Beachten Sie, dass HEVC-Quellen die Zeichenfolge hvc in der URL enthalten.

    Der Native SDK für Android-Quellenauswahl bevorzugt HEVC-Quellen. Im obigen Beispiel wird die HEVC-HTTPS-Quelle ausgewählt.

    Das Native SDK wird mit dem im Android definierten Standardauswahlprozess fortgesetzt: Dokument zur Quellenauswahl :

    1. Bevorzugen Sie HTTPS gegenüber HTTP
    2. Bevorzugen Sie DASH gegenüber HLS
    3. Wenn Sie HLS für die Wiedergabe auswählen, bevorzugen Sie die höchste Version von HLS

    Auswahl der HEVC-Wiedergabe

    Abhängig von Ihren Aufnahmekonfigurationen ist die HEVC-Quelle möglicherweise in verschiedenen Wiedergabeverläufen enthalten. Sie haben möglicherweise Folgendes:

    • Nur HEVC-Formatvarianten
    • Eine Mischung aus HEVC und anderen Codecs
    Diese Art der Auswahl gilt nur bei Verwendung von ExoPlayer.

    Nur HEVC-Formatvarianten

    Wenn die Wiedergabeformate der Videoquelle nur HEVC-Codecs enthalten, ist keine Auswahl der Wiedergabeversion erforderlich. Der ExoPlayer wählt die entsprechende Wiedergabe angesichts der Gerätefähigkeiten und der verfügbaren Bandbreite aus.

    Eine Mischung aus HEVC und anderen Codecs

    Wenn die Videoquelle über Wiedergabeversionen mit einer Mischung aus HEVC und anderen Codecs (wie AVC) verfügt, wird ein Auswahlprozess für die Wiedergabeversion aufgerufen. Wenn das native SDK die Quelle in den ExoPlayer lädt, führt es eine SelectionOverride für die Videowiedergabe durch, um HEVC gegenüber anderen Codecs zu bevorzugen.

    Bei einem Video, dessen Formatvariantenformate vom Gerät unterstützt werden, ist hier der Wiedergabeprozess:

    1. Das Native SDK wählt alle HEVC-Ausgabeformate aus.
    2. Es ignoriert Nicht-HEVC-Ausgabeformate, die eine vorhandene HEVC mit derselben Auflösung duplizieren.

    Dies zwingt ExoPlayer in den meisten Fällen dazu, HEVC zu bevorzugen. Hier sind zwei mögliche Szenarien:

    • Codec-Mix-Vervielfältigung
    • Halb und halb

    Codec-Mix-Vervielfältigung

    In diesem Szenario werden die meisten Auflösungen des HEVC und des AVC-Codecs doppelt durchgeführt. Beispiel: 

    AVC <Representation bandwidth="453000" height="270" width="480">
AVC <Representation bandwidth="704000" height="360" width="640">
AVC <Representation bandwidth="903000" height="360" width="640">
AVC <Representation bandwidth="1202000" height="540" width="960">
AVC <Representation bandwidth="1989000" height="720" width="1280">
AVC <Representation bandwidth="2478000" height="720" width="1280">
AVC <Representation bandwidth="3458000" height="720" width="1280">
AVC <Representation bandwidth="3761000" height="720" width="1280">
HVC <Representation bandwidth="304000" height="270" width="480">
HVC <Representation bandwidth="548000" height="360" width="640">
HVC <Representation bandwidth="993000" height="432" width="768">
HVC <Representation bandwidth="1301000" height="576" width="1024">
HVC <Representation bandwidth="1605000" height="720" width="1280">
HVC <Representation bandwidth="2809000" height="720" width="1280">

    Angesichts der obigen Videowiedergaben wählt das Brightcove Natvie SDK die folgenden Formatvarianten aus: 

    AVC <Representation bandwidth="1202000" height="540" width="960">
HVC <Representation bandwidth="304000" height="270" width="480">
HVC <Representation bandwidth="548000" height="360" width="640">
HVC <Representation bandwidth="993000" height="432" width="768">
HVC <Representation bandwidth="1301000" height="576" width="1024">
HVC <Representation bandwidth="1605000" height="720" width="1280">
HVC <Representation bandwidth="2809000" height="720" width="1280">

    Beachten Sie, dass nur eine AVC-Formatvariante ausgewählt wurde, da ihre Auflösung keine vorhandene HEVC-Formatvariante dupliziert.

    Halb und halb

    In diesem Szenario gibt es Versionen mit niedrigerer Auflösung mit dem AVC-Codec und Wiedergaben mit höherer Auflösung mit dem HVC-Codec. Beispiel: 

    AVC <Representation bandwidth="453000" height="270" width="480">
AVC <Representation bandwidth="704000" height="360" width="640">
AVC <Representation bandwidth="903000" height="360" width="640">
AVC <Representation bandwidth="1202000" height="540" width="960">
AVC <Representation bandwidth="3761000" height="720" width="1280">
HVC <Representation bandwidth="1301000" height="576" width="1024">
HVC <Representation bandwidth="1605000" height="720" width="1280">
HVC <Representation bandwidth="2809000" height="720" width="1280">

    In diesem Fall wählt das Native SDK alle Darstellungen aus, sodass ExoPlayer die Wiedergabeversionen auswählen können, da es keine AVC-Codecs mit Auflösungen gibt, die auf der verfügbaren Bandbreite auswählen können.

    Das native SDK zwingt die HEVC-Auswahl in diesem Szenario nicht in allen Fällen, um zu vermeiden, dass die niedrigste Formatvariante auf eine HEVC-Wiedergabe mit hoher Bitrate gedeckelt wird. Dies würde möglicherweise zu einer schlechten Benutzererfahrung mit einer schlechten Netzwerkverbindung führen.
