Einführung

Durch die Verschmelzung der Brightcove Web Player-Ästhetik mit iOS- und Android-Plattformstandards bieten die Brightcove Native Player-Steuerelemente ein stilvolles Aussehen zusätzlich zu einer soliden Reihe von Funktionen direkt nach dem Auspacken.

Funktionen mit den Brightcove Native Player-Steuerelementen:

  • Dynamische Benutzeroberfläche
  • Anpassung
  • Barrierefreiheit

Dynamische Benutzeroberfläche

Die nativen Player-Steuerelemente von Brightcove sind hochgradig anpassungsfähig, um sicherzustellen, dass Ihre Inhalte in jeder Situation in einer ansprechenden Benutzeroberfläche angezeigt werden.

Adaptive Bildschirmgröße

Für jede mobile Anwendung ist es von entscheidender Bedeutung, sicherzustellen, dass die Elemente der Benutzeroberfläche groß genug sind, um sie klar zu sehen und zu verwenden. Die nativen Player-Steuerelemente von Brightcove enthalten Logik, um zu bestimmen, wie viel Speicherplatz für die Benutzeroberfläche verfügbar ist. Wenn die Breite unter einen bestimmten Schwellenwert fällt, schaltet die Steuerung automatisch auf eine zweizeilige Konfiguration um.

Android

Das native SDK für Android überprüft die Bildschirmgröße des Geräts, um das Layout der Player-Steuerelemente festzulegen.

  • Wenn die Breite ist 480dp oder höher wird das einzeilige Layout verwendet.
  • Wenn die Breite kleiner ist als 480dp , wird das zweizeilige, kompakte Layout verwendet.

Weitere Informationen zu dichteunabhängigen Pixeln (dp) finden Sie unter Android Unterstützt unterschiedliche Pixeldichten dokumentieren.

iOS

Das native SDK für iOS überprüft die Bildschirmgröße des Geräts, um das Layout der Player-Steuerelemente festzulegen.

  • Wenn die Breite ist 450px oder höher wird das einzeilige Layout verwendet.
  • Wenn die Breite kleiner ist als 450px , wird das zweizeilige, kompakte Layout verwendet.

Wiedergabemodi

Ihr Player kann während seines gesamten Lebenszyklus unterschiedliche Arten von Inhalten enthalten, die jeweils ein bestimmtes Layout der Benutzeroberfläche erfordern. Die nativen Player-Steuerelemente von Brightcove enthalten eine Logik, um automatisch zu bestimmen, ob der Inhaltstyp VoD, Live oder Live DVR ist, und die Benutzeroberfläche entsprechend anzupassen, sogar in einer Wiedergabeliste mit wechselnden Inhaltstypen!

Video-on-Demand (VoD)

Die VoD-Benutzeroberfläche macht den Großteil der Wiedergabeerfahrungen aus und ist die Wurzel aller Spieler-UI-Designs.

Zu den bemerkenswerten Funktionen der Brightcove Player-Steuerelemente gehören:

  • Play/Pause-Taste
    • Spielt den Inhalt ab und pausiert ihn
  • Zurückspringen-Taste
    • Durch Drücken dieser Taste wird der Inhalt 10 Sekunden zurückgesucht.
  • Verbesserte Suchleiste
    • Zeigt wiedergegebenen Inhalt in Rosa an
    • Zeigt gepufferten Inhalt in Hellgrau an
    • Unterstützt die Verwendung von Markern für Werbung oder andere Zwecke (Android)
  • Schaltfläche für geschlossene Untertitel
    • Wird nur angezeigt, wenn Untertitel erkannt werden. Wenn Sie diese Schaltfläche auswählen, wird das Einstellungsmenü für Untertitel geöffnet.
  • Vollbild-Schaltfläche
    • Ermöglicht dem Betrachter, zum Vollbildmodus oder zurück zur Inline-Wiedergabe zu wechseln
  • Airplay-Taste (nur iOS)
    • Wird nur angezeigt, wenn Airplay-Geräte erkannt werden. Mit dieser Funktion kann der Zuschauer Airplay-Streaming einrichten.

Live

Die Live-Benutzeroberfläche verbirgt bestimmte Elemente, mit denen ein Benutzer nicht interagieren könnte. Zu den entfernten Elementen gehören die Schaltfläche "Zurückspringen" und die Suchleiste. Es wird jedoch ein neues „LIVE“-Label hinzugefügt, das bei Grün anzeigt, dass der angesehene Inhalt live ist.

Live-DVR

Das Live-DVR-Format bietet die gleichen Funktionen wie das VoD-Layout, plus ein zusätzliches LIVE-Label, das grün wird, wenn der Inhalt live ist, und grau ist, wenn der Zuschauer ältere Inhalte sieht. Darüber hinaus fungiert das LIVE-Label als Taste, die jederzeit gedrückt werden kann, um zum letzten Live-Moment zurückzukehren.

Anpassung

Möchten Sie die Farben oder den Skin an die Farbthemen Ihrer App anpassen? Kein Problem! Die nativen Player-Steuerelemente von Brightcove wurden so konzipiert, dass sie leicht anpassbar sind. Hier sind einige der Möglichkeiten, wie Sie die Steuerelemente anpassen können:

  • Tauschen Sie Farben aus und verwenden Sie Ihre eigenen benutzerdefinierten Schaltflächen, um personalisierte Themen zu erstellen. Mit dem Native SDK für Android können Sie ganz einfach eine Vielzahl gängiger Schaltflächen ein- und ausschließen. Hier wurden die Steuertasten in Orange geändert und die Suchleiste hat ein benutzerdefiniertes Bild.
  • Während Sie mit dem Native SDK für iOS auch aus einer Reihe allgemeiner Schaltflächen und Layouts auswählen können, können Sie auch Ihr eigenes benutzerdefiniertes Layout erstellen. Hier ist ein Beispiel für ein benutzerdefiniertes Layout.
  • Alternativ können Sie Schaltflächen entfernen, um ein einfaches und sauberes Layout zu erstellen.

Barrierefreiheit

Barrierefreiheitsfunktionen sind standardmäßig Bestandteil unserer Benutzeroberfläche, ohne dass eine zusätzliche Einrichtung erforderlich ist.

Hörgeschädigt

Die nativen Player-Steuerelemente von Brightcove integrieren eine Schaltfläche für geschlossene Untertitel in jedes Layout, um sicherzustellen, dass die Anzeige von Untertiteln leicht angepasst werden kann, wenn Untertitel verfügbar sind. Weitere Informationen zu Closed Captions mit den SDKs finden Sie in unserem Videoformate und Untertitel Sektion.
