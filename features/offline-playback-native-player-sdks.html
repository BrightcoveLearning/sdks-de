    Offline-Wiedergabe mit den Native Player-SDKs

    In diesem Thema erfahren Sie, wie die Offline-Wiedergabe mit den Brightcove Native Player SDKs funktioniert.

    Übersicht

    Die Offline-Wiedergabe mit den Brightcove Native Player SDKs für iOS und Android ermöglicht es Publishern, ihre Zuschauer auf völlig neue Weise zu erreichen und gleichzeitig sicherzustellen, dass ihre Inhalte sicher bleiben. Unsere SDKs bieten leistungsstark einfache Lösungen für komplexe Offline-Wiedergabeprobleme, einschließlich Download-Management, Offline-Katalogverwaltung, Analytik, DRM und natürlich der Wiedergabe.

    Mithilfe der Offline-Wiedergabe können Benutzer sowohl DRM-geschützte als auch klare (nicht verschlüsselte) Videoinhalte auf ihre Geräte herunterladen und diese ansehen, wenn sie nicht verbunden sind.

    Anforderungen

    Die Brightcove Native Player SDKs unterstützen die Offline-Wiedergabe mit den folgenden Versionen:

    Brightcove Native SDK-Version

    • Natives SDK für iOS 6.0.1+
    • Natives SDK für Android 5.0.3+

    Betriebssystemversion des Geräts

    • iOS 10.0+, aber 10.3+ wird empfohlen
    • Android 5.0+

    Setup

    Um loszulegen, machen Sie folgendes:

    • Wenden Sie sich an Ihren Account Manager, um Ihr Konto für die Offline-Wiedergabe zu
    • Stellen Sie sicher, dass Sie Dynamic Delivery verwenden, wenn Sie Ihre Videos aufnehmen.
    • Bereiten Sie jedes Video vor, das Sie zum Herunterladen aktivieren möchten.

    Bereitstellung von Inhalten

    Streaming-Video nutzt die Streaming-Funktion mit mehreren Bitraten von Video Cloud, die die Bandbreite an der Quelle erkennt und die beste Qualität für das Gerät bietet. Die Offline-Wiedergabe-Funktion unterstützt die folgenden Streaming-Typen:

    • Androide: Dynamisches adaptives Streaming über HTTP (DASH)
    • iOS: Apple HTTP-Live-Streaming (HLS)

    Sicherheit

    Der Schutz von heruntergeladenen Inhalten ist bei unserer Offline-Lösung im Auge. Die Brightcove Native Player SDKs bieten die folgenden Funktionen, um sicherzustellen, dass Ihre Inhalte sicher auf den Geräten der Zuschauer gespeichert werden:

    DRM-geschützte Inhalte

    • Für DRM-geschützte Inhalte verwenden die Native SDKs die folgenden Technologien:
      • Androide: STRICH mit Widevine Modular

      • iOS: HLS mit FairPlay

    • Videoinhalte können nur auf dem Gerät und der App angesehen werden, die den Inhalt heruntergeladen haben.

    • Videoinhalte können nur bis zu einem bestimmten Datum und einer bestimmten Uhrzeit angesehen werden.

      Mit den Native SDKs können Benutzer Videos ausleihen oder kaufen. Der Ablauf gilt für Anmietungen und wird in der DRM-Lizenz unter Verwendung der Native SDKs API zum Herunterladen von Videos festgelegt. Im Gegensatz dazu gilt der im Fastly-Token festgelegte Ablauf für Streaming-Inhalte.

    • An den Client zurückgegebene Video-URLs laufen ab, damit andere sie nicht verwenden können.

    Alle Inhalte

    • Wenn die App gelöscht wird, werden alle heruntergeladenen Inhalte entfernt.
      Für Android gilt dies, solange der Standardspeicherort nicht überschrieben wird.
    • Ihre App kann lokal gespeicherte Inhalte jederzeit löschen.

    Management herunterladen

    Die Brightcove Native Player SDKs behandeln die Komplexität des Herunterladens von Inhalten auf das Gerät eines Betrachters und vereinfachen den Prozess für native Anwendungen erheblich.

    • Publisher können die Wiedergabequalität der herunterzuladenden Inhalte auswählen.
    • Untertitel können heruntergeladen und offline angezeigt werden.
    • Mehrere Audiospuren werden von den Native SDKs unterstützt. Weitere Informationen zum Herunterladen zusätzlicher Tracks finden Sie in den folgenden Themen:
      • Mehrere Audiospuren für die Offline-Wiedergabe mit dem Native SDK für Android
      • iOS App Entwicklerhandbuch für Offline-Wiedergabe.

    funktionalität

    Die folgenden Funktionen können zu Ihren Apps hinzugefügt werden, um die Download-Verwaltung zu unterstützen:

    • Anzeige von Inhalten
    • Downloadgröße prüfen
    • Downloadstatus anzeigen
    • Verwalten von Downloads
    • Lokalen Speicherplatz anzeigen
    • Pausieren/Resume/Download abbrechen

    Anzeige von Inhalten

    Zeigen Sie eine Liste der Inhalte an, die zum Streamen oder Herunterladen zur Verfügung stehen.

    Wenn Sie die offline_enabled Eigenschaft eines Videos auf true setzen, markieren die Native SDKs dieses Video als herunterladbar.

    Bereiten Sie Videos

    Um ein Video als herunterladbar zu markieren, müssen Sie Folgendes tun:

    1. Wenn Sie dies noch nicht getan haben, wenden Sie sich an Ihren Account Manager, um Ihr Konto für die Offline-Wiedergabe zu aktivieren.

    2. Legen Sie für jedes Video, das Sie herunterladen möchten, offline als aktiviert ein, indem Sie einen der folgenden Schritte ausführen:

      Verwenden von Video Cloud Studio

      Verwenden Sie Video Cloud Studio, um Offline-Wiedergabe zu aktivieren.

      Verwenden Sie CMS API

      Sie können die CMS-API auch verwenden, um das Feld offline_enabled auf den Wert truefestzulegen.

      Hier ist ein Beispiel mit curl:

        curl --header "Authorization: Bearer $oauth_token" --request PATCH --data '
  {"offline_enabled" : true}
  ' https://cms.api.brightcove.com/v1/accounts/your account id/videos/your video id

    Wenn Sie mit klarem Inhalt arbeiten, finden Sie im Abschnitt Best Practices Tipps zur Anzeige Ihrer Inhalte.

    Downloadgröße prüfen

    Brightcove empfiehlt, dass Sie vor der Anforderung eines Downloads sicherstellen, dass genügend Speicherplatz für den Download im Gerätespeicher vorhanden ist. Gehen Sie folgendermaßen vor, um eine Schätzung der Gesamtgröße des Videodownloads zu erhalten:

    • Lesen Sie für iOS die Informationen im Abschnitt „ Downloadgröße prüfen “ des iOS App Developer's Guide to Offline Playback.

    • Für Android können Sie eine Schätzung im onDownloadStarted Callback von der erhalten DownloadEventListener. Hier ist ein Beispiel für das Festlegen des Ereignis-Listeners:

        MediaDownloadable.DownloadEventListener downloadEventListener = new MediaDownloadable.DownloadEventListener() {
  //Your implementation.
  }
  OfflineCatalog catalog = new OfflineCatalog(context, eventEmitter, accountId, policyKey);
  catalog.addDownloadEventListener(downloadEventListener);

     

    Lokalen Speicherplatz anzeigen

    Informieren Sie den Benutzer, wenn nicht genügend lokaler Speicher vorhanden ist, um das Herunterladen des aktuellen Elements und aller Elemente in der Warteschlange abzuschließen.

    Die SDKs geben die geschätzte Größe und den voraussichtlichen Fortschritt an, damit Sie feststellen können, ob genügend Speicherplatz für Downloads vorhanden ist.

    Downloadstatus anzeigen

    Zeigen Sie den aktuellen Download zusammen mit seinem Status an.

    Die geschätzte Gesamtgröße, der aktuelle Downloadstatus und der Prozentsatz der Informationen zum Downloadfortschritt werden von den SDKs zurückgegeben.

    Bei Android-Geräten wird der Downloadstatus auch im Benachrichtigungsbereich angezeigt. Der Titel der Benachrichtigung ist auf den Videotitel festgelegt. Die Benachrichtigung wird entfernt, wenn der Download pausiert oder abgebrochen wird. Einzelheiten finden Sie in den Code-Snippets mit dem Native SDK für Android-Dokument .

    Verwalten von Downloads

    Mehrere Videos herunterladen

    Die Native SDKs ermöglichen es, mehrere Videos für Downloads in die Warteschlange zu stellen. Beispielsweise kann ein Benutzer mehrere Episoden einer Staffel herunterladen, und die Native SDKs verwalten den Prozess, indem sie den Downloadstatus für jedes Asset bereitstellen.

    Wenn Ihre App auf iOS 11.0 - 11.2 abzielt, empfehlen wir keine gleichzeitigen Downloads, da das Verhalten von Pause und Fortsetzung bei mehreren gleichzeitigen Downloads unzuverlässig ist. Das Problem wurde mit iOS 11.3 behoben.

    Herunterladen von sekundären Tracks (iOS)

    Mit iOS 13 hat Apple die Funktionsweise von Downloads geändert. Einzelheiten finden Sie im Leitfaden zum Herunterladen von Videos und zur Offline-Wiedergabe von iOS App Developer's Guide.

    Aufgrund dieser Änderung scheint die iOS-Beispielanwendung Offline Download Videos mit mehreren Audiospuren erneut herunterzuladen. Dies liegt daran, dass für jeden Track der Download-Fortschritt gegeben wird. Der Fortschrittsbalken beginnt also für jede zusätzliche Audiospur erneut bei Null.

    Hier ist ein kurzer Überblick AVMediaSelectionOptions über Untertitel und alternative Audiotracks: Hinzufügen von Untertiteln und alternativen Audiospuren

    Bestimmen des vollständigen Downloads

    Gibt es eine Möglichkeit, einen vollständigen Download didProgressTo für einen Video-Token zu erhalten?

    Nein. Für ein Video-Offline-Token gibt es keinen aggregierten Wert für den Download-Fortschritt. Der App-Entwickler bestimmt, wie viele Asset-Downloads stattfinden werden. Der erste Asset-Download ist das Hauptvideo und die bevorzugten Audio- und Textspuren. Es ist in der Regel der größte Download und der Fortschritt wird von 0,00 auf 100,00 steigen.

    Wenn eine Download-Anfrage zusätzliche Medienauswahlen enthält (die der App-Entwickler explizit anfordert), ist die gesamte Downloadzeit einfach die der Hauptvideo- und der bevorzugten Medienauswahl sowie die jeder zusätzlichen Medienauswahl. Wenn Sie den Gesamtfortschritt in der Benutzeroberfläche darstellen, wählen Sie für jeden Download einen Skalierungswert, sodass die Summe aller Downloads 100% beträgt, und die skalierten Download-Zeiten jeder Medienauswahl summieren.

    Identifizieren Objekttypen

    Gibt es eine Möglichkeit zu identifizieren, auf welche Art von Objekt (Video oder Textspur) sich die didProgressTo Ereignisse beziehen?

    Ja. Die zusätzlichen Downloads bestehen aus Objekten der Apple Media Selection Option (AVMediaSelectionOption), nicht von Tracks. Das heruntergeladene Medienauswahlobjekt wird im Fortschrittsrückruf identifiziert. (bCOVofflineVideomanagerDelegate.h)

    Apple bietet Methoden zur Untersuchung der Attribute einer Medienauswahloption. Einzelheiten finden Sie unter AVMediaSelectionOptionvon Apple.

    Sie können Metadatenwerte in der Xcode-Debugger-Konsole anzeigen, wenn Sie die OfflinePlayer-Beispielanwendung ausführen (lassen Sie sich nicht von den beiden englischen Optionen irreführen - es ist nur eine Beispiel-App) 

    OfflinePlayer[523:355259] AVMediaSelection option 0 | legible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 0 | audible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 1 | legible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 1 | audible display name: English

    Wie das geht, sehen Sie im DownloadManager-Code.

    Pausieren/Resume/Download abbrechen

    Erlauben Sie dem Benutzer, den Download von Inhalten anzuhalten, fortzusetzen oder abzubrechen.

    Die Download-Funktion wird dort fortgesetzt, wo sie aufgehört hat, wenn der Benutzer pausiert oder wenn der Download unterbrochen wird. Wenn Sie beispielsweise während des Herunterladens die Wi-Fi-Verbindung verlieren, wird sie dort fortgesetzt, wo sie aufgehört hat, wenn Sie wieder verbunden sind.

    Wenn Sie einen Videodownload abbrechen, werden alle erzielten Fortschritte gelöscht.

    Verwaltung lokaler Kataloge

    Die Brightcove Native Player SDKs vereinfachen die Verwaltung von heruntergeladenen lokalen Inhalten.

    Die folgenden Funktionen können Ihren Apps hinzugefügt werden, um die lokale Katalogverwaltung zu unterstützen:

    • Downloadinhalte zeigen
    • Spielen Sie lokale Inhalte
    • Löschen lokaler Inhalte

    Downloadinhalte zeigen

    Zeige alle heruntergeladenen Folgen für jede Show/Staffel.

    Die SDKs geben Metadaten und Wiedergabedetails für alle heruntergeladenen Diese Metadaten umfassen Videotitel, Beschreibung, Miniaturansichten, verfügbare Untertitel, verfügbare Audiospuren usw.

    Es gibt neue Metadateneigenschaften, die für die Offline-Wiedergabe spezifisch sind und Folgendes umfassen:

    DRM-geschützte Inhalte

    • Token-ID
    • Ablaufdatum der Lizenz

    Alle Inhalte

    • Start- und Endzeit herunterladen
    • Der Name der Miniaturansicht wurde zusammen mit der Datei-URL heruntergeladen
    • Der Name des Posters zusammen mit seiner Datei-URL heruntergeladen
    • Relative und abgeleitete Datei-URL des Videobundles

    Sie können dann die Ergebnisse für Ihre spezifische Implementierung filtern.

    Sie können die verbleibende Zeit für eine Mietlizenz sehen.

    Löschen lokaler Inhalte

    Erlauben Sie Benutzern, Inhalte aus dem lokalen Speicher zu löschen.

    Die SDKs stellen eine Methode zum Löschen eines Videos zusammen mit seinen Metadaten und zugehörigen Assets bereit.

    Wiedergabe von Inhalten

    Mit den Native SDKs können Sie Inhalte in den folgenden Situationen wiedergeben:

    • Spielen Sie ein Video ab, während es heruntergeladen wird.
    • Spielen Sie ein Video ab, nachdem es heruntergeladen wurde. Heruntergeladene Inhalte können lokal wiedergegeben werden, wenn das Gerät offline oder online ist. Das Offline-Video kann nur aus dem lokalen Speicher abgespielt werden.
    • Wenn Sie eine Online-Version des Videos streamen möchten, sollten Sie ein neues Video-Objekt vom Playback-Dienst abrufen.

    Analytics

    Es wurden Analysen zur Messung von Daten im Zusammenhang mit dem Offline-Fluss eingeführt. Einzelheiten finden Sie im Dokument „ Analytics im Zusammenhang mit Offline-Wiedergabe mit dem Native SDKs “.

    Proben

    Die folgende Tabelle enthält Links zu Codebeispielen, die Ihnen zeigen, wie Sie mit der Offline-Wiedergabe beginnen:

    für LG herunterladen Beschreibung
    iOS Offline Wiedergabe Downloaden und spielen Sie HLS-Videos ab, einschließlich solcher, die mit FairPlay-Verschlüsselung geschützt sind. Einzelheiten zum Entwickler finden Sie im iOS App Developers Guide for Offline Playback.
    Android-Offline- Downloaden und spielen Sie DASH-Videos ab, einschließlich solcher, die mit Widevine-Verschlüsselung geschützt sind.

    Hinweise

    Die folgenden Hinweise gelten für die Offline-Wiedergabe-Funktion, die von den Native SDKs unterstützt wird:

    Alle

    Advertising

    Weder clientseitige oder serverseitige Werbung wird bei der Offline-Wiedergabe unterstützt.

    HLSe

    Derzeit werden hLSE-Inhalte bei der Offline-Wiedergabe mit dem Brightcove Native SDK für Android, iOS oder tvOS nicht unterstützt.

    Energiesparmodus

    Der Energiesparmodus auf Geräten kann sich auf bestimmte Funktionen auswirken, einschließlich des Herunterladens. Weitere Informationen finden Sie in den folgenden Themen:

    • Energiemanagement Android 9 (API-Ebene 28)
    • Energiesparmodus auf Ihrem iPhone
    • Reagieren auf Low-Power-Modus auf iPhones

    Geteilte Videos

    Geteilte Videos können nur dann zur Offline-Ansicht heruntergeladen werden, wenn das Originalvideo sein kann. Die Eigenschaft wird vom Originalvideo geerbt und kann nicht nur für freigegebene Videos geändert werden.

    Nur Audio

    Die Native SDKs unterstützen Nur-Audio-Inhalte mit Offline Playback. Sie steuern, was Zuschauer sehen, ob es sich um ein Posterbild oder einen benutzerdefinierten Bildschirm handelt.

    Das native SDK für Android erfordert ein VideoView, und das native SDK für iOS erfordert ein UIView. Aus diesem Grund wird Hintergrundaudio nicht unterstützt. Dies bedeutet, dass Audio nicht weitergespielt wird, wenn ein Benutzer die App in den Hintergrund schickt.

    Android

    App-Löschung

    Wenn die App gelöscht wird, werden alle heruntergeladenen Inhalte entfernt. Dies gilt, solange der Standardspeicherort nicht überschrieben wird.

    iOS

    AirPlay

    Als Sicherheitsfunktion können Sie kein Offline-HLS-Video zur Wiedergabe auf ein AirPlay-Gerät streamen. Dies wird von Apple als AVFoundation Einschränkung bestätigt.

    Häufige Fragen

    Im Folgenden finden Sie Antworten auf einige allgemeine Fragen.

    DRM-geschützte Inhalte

    Kann der Benutzer die Systemuhr ändern, um das Ablaufdatum zu umgehen?

    Androide: Absoluter Ablauf bestimmt das Datum und die Uhrzeit, bis zu der eine Lizenz gültig ist. Dies wird von Widevine und der Android DRM-Schicht durchgesetzt. Theoretisch muss die DRM-Schicht in Android verhindern, dass der Benutzer die Ablaufzeit umgeht. Alle Probleme, die wir möglicherweise gefunden haben, werden in den Versionshinweisen für Android-Betriebssysteme beschrieben.

    iOS: Der Ablauf der Lizenz wird von FairPlay durchgesetzt. iOS überprüft alle auf Uhrenänderungen usw. Eine App hat die Möglichkeit, den Ablauf der Lizenz zu überprüfen, um den Benutzer darauf hinzuweisen, dass die Lizenz abgelaufen ist. Dies kann vorzuziehen sein, wenn Sie einfach ein abgelaufenes Video wiedergeben und dann einen Wiedergabefehler erhalten.


    Werden die Lizenz und die Offline-Videos weiterhin geladen, während sich die App im Hintergrund befindet?

    Androide: Ja. In der Regel wird die Lizenz sofort heruntergeladen, aber selbst die Lizenz kann heruntergeladen werden, während die App im Hintergrund steht.

    iOS: Das Laden von Lizenzen sollte im Vordergrund erfolgen. Wir haben eine Methode, um die Lizenz vorab zu laden. Das Herunterladen der wichtigsten Videoinhalte kann erfolgen, während sich die App im Hintergrund befindet.

    Alle Inhalte

    Welches Videoformat wird zum Herunterladen von Offline-Inhalten verwendet?

    Androide: DRM-geschütztes DASH oder unverschlüsseltes DASH.

    iOS: HLS dampft. Sie können eine Bitrate für den Download angeben, um eine bestimmte Variante zu verwenden.


    Werden heruntergeladene Videos lokal wiedergegeben oder gestreamt, wenn der Benutzer online ist?

    Ein heruntergeladenes Video kann online oder offline wiedergegeben werden, verwendet jedoch immer das heruntergeladene Video. Wenn Sie eine Online-Version des Videos wiedergeben möchten, sollten Sie ein neues Video-Objekt vom Playback-Dienst abrufen. Das Offline-Video kann nur aus dem lokalen Speicher abgespielt werden.


    Best-Practices

    Die folgenden Richtlinien sollten bei der Entwicklung Ihrer App für die Offline-Wiedergabe hilfreich sein.

    Schaltflächen zum Pause/Abbrechen anzeigen

    Die Schaltflächen Pause und Abbrechen sollten angezeigt werden, wenn der Videodownload begonnen hat.

    Androide: Sie sollten die Schaltflächen Pause und Abbrechen im Rahmen des Aufrufs des onDownloadStarted Rückrufs im anzeigen DownloadEventListener. Einzelheiten finden Sie in der Beispiel-App für die Offline-Wiedergabe.

    iOS: Einzelheiten zum Erstellen einer App für die Offline-Wiedergabe finden Sie im iOS App Developer's Guide.

    Anzeige klarer Inhalte

    Wenn Ihr Konto nicht DRM-fähig ist oder Sie mit klarem Inhalt arbeiten, führen die Brightcove Native SDKs keine serverseitige Lizenzanforderung aus. Daher werden klare Inhalte nicht als zur Vermietung oder zum Kauf verfügbar angesehen. In den meisten Fällen wird empfohlen, die Leih- und Kauf-Buttons für klare Inhalte auszublenden und stattdessen einen einzigen Download-Button anzuzeigen.

    Aktivieren der Funktion

    Wenden Sie sich an Ihren Account Manager, um mehr über die Preise zu erfahren und die Offline Playback-Funktion für Ihr Konto zu aktivieren

    Fehlerbehebung

    Die folgenden Tipps können Ihnen dabei helfen, Probleme beim Herunterladen von Videos zu untersuchen.

    Ein Fehler kann auftreten, wenn ein Benutzer versucht, ein Asset mit einer höheren Bitrate herunterzuladen, als das Gerät aufgrund eines langsamen Netzwerks verarbeiten kann.

    Um dies zu vermeiden, können Sie Code hinzufügen, um die Gerätefunktionen zu überprüfen und zu verhindern, dass Geräte mit langsamen Verbindungen versuchen, Assets mit hoher Bitrate herunterzuladen. Sie können mit einem echten Gerät testen, während Sie die Verbindung drosseln, und den Netzwerkverkehr mit dem Charles-Proxy-Tool anzeigen.

    Android

    Einzelheiten zur Behandlung von Fehlern finden Sie im Dokument Fehlermeldungen aus dem Native SDK for Android . Dies schließt Fehlermeldungen im Zusammenhang mit der Offline-Wiedergabe ein.

    iOS

    Einzelheiten zur Behandlung von Fehlern finden Sie im Dokument Error Handling with the Native SDK for iOS .

    Wenn der Offline-Wiedergabecode einen Fehler erhält, fangen wir ihn auf und werfen einen NSError. Um den zugrunde liegenden Fehler zu erhalten, können Sie Folgendes versuchen:

      NSError *underlyingError = initialError.userInfo[NSUnderlyingErrorKey];

    Weitere Informationen finden Sie im iOS App Developers Guide for Offline Playback.
