Überblick

BCOVPUILayoutView-Ansichtsobjekte werden verwendet, um Steuerelemente in einem BCOVPUIBasicControlView. Layoutansichten werden normalerweise mit BCOVPUIBasicControlView erstellt’s +layoutViewWithControlFromTag:width:elasticity:-Methode und enthalten ein standardmäßiges Player-UI-Element wie eine Wiedergabeschaltfläche, ein Timecode-Label oder einen Fortschrittsschieberegler. Sie können auch leere Layoutansichten erstellen, um Ihre eigenen Unteransichten hinzuzufügen. Die Eigenschaften der Layoutansicht bestimmen genau, wie die Ansichten in Layouts mit variabler Breite zueinander positioniert werden.

  Mindestbreite

Die kleinste Breite, die für die Ansicht festgelegt wird, wenn die Größe für das Layout geändert wird. Wenn Elastizität auf Null gesetzt ist, ist dies die genaue Breite der Ansicht.

@property (nichtatomar, readwrite) float minimumWidth

  Elastizität

Die Elastizität bestimmt, wie stark die Größe jeder Ansicht geändert wird, um die verfügbare Breite der Player-Benutzeroberfläche auszufüllen’s Steuerleiste relativ zu allen anderen Ansichten. Wenn in einer Steuerleiste zwei Layoutansichten vorhanden sind und eine eine Elastizität von 1,0 und eine andere eine Elastizität von 2,0 hat, wächst letztere bei einer Größenänderung der Ansicht doppelt so stark an. Jede Zeile eines Steuerelementlayouts sollte normalerweise mindestens eine Layoutansicht mit einer Elastizität größer Null enthalten. Eine Ansicht mit einer Elastizität von Null bleibt auf ihrer minimalen Breite.

@property (nichtatomar, readwrite) Float-Elastizität

  ENTFERNT

Die “ENTFERNT” -Eigenschaft kann auf JA oder NEIN gesetzt werden, um eine bestimmte Layoutansicht schnell auszublenden oder anzuzeigen. Das ist effizienter, als ein komplett neues BCOVPUIControlLayout mit einer entfernten oder hinzugefügten Ansicht. Rufen Sie -setNeedsLayout auf der BCOVPUIBasicControlView nachdem Sie diese Eigenschaft festgelegt haben, um das neue Layout anzuzeigen.

@property (nonatomic, readwrite, getter=isRemoved) BOOL entfernt
