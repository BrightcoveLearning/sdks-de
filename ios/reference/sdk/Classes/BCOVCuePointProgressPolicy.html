Überblick

Diese Richtlinie bietet Clients von BCOVPlayerSDK die Möglichkeit, das Verhalten von Code anzugeben, der als Reaktion auf Cue-Point-Ereignisse reagiert. Die häufigste Anwendung hierfür ist das Erstellen von Anzeigenrichtlinien: Der Abspielkopf erreicht einen Cue-Point, der angibt, dass eine Anzeige geschaltet werden soll, und eine Instanz dieser Richtlinie wird vom Clientcode bereitgestellt, um anzugeben, wie sich der Player in Bezug auf Folgendes verhalten soll:

  1. Welche Cue-Punkte vom Code verarbeitet werden sollen, der die Richtlinie konsultiert.
  2. Die Zeit (Abspielposition), zu der die Wiedergabe fortgesetzt werden soll, nachdem die Cue-Punkte verarbeitet wurden. Dies wird mit einem der Cue-Points angegeben, der an die Richtlinie übergeben wurde, oder Null (was angibt, dass der Inhalt an seiner aktuellen Abspielposition fortgesetzt werden soll).

– ApplyToEvent:

Gibt zurück BCOVCuePointProgressPolicyResult die angibt, welche Cue-Punkte vom Code verarbeitet werden sollen, der diese Richtlinie konsultiert, sowie die Position, an der die Wiedergabe fortgesetzt werden soll, nachdem alle Cue-Punkte verarbeitet wurden.

- (BCOVCuePointProgressPolicyResult *)applyToEvent:(NSDictionary *)cuePointEvent

Parameter

cuePointEvent

Ein Wörterbuch mit Informationen zu Cue-Point-Ereignissen, wie in der BCOVPlaybackController Cue-Point-Delegatenmethode.

Rückgabewert

Ein Richtlinienergebnis für das angegebene Cue-Point-Ereignis.

Diskussion

Gibt zurück BCOVCuePointProgressPolicyResult die angibt, welche Cue-Punkte vom Code verarbeitet werden sollen, der diese Richtlinie konsultiert, sowie die Position, an der die Wiedergabe fortgesetzt werden soll, nachdem alle Cue-Punkte verarbeitet wurden.

Diese Methode wird vom Verbraucher der Cue-Point-Fortschrittsrichtlinie aufgerufen. In den meisten Fällen ist dieser Consumer ein Werbe-Plug-in für das Brightcove Player SDK, nicht der Client (App) des Player SDK. Eine benutzerdefinierte Cue-Point-Richtlinie sollte diese Methode überschreiben, wenn eine Richtlinie nicht mit einer Convenience-Factory-Klassenmethode erstellt werden kann.

Parameter

cuePointsToProcess

EIN BCOVProgressPolicyCuePointsToProcess die die zurückgegebene Richtlinie konfiguriert, um ihrem Aufrufer mitzuteilen, welche Cue-Points aus den Cue-Points in einem bestimmten Cue-Point-Ereignis verarbeitet werden sollen.
fortsetzenPosition

EIN BCOVProgressPolicyResumePosition das konfiguriert die zurückgegebene Richtlinie, um ihrem Aufrufer mitzuteilen, wo die Inhaltswiedergabe fortgesetzt werden soll, sobald alle angegebenen Cue-Punkte verarbeitet wurden.
