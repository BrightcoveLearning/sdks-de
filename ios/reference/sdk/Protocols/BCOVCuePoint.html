Überblick

Wertklasse, die einen Cue-Point im Brightcove Player SDK-Ökosystem darstellt. Ein Cue-Punkt wird verwendet, um a . anzugeben Position auf der Timeline eines Inhalts, der für jeden Zweck verwendet werden kann (oft zur programmatischen Kontrolle darüber, wann Werbung gezeigt werden soll). Wenn die Wiedergabe einen Cue-Punkt erreicht, sendet die Wiedergabesitzung ein Ereignis auf ihrem cuePoints Signal, das den BCOVCuePoint-Wert enthält.

  Position erforderliche Methode

Die Zeit, zu der der Cue-Punkt auf der Timeline des BCOVVideo der es besitzt.

– aktualisieren: erforderliche Methode

Gibt eine geänderte Version dieses Cue-Points zurück. Da BCOVCuePoint-Objekte unveränderlich sind, muss ein völlig neuer BCOVCuePoint erstellt werden, selbst wenn nur eine einzige Änderung erforderlich ist. Daher bietet diese Methode eine bequeme Möglichkeit, eine temporäre veränderbare Kopie dieser Instanz zu erhalten und sie zu ändern (innerhalb von updateBlock).

- (Instanztyp)update:(void ( ^ ) ( id<BCOVMutableCuePoint> mutableCuePoint ))updateBlock

Parameter

updateBlock

Ein Block, dem eine veränderbare Kopie dieses Cue-Punkts übergeben wird.

Rückgabewert

Die Kopie dieses Cue-Points geändert von updateBlock.

Diskussion
