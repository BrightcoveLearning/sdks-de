Überblick

Implementieren Sie dieses Protokoll bei der Interaktion mit einem benutzerdefinierten FPS-Lizenzserver.

– contentIdentifierFromLoadingRequest:

Optionale Methode. Implementieren Sie diese Methode, um das Abrufen der Inhaltskennung aus der Ladeanforderung zu verarbeiten. Dies wird für jedes Asset aufgerufen. Es wird keine Gewähr dafür übernommen, dass der Thread aufgerufen wird. Sie sollten die dieser Ladeanforderung entsprechende Inhaltskennung zurückgeben. Dies wird normalerweise aus dem extrahiert “skd://…” URL in loadingRequest.request.URL. Die Inhalts-ID wird an [AVAssetResourceLoadingRequest streamingContentKeyRequestDataForApp:contentIdentifier:options:error:] übergeben, um den SPC abzurufen.

- (nullable NSData *)contentIdentifierFromLoadingRequest:(nonnull AVAssetResourceLoadingRequest *)loadingAnfrage

Parameter

loadingAnfrage

Die skd://-Ladeanforderung für das Asset.

Diskussion

Optionale Methode. Implementieren Sie diese Methode, um das Abrufen der Inhaltskennung aus der Ladeanforderung zu verarbeiten. Dies wird für jedes Asset aufgerufen. Es wird keine Gewähr dafür übernommen, dass der Thread aufgerufen wird. Sie sollten die dieser Ladeanforderung entsprechende Inhaltskennung zurückgeben. Dies wird normalerweise aus dem extrahiert “skd://…” URL in loadingRequest.request.URL. Die Inhalts-ID wird an [AVAssetResourceLoadingRequest streamingContentKeyRequestDataForApp:contentIdentifier:options:error:] übergeben, um den SPC abzurufen.

Wenn diese Methode nicht bereitgestellt wird, wird die Standardmethode verwendet. Die Standardmethode entfernt “skd://” from loadingRequest.request.URL, um die Inhaltskennung zu generieren.
