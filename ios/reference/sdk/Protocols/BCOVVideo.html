Überblick

Ein Video im Brightcove Player SDK-Ökosystem. Brightcove organisiert Inhalte in Entitäten namens Videos, die null oder mehr haben können Quellen Informationen enthalten, die für den Zugriff auf den Inhalt spezifisch sind. Das Video bietet einen Container, um auf diese zu verweisen Quellen aggregiert und ein Ort für Metadaten über das Video oder die Sammlung von Quellen in Form von Eigenschaften. Außerdem kann ein Video über eine Cue-Point-Sammlung verfügen, bei der es sich um eine Reihe von Cue-Points handelt, die bestimmte Orte im Video beschreiben’s Inhaltszeitleiste.

  cuePoints erforderliche Methode

– aktualisieren: erforderliche Methode

Gibt eine geänderte Version dieser Quelle zurück. Da BCOVVideo-Objekte unveränderlich sind, muss ein völlig neues BCOVVideo erstellt werden, selbst wenn nur eine einzige Änderung erforderlich ist. Daher bietet diese Methode eine bequeme Möglichkeit, eine temporäre veränderbare Kopie dieser Instanz zu erhalten und sie zu ändern (innerhalb von updateBlock).

- (Instanztyp)update:(void ( ^ ) ( id<BCOVMutableVideo> mutableVideo ))updateBlock

Parameter

updateBlock

Ein Block, dem eine veränderliche Kopie dieses Videos übergeben wird.

Rückgabewert

Die Kopie dieses Videos geändert von updateBlock.

Diskussion

Gibt eine geänderte Version dieser Quelle zurück. Da BCOVVideo-Objekte unveränderlich sind, muss ein völlig neues BCOVVideo erstellt werden, selbst wenn nur eine einzige Änderung erforderlich ist. Daher bietet diese Methode eine bequeme Möglichkeit, eine temporäre veränderbare Kopie dieser Instanz zu erhalten und sie zu ändern (innerhalb von updateBlock).

Versuchen Sie nicht, einen Verweis auf das veränderbare Video zu speichern, das an . übergeben wurde updateBlock , da das Verhalten dabei undefiniert ist. Nehmen Sie stattdessen alle erforderlichen Änderungen im Block vor und erfassen Sie dann das von dieser Methode zurückgegebene Videoobjekt.
