Überblick

Wenn diese Eigenschaften geändert werden, werden die Änderungen bei der nächsten bereitgestellten Sitzung wirksam. Um diese Eigenschaften auf alle Sitzungen anzuwenden, ändern Sie sie vor dem Aufruf an [BCOVPlaybackController setVideos:].

  Konto erforderliche Methode

Diese Eigenschaft legt den Wert der Konto-ID für Brightcove Analytics fest. Durch Festlegen dieser Eigenschaft wird auch der Wert accountId für jedes Video ersetzt, das über eine Brightcove Media API-Anforderung abgerufen wird.

@property (nichtatomar, Kopie) NSString *Konto

  Ziel erforderliche Methode

Diese Eigenschaft muss ein URI mit einer gültigen Struktur sein und eine Autorität enthalten. Der Standardwert für diese Eigenschaft, wenn sie nicht überschrieben wird, ist “bcsdk://” gefolgt von der Bundle-ID.

@property (nichtatomar, Kopie) NSString *Ziel

Diskussion

Diese Eigenschaft muss ein URI mit einer gültigen Struktur sein und eine Autorität enthalten. Der Standardwert für diese Eigenschaft, wenn sie nicht überschrieben wird, ist “bcsdk://” gefolgt von der Bundle-ID.

Bitte beziehen Sie sich auf http://en.wikipedia.org/wiki/URI_scheme#Generic_syntax für weitere Informationen und Beispiele zur URI-Syntax.

Insbesondere ein Ziel ohne einen hierarchischen Teil (zB nur ein Schema) gilt als ungültig, ebenso wie jeder Wert ohne Schema.

  uniqueIdentifierEnabled erforderliche Methode

Diese Eigenschaft schaltet die Erzeugung eindeutiger Kennungen auf der Clientseite um. Wenn aktiviert, identifiziert das SDK mithilfe des Geräts eindeutige Benutzer’s Anbieterkennung. Wenn es deaktiviert ist, bietet das SDK keinen eindeutigen Wert und die Analyse stützt sich auf serverseitige Heuristiken, um eindeutige Elemente zu bestimmen.

@property (nonatomic, Assign, getter=isUniqueIdentifierEnabled) BOOL uniqueIdentifierEnabled

Diskussion

Diese Eigenschaft schaltet die Erzeugung eindeutiger Kennungen auf der Clientseite um. Wenn aktiviert, identifiziert das SDK mithilfe des Geräts eindeutige Benutzer’s Anbieterkennung. Wenn es deaktiviert ist, bietet das SDK keinen eindeutigen Wert und die Analyse stützt sich auf serverseitige Heuristiken, um eindeutige Elemente zu bestimmen.

Der Standardwert ist JA.
