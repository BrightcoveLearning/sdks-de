Protokoll, das von Objekten übernommen wird, die Wiedergabefunktionen bereitstellen.

Implementierungen dieses formalen Protokolls müssen seine Standard-Wiedergabeoperationen unterstützen, können jedoch die API erweitern, um zusätzliche Funktionen auszuführen, die für ihren Funktionssatz spezifisch sind.

  WerbungDeaktiviert erforderliche Methode

Nur SSAI: Wenn Sie andere Anzeigen-Plug-ins verwenden, verwenden Sie seekWithoutAds:completionHandler: stattdessen.

@property (nonatomic, readwrite) BOOL adsDisabled

Diskussion

Nur SSAI: Wenn Sie andere Anzeigen-Plug-ins verwenden, verwenden Sie seekWithoutAds:completionHandler: stattdessen.

Deaktiviert die Anzeigenwiedergabe, die andernfalls nach einem forward -seekTo: erfolgen könnte. Der Verwendungszweck besteht beispielsweise darin, die Videowiedergabe fortzusetzen, wenn eine App beendet und neu gestartet wurde; Die App kann nach einer gespeicherten Zeit suchen und Anzeigen, die zuvor angesehen wurden, werden nicht angezeigt abspielen wieder. Setzen Sie adsDisabled auf YES, rufen Sie dann eine der -seekToTime:-Methoden auf und setzen Sie adsDisabled schließlich im -seekToTime:-Abschlusshandler auf NO. Der Standardwert von adsDisabled ist NO.

Beachten Sie, dass, es sei denn, die BCOVPlaybackSessionLifecycleEvent kBCOVPlaybackSessionLifecycleEventReady empfangen wurde, wird der -seekTo:-Abschlusshandler nie aufgerufen. Aus diesem Grund wird empfohlen, -seekTo: aus dem -playbackController:playbackSession:didReceiveLifecycleEvent heraus aufzurufen delegieren -Methode für das kBCOVPlaybackSessionLifecycleEventReady-Ereignis.

– addFairPlayApplicationCertificate:Kennung: erforderliche Methode

Diese Methode wird verwendet, um dem Wiedergabe-Controller ein FairPlay-Anwendungszertifikat hinzuzufügen’s Liste der verfügbaren FairPLAy-Anwendungszertifikate. Zertifikate werden für die Lebensdauer des Playback-Controllers aufbewahrt.

- (ungültig)FairPlayApplicationCertificate hinzufügen:(NSData *)applicationCertificateData Bezeichner:(NSString *)Kennung

Diskussion

Diese Methode wird verwendet, um dem Wiedergabe-Controller ein FairPlay-Anwendungszertifikat hinzuzufügen’s Liste der verfügbaren FairPLAy-Anwendungszertifikate. Zertifikate werden für die Lebensdauer des Playback-Controllers aufbewahrt.

Wenn Sie Dynamic Delivery verwenden, werden Anwendungszertifikate automatisch vom FairPlay-Plugin abgerufen, sodass diese Methode nicht erforderlich ist. Sie können diese Methode jedoch verwenden, um Ihr Bewerbungszertifikat vorab zu laden, um die Wiedergabe des ersten FairPlay-verschlüsselten Videos zu beschleunigen. Zertifikate werden gespeichert und für nachfolgende Videos wiederverwendet.

@param applicationCertificateData Das FairPlay-Anwendungszertifikat in einem NSData-Objekt. Wenn auf nil gesetzt, werden alle vorhandenen Anwendungszertifikate für den angegebenen Bezeichner entfernt. @param bezeichner Eine Zeichenfolge, die zum Auffinden des Anwendungszertifikats verwendet wird. Diese Zeichenfolge darf nicht null sein. - Bei Dynamic Delivery muss der Bezeichner die URL sein, die zum Abrufen dieses Zertifikats verwendet wurde. - Bei älteren Video Cloud-Konten sollte dies auf kBCOVDefaultFairPlayApplicationCertificateIdentifier gesetzt werden, um auf alle Konten angewendet zu werden. - Wenn Sie mehrere ältere Video Cloud-Konten verwenden, setzen Sie diesen Parameter auf die Konto-ID.
