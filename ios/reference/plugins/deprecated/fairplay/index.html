BCOVFairPlay-Referenz

FairPlay-Plugin für Brightcove Player SDK für iOS, Version 1.1.3.89

Unterstützte Plattformen

iOS 8.0 und höher.

tvOS 9.0 und höher.

Installation

Das FairPlay-Plug-in für das Brightcove Player SDK bietet zwei Installationspakete für iOS, ein statisches Bibliotheksframework und ein dynamisches Framework. Die Bereitstellung wird unter iOS 8.0 und höher unterstützt.

Das FairPlay-Plugin bietet ein dynamisches Framework zur Unterstützung von tvOS 9.0 und höher.

Bitte beachten Sie die folgenden iOS/tvOS + FairPlay-Einschränkungen:

  • Obwohl das Plugin Simulatorarchitekturen enthält, können Sie FairPlay-geschützte Inhalte nicht in einem Simulator abspielen.

  • FairPlay-Streaming-Inhalte werden im AirPlay-Spiegelungsmodus nicht gerendert. Weitere Informationen finden Sie in der WWDC-Präsentation 502 von 2015: Inhaltsschutz für HLS.

Kakaoschoten

Sie können verwenden Kakaoschoten um das Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-SDK-FairPlay Podspec Hier. Die Podspec unterstützt sowohl iOS als auch tvOS. CocoaPods-Version 1.0 oder höher ist erforderlich.

Angeben des Standard-Pods Brightcove-Player-SDK-FairPlay installiert das statische Bibliotheksframework. Um das dynamische Framework zu installieren, deklarieren Sie den Pod mit dem dynamisch Unterspezifikation: Brightcove-Player-SDK-FairPlay/dynamisch

Beispiel für ein statisches Framework:

Pod 'Brightcove-Player-SDK-FairPlay'

Beispiel für ein dynamisches Framework:

pod 'Brightcove-Player-SDK-FairPlay/dynamic'

Die Pflege eines aktuellen Master-Podspec-Repositorys ist erforderlich, um sicherzustellen, dass Sie immer die neuesten Versionen der Brightcove-Software verwenden. Ab CocoaPods 1.0.0 sind Podspec-Repository-Updates keine automatische Funktion mehr. Um Ihr Master-Repository zu aktualisieren, führen Sie Folgendes auf der Befehlszeile aus:

Pod-Repository-Update

Handbuch

So fügen Sie Ihrem Projekt das FairPlay-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des Plugins von unserem . herunter Release-Seite.
  3. Ergänzen Sie die BrightcoveFairPlay.framework zu Ihrem Projekt.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. Auf der “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie Folgendes zum “Binärdateien mit Bibliotheken verknüpfen” Sektion:
    • BrightcoveFairPlay.framework
  6. (Nur dynamisches Framework) Auf dem “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu ‘BrightcoveFairPlay.framework’ zum “Eingebettete Binärdatei” Sektion.
  7. (Nur dynamisches Framework) Auf dem “Build-Phasen” Registerkarte, fügen Sie ein “Skript ausführen” Phase mit dem Befehl Schlag ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcoveFairPlay.framework/strip-frameworks.sh. Prüfen “Skript nur bei der Installation ausführen”. Dadurch werden nicht benötigte Architekturen aus dem Build entfernt, was für die Einreichung im App Store wichtig ist.
  8. (nur statisches Framework) Auf dem “Build-Einstellungen” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu -ObjC zum “Andere Linker-Flags” Einstellung bauen.

Importe

Das Brightcove Player SDK für iOS kann auf verschiedene Weise in Code importiert werden. @import BrightcoveFairPlay; , #import <BrightcoveFairPlay/BrightcoveFairPlay.h> oder #import <BrightcoveFairPlay/[spezifische Klasse].h>.

Schnellstart

Brightcove FairPlay ist eine Brücke zwischen Apple FairPlay und dem Brightcove Player-SDK für iOS. Die folgenden Abschnitte zeigen, wie Sie Ihren Code für die grundlegende Wiedergabe einrichten.

Dynamische Bereitstellung von Video Cloud

Wenn Sie die dynamische Bereitstellung von Video Cloud verwenden, wird ein Verweis auf das FairPlay-Anwendungszertifikat in das vom Wiedergabedienst zurückgegebene BCOVVideo-Objekt eingebettet. Das SDK verwendet diese Referenz, um das Anwendungszertifikat abzurufen, wenn Sie Ihre FairPlay-Videos abspielen.

Hier sind die grundlegenden Schritte, die Sie befolgen müssen:

@property (nichtatomar, stark) id<BCOVPlaybackController>playbackController;
---
---- // Erstellen Sie ein BCOVFPSBrightcoveAuthProxy-Objekt mit der // Anwendungs-ID und der Herausgeber-ID, die zum Registrieren Ihrer // FairPlay-Anmeldeinformationen bei Brightcove verwendet wurden. BCOVFPSBrightcoveAuthProxy *proxy = [[BCOVFPSBrightcoveAuthProxy alloc] initWithPublisherId:<pub-id> applicationId:<app-id>]; BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager sharedManager]; // Brightcove FairPlay fügt BCOVPlaybackManager einige Kategoriemethoden hinzu. // Wenn Sie Ihren eigenen FairPlay-Lizenzserver verwenden, geben Sie hier // Ihre eigene Proxy-Implementierung ein. id<BCOVPlaybackController>playbackController = [sdkManager createFairPlayPlaybackControllerWithAuthorizationProxy:proxy]; playController.delegate = self; _playbackController = PlaybackController; [self.view addSubview:playbackController.view]; BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithAccountId:<Konto-ID> policyKey:<Richtlinienschlüssel>]; [playbackService findVideoWithVideoID:<video-id> Parameter:@{} Vervollständigung:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *Fehler) { if(video) { [self.playbackController setVideos:@[video]]; } }];

Legacy-Video Cloud-Workflow

Wenn Sie einen älteren Video Cloud-Workflow verwenden, müssen Sie vor der Wiedergabe Ihrer Videos ein FairPlay-Zertifikat abrufen. Dieser Code zeigt die grundlegende Einrichtung und Wiedergabe:

@property (nichtatomar, stark) id<BCOVPlaybackController>playbackController;
---
---- // Erstellen Sie ein BCOVFPSBrightcoveAuthProxy-Objekt mit der // Anwendungs-ID und der Herausgeber-ID, die zum Registrieren Ihrer // FairPlay-Anmeldeinformationen bei Brightcove verwendet wurden. BCOVFPSBrightcoveAuthProxy *proxy = [[BCOVFPSBrightcoveAuthProxy alloc] initWithPublisherId:<pub-id> applicationId:<app-id> BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager sharedManager]; // Erstellen Sie Ihren FairPlay-Wiedergabe-Controller mit dem Authentifizierungs-Proxy. id<BCOVPlaybackController>playbackController = [sdkManager createFairPlayPlaybackControllerWithAuthorizationProxy:proxy]; playController.delegate = self; _playbackController = PlaybackController; [self.view addSubview:playbackController.view]; // Rufen Sie mithilfe des Proxys das FairPlay-Anwendungszertifikat ab. // In einer Produktions-App möchten Sie dies möglicherweise tun, sobald // die App gestartet wird. [proxy retrieveApplicationCertificate:^(NSData * _Nullable applicationCertificate, NSError * _Nullable Fehler) { if (applicationCertificate) { // Add the application certificate to the playback controller [playbackController addFairPlayApplicationCertificate:applicationCertificate identifier:kBCOVDefaultFairPlayApplicationCertificateIdentifier]; BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithAccountId:<account-id> policyKey:<policy-key>]; [playbackService findVideoWithVideoID:<video-id> parameters:@{} Vervollständigung:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *Fehler) { if(video) { [playbackController setVideos:@[video]]; } }]; } }];

Sie können dem Wiedergabe-Controller mehrere FairPlay-Anwendungszertifikate hinzufügen, aber bei der Legacy-Wiedergabe benötigen Sie normalerweise nur eines. Wenn Sie Videos von zwei verschiedenen Konten mit demselben Wiedergabe-Controller wiedergeben, können Sie für jedes Konto ein Anwendungszertifikat mit der Konto-ID als “Kennung” Parameter.

Wenn Sie ältere Methoden zum Erstellen des Wiedergabe-Controllers verwenden, funktionieren diese Methoden weiterhin. Die neueren Methoden wurden entwickelt, um mehr Flexibilität zu bieten, indem Sie das FairPlay-Anwendungszertifikat vor oder nach der Erstellung des Wiedergabe-Controllers abrufen können’Sie müssen die Bewerbungsbescheinigung überhaupt nicht abrufen oder weitergeben.

Wenn Sie Fragen haben oder Hilfe benötigen, haben wir ein Support-Forum für Brightcove’s native Player-SDKs unter [https://groups.google.com/forum/](https://groups.google.com/forum/)#!forum/brightcove-native-player-sdks .

Bewerbungszertifikate

Die Wiedergabe von FairPlay-geschützten Videos erfordert ein mit dem Inhalt verknüpftes Anwendungszertifikat.

Wenn Sie die dynamische Bereitstellung von Video Cloud verwenden, werden Anwendungszertifikate heruntergeladen und nach Bedarf verwendet. Zertifikate werden zur Wiederverwendung zwischengespeichert, sodass in der Regel ein einzelner Download pro Konto erfolgt.

Wenn Sie ältere Video Cloud-Dienste verwenden, müssen Sie dennoch ein Anwendungszertifikat abrufen, um es mit Ihren Initialisierungsmethoden zu übergeben. Die BCOVFPSBrightcoveAuthProxy -Klasse bietet eine bequeme Methode, um dies von fps.brightcove.com abzurufen.

Wenn Sie Ihr eigenes Bewerbungszertifikat aus der Ferne speichern, können Sie es mit einem normalen NSURLSessionDataTask Abrufen des Zertifikats als NSData Objekt. Sobald Sie das Zertifikat erhalten haben, können Sie es mit zu Ihrem Wiedergabe-Controller hinzufügen [BCOVPlaybackController addFairPlayApplicationCertificate:identifier:]. In den meisten Fällen würden Sie verwenden kBCOVDefaultFairPlayApplicationCertificateIdentifier als Kennung.

Verwenden eines FairPlay-Lizenzservers eines Drittanbieters

Die BCOVFPSBrightcoveAuthProxy soll mit arbeiten fps.brightcove.com. Wenn Sie einen anderen FairPlay-Lizenzserver verwenden müssen, können Sie entweder die BCOVFPSAuthorizationProxy Protokoll, um die Lizenzkommunikation abzuwickeln, oder wenn Ihre Anforderungen einfach sind, können Sie einige der BCOVFPSAuthorizationProxy Eigenschaften:

/** * Die Basis-URL für FairPlay-bezogene Lizenzanforderungen. Die Standard-URL verweist auf * fps.brightcove.com. * Wenn auf nil gesetzt, wird die Standard-NSURL, die auf fps.brightcove.com zeigt, neu erstellt. */ @property (nichtatomar, stark, null_resettable) NSURL *fpsBaseURL; /** * Die Schlüsselanforderungs-URL für FairPlay-bezogene Schlüsselanforderungen. * Normalerweise auf nil gesetzt. In diesem Fall wird die Schlüsselanforderungs-URL * aus der Antwort der Video Cloud-Wiedergabe-API abgerufen. */ @property (nichtatomar, stark, null_resettable) NSURL *keyRequestURL;

Die fpsBaseURL -Eigenschaft wird auch verwendet, um die Basis-URL für Anwendungszertifikatsanforderungen festzulegen. Zertifikate werden wie folgt angefordert:

[fpsBaseURL]/application_certificate/[publisherID]/[applicationID]

Ob Sie Video Cloud oder Ihren eigenen Server verwenden, Ihre BCOVVideo Objekt sollte a . enthalten BCOVQuelle -Objekt, dessen URL auf einige FairPlay-geschützte Inhalte verweist. Ungeschützte Video-URLs werden weiterhin in einem FairPlay-Workflow wiedergegeben.
