BCOVPlayerUI-Referenz

BCOVPlayerUI-Plug-in für Brightcove Player SDK für iOS, Version 1.0.4.143

Unterstützte Plattformen

iOS 7.0 und höher.

Installation

Das Brightcove PlayerUI-Plug-in bietet zwei Installationspakete für iOS, ein statisches Bibliotheksframework und ein dynamisches Framework. Das statische Bibliotheksziel unterstützt die Bereitstellung unter iOS 7, während das dynamische Framework nur iOS 8 und höher unterstützt.

Kakaoschoten

Sie können verwenden Kakaoschoten um das PlayerUI-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-SDK-Player-UI Podspec Hier.

Beispiel für ein statisches Framework:

Pod 'Brightcove-Player-SDK-Player-UI'

Beispiel für ein dynamisches Framework:

Pod 'Brightcove-Player-SDK-Player-UI/dynamisch'

Handbuch

So fügen Sie das PlayerUI-Plug-in für Brightcove Player SDK manuell zu Ihrem Projekt hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des Plugins von unserem . herunter Release-Seite.
  3. Ergänzen Sie die ‘BrightcovePlayerUI.framework’ zu Ihrem Projekt.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. Auf der “Build-Phasen” Registerkarte Ihres Anwendungsziels, fügen Sie Folgendes zum “Binärdateien mit Bibliotheken verknüpfen” Phase:
    • BrightcovePlayerUI.framework
  6. (Nur dynamisches Framework) Auf dem “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu ‘BrightcovePlayerUI.framework’ zum “Eingebettete Binärdatei” Sektion.
  7. (Nur dynamisches Framework) Auf dem “Build-Phasen” Registerkarte, fügen Sie ein “Skript ausführen” Phase mit dem Befehl Schlag ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcovePlayerUI.framework/strip-frameworks.sh. Prüfen “Skript nur bei der Installation ausführen”. Dadurch werden nicht benötigte Architekturen aus dem Build entfernt, was für die Einreichung im App Store wichtig ist. ([rdar://19209161][19209161])
  8. (nur statisches Framework) Auf dem “Build-Einstellungen” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu -ObjC zum “Andere Linker-Flags” Einstellung bauen.
  9. (Nur statisches Framework) Datei hinzufügen bcovpuiiconfont.ttf von dem BrightcovePlayerUI.framework Bundle direkt in Ihren Projekteintrag, sodass die Schriftartendatei in das App-Bundle kopiert wird. Am Ende sollte die Schriftartendatei auf dem gleichen Level wie die App landen’S Info.plist Datei. Die Schriftartdatei stellt einige der BrightcovePlayerUI-Oberflächenelemente bereit, muss jedoch nicht in der plist selbst aufgeführt werden.

Importe

Das Brightcove Player SDK für iOS kann auf verschiedene Weise in Code importiert werden. @import BrightcovePlayerUI; , #import <BrightcovePlayerUI/BrightcovePlayerUI.h> oder #import <BrightcovePlayerUI/[spezifische Klasse].h>.

Schnellstart

Seine Verwendung ist relativ einfach. In diesem Beispiel wird ein grundlegendes Layout erstellt, das für allgemeine On-Demand-Videostreams geeignet ist:

 BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; [1] id<BCOVPlaybackController> Controller = [Manager createPlaybackController]; [2] BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:playbackController]; playerView.frame = self.videoContainer.bounds; playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth; [self.videoContainer addSubview:playerView]; [3] NSString *Token; // (Brightcove Media API-Token mit URL-Zugriff) NSString *playlistID; // (ID der Playlist, die Sie verwenden möchten) BCOVCatalogService *catalog = [[BCOVCatalogService alloc] initWithToken:token]; [Katalog findPlaylistWithPlaylistID:playlistID Parameter:nil Vervollständigung:^(BCOVPlaylist *playlist, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:playlist]; [controller play]; }];

Lassen’s zerlegen Sie diesen Code in Schritte, um ihn etwas einfacher zu verdauen:

  1. Erstellen Sie einen Wiedergabe-Controller mit seinen Standardeinstellungen.
  2. Erstellen Sie eine Player-Ansicht mit dem Wiedergabe-Controller. Dadurch wird eine Spieleransicht mit ihren Standardeinstellungen instanziiert. Stellen Sie die Player-Ansicht so ein, dass sie der Größe Ihrer Video-Container-Ansicht entspricht.
  3. Abrufen und Abspielen eines Videos.

Anpassen des Plugin-Verhaltens

Das BCOVPlayerUI-Plugin bietet ein einfaches Out-of-the-Box-Erlebnis. Wie auch immer, es’Es ist möglich, das Standardverhalten zu ändern.

BCOVPUIPlayerViewOptions

Die BCOVPUIPlayerViewOptions -Klasse können Sie einige BCOVPlayerUI-Verhalten bei der Initialisierung anpassen. Sie können anpassen:

  • jumpBackInterval Die Zeit in Sekunden, in der der Player zurückspringt, wenn die Rücksprungtaste gedrückt wird.
  • hideControlsInterval Die Zeit in Sekunden nach dem letzten Touch-Ereignis, bevor die Steuerelemente ausgeblendet werden.
  • HideControlsAnimationDauer Die Zeit in Sekunden, die es dauert, bis die Steuerelemente ausgeblendet animiert sind.
  • showControlsAnimationDauer Die Zeit in Sekunden, die es dauert, bis die Steuerelemente sichtbar animiert sind.
  • präsentierenViewController Der Ansichtscontroller, der zum Präsentieren von Dingen wie Modals und Vollbild verwendet werden soll.

Optionen können mit der folgenden Methode festgelegt werden:

 BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController> Controller = [Manager createPlaybackController]; BCOVPUIPlayerViewOptions *options = [[BCOVPUIPlayerViewOptions alloc] init]; options.jumpBackInterval = 5; BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:playbackController options:options];

Mitgelieferte Layouts

Drei Layouts werden bereitgestellt, um verschiedene Arten von Videos zu unterstützen:

  • BCOVPUIControlLayout basicVODControlLayout ist ein grundlegendes Layout für allgemeine On-Demand-Videostreams.

  • BCOVPUIControlLayout basicLiveControlLayout ist ein auf Live-Video ausgerichtetes Layout.

  • BCOVPUIControlLayout basicLiveDVRControlLayout ist auf Live-Videostreams mit DVR-Steuerung ausgerichtet.

Normalerweise legen Sie ein neues Layout unmittelbar nach Ihrem . fest BCOVPUIPlayerView erstellt wird, Sie können aber auch jederzeit ein neues Layout festlegen. Sie können beispielsweise ein neues VOD-Layout wie folgt festlegen:

playerview.controlView.layout = [BCOVPUIControlLayout basicVODControlLayout]

Benutzerdefinierte Layouts

Zusätzlich zu den Standardlayouts können Sie Ihre eigenen hochgradig angepassten Layouts erstellen, indem Sie ein neues instanziieren BCOVPUIControlLayout mit Ihrem eigenen Design.

Erstellen Sie zuerst die Steuerelemente, die in Ihr Layout eingefügt werden, mit BCOVPUIBasicControlView-LayoutViewWithControlFromTag:width:elastity:. Jedes Steuerelement ist verpackt in a BCOVPUILayoutView das bestimmt den Kontrollabstand. Sie können die Breite jeder Layoutansicht auf die Standardbreite (basierend auf dem Steuerelementtyp) oder Sie können Ihre eigene Breite angeben.

Die Elastizität -Argument bestimmt, wie stark die Layoutansicht, die das Steuerelement enthält, ihre Breite ändert, um die Steuerleiste auszufüllen. Eine Elastizität von Null bedeutet, dass die Größe der Layoutansicht festgelegt wird. Ein Elastizitätswert größer als Null bestimmt, wie stark die Layoutansicht vergrößert wird, um den verfügbaren Platz relativ zu allen anderen elastischen Ansichten in dieser Steuerleiste auszufüllen. Eine Layoutansicht mit einem Elastizitätswert von 2,0 wächst doppelt so schnell wie eine Layoutansicht mit einem Elastizitätswert von 1,0. Normalerweise hat eine Reihe von Layoutansichten mindestens ein Steuerelement mit einer Elastizität größer als Null.

Hier sind Beispiele für das Erstellen einer Vielzahl von grundlegenden Steuerelementen.

BCOVPUILayoutView *playbackLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonPlayback width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *jumpBackButtonLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonJumpBack width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *currentTimeLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagLabelCurrentTime width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *timeSeparatorLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagLabelTimeSeparator width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *durationLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagLabelDuration width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *progressLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagSliderProgress width:kBCOVPUILayoutUseDefaultValue Elastizität:1.0]; BCOVPUILayoutView *closedCaptionLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonClosedCaption width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *screenModeLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagButtonScreenMode width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *externalRouteLayoutView = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewExternalRoute width:kBCOVPUILayoutUseDefaultValue Elastizität:0.0]; BCOVPUILayoutView *spacerLayoutView1 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewLeere Breite:1.0 Elastizität:1.0]; BCOVPUILayoutView *spacerLayoutView2 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewLeere Breite:1.0 Elastizität:1.0]; BCOVPUILayoutView *logoLayoutView1 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewLeere Breite:80.0 Elastizität:1.0]; BCOVPUILayoutView *logoLayoutView2 = [BCOVPUIBasicControlView layoutViewWithControlFromTag:BCOVPUIViewTagViewLeere Breite:36.0 Elastizität:0.0]; // Untertitel und AirPlay-Steuerelemente ausblenden, bis sie ausdrücklich benötigt werden. closedCaptionLayoutView.removed = JA; externalRouteLayoutView.removed = JA;

Beachten Sie, dass Sie auch eine leere Layout-Ansicht erstellen können, in die Sie Ihre eigene Ansicht (Logo, Control, nichts usw.) platzieren können. Dieser Code zeigt, wie Sie ein UIImage-Logo in das oben erstellte logoLayoutView1 einfügen.

// Logobild in einer Bildansicht zur Anzeige in der Steuerleiste erstellen. UIImage *logoImage1 = [UIImage imageNamed:@"myLogo"]; UIImageView *logoImageView1 = [[UIImageView alloc] initWithImage:logoImage1]; logoImageView1.autoresizingMask = UIViewAutoresizingFlexibleWidth | UIViewAutoresizingFlexibleHeight; logoImageView1.contentMode = UIViewContentModeScaleAspectFit; logoImageView1.frame = logoLayoutView1.frame; // Bildansicht zu unserer leeren Layoutansicht hinzufügen. [logoLayoutView1 addSubview:logoImageView1];

Nachdem wir nun eine Sammlung von Steuerelementen in Layout-Ansichten verpackt haben, können wir sie in Arrays anordnen, von denen jedes eine einzelne Zeile von Steuerelementen darstellt, dh eine Steuerleiste. Beachten Sie, dass Sie unterschiedliche Layouts für Hoch- und Querformat verwenden können, sodass Sie normalerweise zwei verschiedene Arrays von Steuerleisten einrichten.

In diesem Standardlayout für die Ausrichtung im Querformat werden wir unsere Steuerelemente in einer einzigen Zeile anordnen und dann dieses Array in ein anderes Array einfügen, das die gesamte Konfiguration darstellt.

 NSArray *standardLayoutLine1 = @[ PlaybackLayoutView, jumpBackButtonLayoutView, currentTimeLayoutView, timeSeparatorLayoutView, DurationLayoutView, progressLayoutView, spacerLayoutView1, logoLayoutView1, spacerLayoutView2, closedCaptionLayoutView, screenModeLayoutView, externalRoute];Layout NSArray *standardLayoutLines = @[ standardLayoutLine1 ];

Im kompakten Layout erstellen wir für die Ausrichtung im Hochformat zwei Arrays von Steuerelementen, und dann werden diese beiden Arrays in ein anderes Array verpackt, das das kompakte Layout repräsentiert.

Beachten Sie, dass wir für die meisten Steuerelemente in jedem Layout genau dieselben Objekte verwendet haben. Wenn dies erledigt ist und Sie zwischen Hoch- und Querformat wechseln, wird das Objekt mit einer weichen Animation an seine neue Position verschoben.

 NSArray *compactLayoutLine1 = @[ currentTimeLayoutView, progressLayoutView, DurationLayoutView ]; NSArray *compactLayoutLine2 = @[ PlaybackLayoutView, jumpBackButtonLayoutView, spacerLayoutView1, closedCaptionLayoutView, screenModeLayoutView, externalRouteLayoutView, logoLayoutView2 ]; NSArray *compactLayoutLines = @[ compactLayoutLine1, compactLayoutLine2];

Da wir nun endlich zwei Layout-Konfigurationen haben, eine für die volle Breite und eine für die kompakte Breite, können wir eine neue erstellen BCOVPUIControlLayout Objekt und legen Sie es in den Player’s Kontrollansicht.

 BCOVPUIControlLayout *customLayout = [[BCOVPUIControlLayout alloc] initWithStandardControls:standardLayoutLines compactControls:compactLayoutLines]; playerView.controlView.layout = customLayout;

Wenn Sie Steuerelemente haben, die Sie häufig ein- oder ausblenden müssen, können Sie die ENTFERNT Eigenschaft auf diesem Steuerelement’s-Layout-Ansicht. Wenn Sie Ihre Steuerung geändert haben, rufen Sie an setNeedsLayout auf dem PlayerView’s controlView:

 logoLayoutView1.removed = JA; [playerView.controlsView setNeedsLayout];

Sie können auch mehrere allgemeine BCOVPUIControlLayout Eigenschaften:

  • controlBarHeight legt die Größe jeder Reihe von Steuerelementen fest.
  • horizontalItemSpacing legt den Abstand zwischen den einzelnen fest BCOVPUILayoutView in jeder Kontrollleiste.
  • KompaktLayoutMaximale Breite bestimmt, welcher Satz von Steuerelementen verwendet wird. Wenn die Kontrollansicht kleiner ist als KompaktLayoutMaximale Breite , wird die Kompaktsteuerung verwendet, ansonsten werden die Standardsteuerungen verwendet.

Um die angezeigten Steuerelemente zu ändern, müssen Sie eine neue erstellen und installieren BCOVPUIControlLayout.

Häufig gestellte Fragen

Funktioniert dies mit allen Brightcove Player SDK-Plugins?

Diese Version unterstützt grundlegende VOD-, Live- und Live-DVR-Wiedergabe ohne Werbung. Wir werden in zukünftigen Versionen Unterstützung für andere Plugins hinzufügen.

Wie kann ich andere als die oben genannten Eigenschaften anpassen?

Unser Ziel ist es, etwas zu schaffen, das’Es lässt sich einfach anpassen und bietet dennoch viele Funktionen und Verhaltensweisen, die sofort einsatzbereit sind. In kommenden Versionen werden wir mehr Anpassungen mit neuen APIs und mehr Dokumentation abdecken. Wenn Sie eine Funktion suchen, teilen Sie uns dies bitte in unserem Forum mit.

Wird es in zukünftigen Versionen bahnbrechende API-Änderungen geben?

Wir werden unser Bestes tun, um die Dinge nicht zu ändern, es sei denn, wir müssen es wirklich. Wenn wir jedoch API-Änderungen vornehmen müssen, werden wir im CHANGELOG klare Migrationsschritte bereitstellen.
