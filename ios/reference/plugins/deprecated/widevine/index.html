BCOVWidevine Referenz

Widevine-Plug-in für Brightcove Player SDK für iOS, Version 2.1.4.358

Unterstützte Plattformen

iOS 8.0 und 9.0. Widevine läuft nicht auf dem iOS-Simulator.

Wichtig - Bitte beachten

Das Widevine-Plug-in für Brightcove Player SDK für iOS ist veraltet.

Installation

Das Widevine-Plug-in für Brightcove Player SDK bietet ein statisches Bibliotheksframework für die Installation.

Kakaoschoten

Sie können verwenden Kakaobohnen um das Widevine-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-SDK-Widevine Podspec Hier. Um diese Spezifikation zu verwenden, fügen Sie oben in Podfile Folgendes hinzu:Quelle 'https://github.com/brightcove/BCOVSpecs.git' oder laufe Pod-Repository BCOVSpecs hinzufügen https://github.com/brightcove/BCOVSpecs.git. CocoaPods 0.39 oder neuer ist erforderlich.

Das Widevine-SDK ist nicht in diesem Pod enthalten. Du muss fügen Sie die Widevine-Bibliothek manuell zu Ihrem Projekt hinzu, auch wenn Sie Cocoapods verwenden. Der Pod fügt jedoch alle Bibliotheken hinzu, die von der Widevine-Bibliothek benötigt werden, wie z libc++ , libxml2 , libresolv , libSystem , und libz. Bitte wenden Sie sich an den Brightcove-Support, um Zugriff auf die Widevine-Bibliothek zu erhalten.

So installieren Sie die Widevine-Bibliothek manuell:

  1. Fordern Sie die Widevine-Bibliothek beim Brightcove-Support an.
  2. Fügen Sie den Inhalt des Pakets zu Ihrem Projekt hinzu.
  3. Auf der “Build-Phasen” Registerkarte Ihres Anwendungsziels, fügen Sie Folgendes zum “Binärdateien mit Bibliotheken verknüpfen” Phase:
    • libWViPhoneAPI.a
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels:
    • Stellen Sie sicher, dass sich Widevine-Header in Ihrer Anwendung befinden’S “Header-Suchpfad”.
    • Sicher gehen, dass -ObjC wurde hinzugefügt “Andere Linker-Flags” Einstellung bauen.

Beispiel für ein statisches Framework:

Quelle 'https://github.com/CocoaPods/Specs.git' Quelle 'https://github.com/brightcove/BCOVSpecs.git' Pod 'Brightcove-Player-SDK-Widevine'

Handbuch

So fügen Sie Ihrem Projekt das Widevine-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des Plugins von unserem . herunter Release-Seite.
  3. Fordern Sie die Widevine-Bibliothek vom Brightcove-Support an und fügen Sie sie Ihrem Projekt hinzu.
  4. Ergänzen Sie die ‘BrightcoveWidevine.framework’ zu Ihrem Projekt.
  5. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  6. Auf der “Build-Phasen” Registerkarte Ihres Anwendungsziels, fügen Sie Folgendes zum “Binärdateien mit Bibliotheken verknüpfen” Phase:
    • BrightcoveWidevine.framework
    • libWViPhoneAPI.a
    • libc++
    • libxml2
    • libresolv
    • libSystem
    • libz
  7. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels:
    • Sicher gehen, dass -ObjC wurde hinzugefügt “Andere Linker-Flags” Einstellung bauen.
    • Stellen Sie sicher, dass sich Widevine-Header in Ihrer Anwendung befinden’S “Header-Suchpfad”.

Schnellstart

 NSString *token; // (Brightcove Media API-Token mit URL-Zugriff) NSString *playlistID; // (ID der Playlist, die Sie verwenden möchten) BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; [1] id<BCOVPlaybackController> controller = [playbackManager createWidevinePlaybackControllerWithViewStrategy:nil]viewStrategy:nil]; [self.view addSubview:controller.view]; BCOVDatalogService *catalog = [[BCOVDatalogService alloc] initWithToken:token]; [2] [Katalog findWidevinePlaylistWithPlaylistID:playlistID Parameter:nil Vervollständigung:^(BCOVPlaylist *playlist, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:playlist]; [controller play]; }];

Lassen’s zerlegen Sie diesen Code in Schritte, um ihn etwas einfacher zu verdauen:

  1. BCOVWidevine fügt dem BCOVPlaybackManager einige Kategoriemethoden hinzu. Die erste davon ist -createWidevinePlaybackControllerWithViewStrategy. Verwenden Sie diese Methode, um Ihren Wiedergabe-Controller zu erstellen. Alternativ (wenn Sie mehr als einen Sitzungsanbieter verwenden) können Sie eine BCOVWidevineSessionProvider und übergeben Sie dies an die Manager-Methode, die einen Wiedergabe-Controller mit Upstream-Sitzungsanbietern erstellt.*
  2. BCOVWidevine fügt BCOVPlaybackManager einige Kategoriemethoden hinzu, um Widevine-Wiedergaben aus dem Katalog abzurufen.

* Beachten Sie, dass BCOVWidevineSessionProvider vor allen Anzeigensitzungsanbietern in der Kette stehen sollte, die beim Erstellen des Wiedergabe-Controllers an den Manager übergeben werden.

Wenn Sie Fragen haben oder Hilfe benötigen, haben wir ein Support-Forum für Brightcove’s native Player-SDKs unter https://groups.google.com/forum/#!forum/brightcove-native-player-sdks .

Anpassen des Plugin-Verhaltens

Sie können das Standardverhalten des Plugins anpassen, indem Sie eine Instanz von . erstellen BCOVWidevineSessionProviderOptions und Überschreiben der Standardeigenschaften. Um a . zu verwenden BCOVWideSessionProviderOptions options-Instanz müssen Sie die BCOVWideSessionProvider mit [BCOVSDKManager createWidevineSessionProviderWithOptions:].

 BCOVWidevineSessionProviderOptions *options = [[BCOVWidevineSessionProviderOptions alloc] init]; options.widevineSettings = @{ WVPlayerDrivenAdaptationKey: @0 } ; id<BCOVPlaybackSessionProvider> sessionProvider = [playbackManager createWidevineSessionProviderWithOptions:options]; id<BCOVPlaybackController>playbackController = [playbackManager createPlaybackControllerWithSessionProvider:sessionProvider viewStrategy:nil];

Quellenauswahl (WVM, HTTP/HTTPs)

Das Brightcove Player SDK für iOS bietet Clients die Möglichkeit, mehrere URLs und Übermittlungstypen (BCOVQuelle ) zu einem einzelnen Video (BCOVVideo). Wenn Ihre Videos beispielsweise vom Katalog- oder Wiedergabedienst abgerufen werden, kann es für ein einzelnes Video eine Mischung aus WVM-, HLS- oder MP4-Versionen sowie HTTP- und HTTPs-Versionen geben. Welche dieser Quellen ausgewählt wird, wird durch einen Quellenauswahlblock bestimmt. Die vom Widevine-Plugin verwendete Standard-Quellenauswahlrichtlinie wählt das erste Widevine . aus BCOVQuelle auf jeder BCOVVideo , unabhängig vom Schema.

Die Quellenauswahl kann überschrieben werden, indem a . erstellt wird BCOVWidevineSessionProviderOptions und verwenden Sie es, um a . zu erstellen BCOVWidevineSessionProvider. Zum Beispiel:

BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager sharedManager]; BCOVWidevineSessionProviderOptions *options = [[BCOVWidevineSessionProviderOptions alloc] init]; options.sourceSelectionPolicy = <policy> id<BCOVWidevineSessionProvider> provider = [sdkManager createWidevineSessionProviderWithOptions:options]; id<BCOVPlaybackController>playbackController [sdkManager createPlaybackControllerWithSessionProvider:provider viewStrategy:nil];

Wenn diese Standardauswahlrichtlinie für Sie nicht funktioniert, gibt es einige Alternativen zur Auswahl einer Quelle:

  • Beim Abrufen von Videos aus Video Cloud über den Katalogdienst oder den Wiedergabedienst vor dem Anrufen [BCOVPlaybackController setVideos:] , verwenden Sie die Update-Methode auf dem BCOVVideo nur die gewünschte Quelle enthalten (siehe “Werte” Abschnitt für weitere Informationen).

  • Wenn Sie HTTPS-Quellen bevorzugen, bietet das Widevine-Plugin zusätzliche Optionen zur Auswahl von Widevine-Quellen mit einem bestimmten Schema. [BCOVWidevineSourceSelectionPolicy sourceSelectionWidevineWithScheme:kBCOVSourceURLSchemeHTTPS] stellt eine Richtlinie zur Quellenauswahl bereit, die Widevine-Quellen bevorzugt, die HTTPS verwenden. Dadurch werden Nicht-HTTP-URLs nicht in HTTPs-URLs konvertiert. Wenn Sie HTTPs auswählen, stellen Sie sicher, dass Ihr CDN für HTTPs konfiguriert ist. Wenn das CDN für HTTPs konfiguriert ist, verwenden Sie BCOVPlaybackService Anstatt von BCOV KatalogService um Video-/Playlist-Metadaten abzurufen.

  • Ähnlich wie beim Aktualisieren des Videoobjekts können Sie auch Ihren eigenen Quellenauswahlblock implementieren.

     options.sourceSelectionPolicy = ^ BCOVSource *(BCOVVideo *video) { <Video.sources auf Quelle prüfen>
