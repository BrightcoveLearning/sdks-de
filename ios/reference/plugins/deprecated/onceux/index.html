Brightcove Native Player für iOS-Referenz

Brightcove OnceUX-Plug-in für Brightcove Player SDK für iOS, Version 6.4.3.616

Hinweis zur Einstellung:

Das OnceUX-Plug-in wird durch das SSAI-Plug-in für Brightcove Player SDK für iOS ersetzt. Alle Funktionen von OnceUX sind in SSAI verfügbar. Siehe die “Migration von OnceUX zu SSAI” Abschnitt dieser README-Datei für Details zum Upgrade Ihrer Anwendung.

Unterstützte Plattformen

iOS 10.0 und höher.

tvOS 9.0 und höher.

Installation

Das Brightcove OnceUX für Brightcove Player SDK bietet zwei Installationspakete für iOS, ein statisches Bibliotheksframework und ein dynamisches Framework.

Kakaoschoten

Sie können verwenden Kakaoschoten um das OnceUX-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-OnceUX Podspec Hier.

Beispiel für ein statisches Framework:

Quelle 'https://github.com/brightcove/BrightcoveSpecs.git' Pod 'Brightcove-Player-OnceUX'

Beispiel für ein dynamisches Framework:

Quelle 'https://github.com/brightcove/BrightcoveSpecs.git' Pod 'Brightcove-Player-OnceUX/dynamic'

Wenn Sie Ihre Installation aktualisieren,’Es ist eine gute Idee, die lokale Kopie Ihres BrightcoveSpecs-Repositorys zu aktualisieren, damit Sie lokal über die neuesten Podspecs verfügen, genau wie Sie Ihr CococaPods-Master-Repository aktualisieren würden.

Normalerweise, wenn Sie verwenden Pod-Update im Terminal geschieht dies automatisch, oder Sie können alternativ explizit mit aktualisieren Pod-Repo-Update Brightcove. (Ihr BrightcoveSpecs-Repository kann einen anderen Namen haben, wenn Sie es explizit zu Ihrer Liste der Podspecs-Repositorys hinzugefügt haben.)

Handbuch

So fügen Sie Ihrem Projekt das OUX-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des BrightcoveOUX-Plugins von unserem . herunter Release-Seite.
  3. Hinzufügen BrightcoveOUX.framework zu Ihrem Projekt.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. (Nur dynamisches Framework) Auf dem “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu ‘BrightcoveOUX.framework’ zum “Eingebettete Binärdateien” Sektion.
  6. (Nur dynamisches Framework) Auf dem “Build-Phasen” Registerkarte, fügen Sie ein “Skript ausführen” Phase mit dem Befehl Schlag ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcoveOUX.framework/strip-frameworks.sh. Prüfen “Skript nur bei der Installation ausführen”. Dadurch werden nicht benötigte Architekturen aus dem Build entfernt, was für die Einreichung im App Store wichtig ist.
  7. (nur statisches Framework) Auf dem “Build-Einstellungen” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu -ObjC zum “Andere Linker-Flags” Einstellung bauen.

Importe

Das OUX-Plug-in für Brightcove Player SDK für iOS kann auf verschiedene Weise in Code importiert werden. @import BrightcoveOUX; , #import <BrightcoveOUX/BrightcoveOUX.h> oder #import <BrightcoveOUX/[spezifische Klasse].h>.

Schnellstart

BrightcoveOUX ist ein Plugin für Brightcove Player-SDK für iOS die Unterstützung für das serverseitige Anzeigen-Stitching von Brightcove OnceUX bietet. Seine Verwendung ist relativ einfach.

[1] BCOVOUXAdComponentDisplayContainer *displayContainer = [[BCOVOUXAdComponentDisplayContainer alloc] initWithCompanionSlots:@[]]; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; [2] id<BCOVPlaybackController>playbackController = [sdkManager createOUXPlaybackControllerWithViewStrategy:nil]; [3] [playbackController addSessionConsumer:displayContainer]; [4] [self.videoContainer addSubview:playbackController.view]; [5] BCOVVideo *video = [BCOVVideo videoWithURL:[NSURL URLWithString:<url-to-once-ux-video>]]; [6] [playbackController setVideos:@[video]]; [WiedergabeController-Wiedergabe];

Zusammenfassen:

  1. Erstellen Sie zuerst a BCOVOUXAdComponentDisplayContainer. Dieses Objekt hilft bei der Verwaltung von Companion-Slots. Geben Sie ggf. die vorhandenen Companion-Slots ein.
  2. BrightcoveOUX fügt einige Kategoriemethoden hinzu BCOVPlaybackManager. Die erste davon ist -createOUXPlaybackControllerWithViewStrategie:. Verwenden Sie diese Methode, um Ihren Wiedergabe-Controller zu erstellen. Sie werden in der Regel bestehen Null für die Ansichtsstrategie.
  3. Damit die BCOVOUXAdComponentDisplayContainer Um Anzeigeninformationen anzuzeigen und Companion-Anzeigenansichten zu füllen, muss sie als Sitzungsnutzer hinzugefügt werden.
  4. Fügen Sie den Wiedergabe-Controller hinzu’s Ansicht zum Videocontainer in Ihrer eigenen Ansichtshierarchie.
  5. Ein ... kreieren BCOVVideo Verwenden Sie die URL zu Ihrem OnceUX-Video.
  6. Laden Sie das Video in den Wiedergabecontroller.

Wenn Sie Fragen haben oder Hilfe benötigen, besuchen Sie das Supportforum für Brightcove Native Player SDKs unter [https://groups.google.com/forum/](https://groups.google.com/forum/)#!forum/brightcove-native-player-sdks .

Migration von OnceUX zu SSAI

Das BrightcoveSSAI-Plugin-Framework ersetzt BrightcoveOUX. BrightcoveSSAI unterstützt alle aktuellen Funktionen von BrightcoveOUX, zukünftige Updates werden jedoch nur BrightcoveSSAI hinzugefügt. Befolgen Sie diese Anweisungen, um Ihre OnceUX-Anwendung zu Brightcove SSAI zu migrieren.

1) Wenn Sie Brightcove-Plugins mit CocoaPods installieren, aktualisieren Sie Ihre Poddatei, indem Sie Brightcove-Player-OnceUX zu Brightcove-Player-SSAI , zum Beispiel dies: Ziel 'MyVideoPlayer' do pod 'Brightcove-Player-OnceUX/dynamic' Ende wird das: Ziel 'MyVideoPlayer' do Pod 'Brightcove-Player-SSAI/dynamisch' Ende Wenn Sie statische Frameworks verwenden, lassen Sie es weg /dynamisch so was: Pod 'Brightcove-Player-SSAI'. Lauf 'Pod-Update' wenn bereit.

Wenn Sie die Brightcove-Plug-ins manuell installieren, löschen Sie das Brightcove-Player-OnceUX-Framework aus Ihrem Xcode-Projekt und führen Sie die obigen Schritte aus, um das Brightcove-Player-SSAI-Framework manuell hinzuzufügen.

2) Ändern Sie in Ihrem Quellcode für alle @import / #import-Anweisungen, Datentypen, Methodennamen, Eigenschaften, Konstanten und Protokolle, die sich auf das BrightcoveOUX-Framework beziehen, jedes Vorkommen von “OUX” zu “SSAI”. Eine unvollständige Liste ist hier:

"@import BrightcoveOUX;" ist jetzt "@import BrightcoveSSAI;" "createOUXSessionProviderWithUpstreamSessionProvider" ist jetzt "createSSAISessionProviderWithUpstreamSessionProvider" "createOUXPlaybackControllerWithViewStrategy" ist jetzt "createSSAIPlaybackControllerWithViewStrategy" "oux_seekToTime" ist jetzt "ssai_seekToTime" "CompAI

Abrufen von Informationen zur Anzeigenwiedergabe

BrightcoveOUX bietet Informationen zur Anzeigenwiedergabe über das BCOVPlaybackControllerAdsDelegate. Wenn Sie beispielsweise Ihre Steuerelemente während einer Anzeige ausblenden möchten, können Sie Folgendes implementieren: [BCOVPlaybackControllerAdsDelegieren von PlaybackController:playbackSession:didEnterAdSequence:] sie zu verstecken.

Weitere Informationen zur Verwendung dieser Delegatmethoden finden Sie unter Brightcove Player-SDK für iOS.

Ich suche

Das BrightcoveOUX-Plugin bietet eine Suchfunktion, die beim Implementieren von Steuerelementen verwendet werden sollte. Diese Suchfunktion wird in der Sitzung durch die AnbieterErweiterung Eigentum. So wird es verwendet:

CMTime contentTimeToSeekTo = <Berechnung-aus-Scrub-Leiste>; [self.currentSession.providerExtension oux_seekToTime:contentTimeToSeekTo completeHandler:^(BOOL beendet) { if (finished) { [self.currentSession.player play]; } }];

Die FertigstellungHandler wird nach Abschluss einer erfolgreichen Suche ausgeführt. Es wird nicht ausgeführt, wenn eine Suche bereits durch einen vorherigen Aufruf an . eingeleitet wurde -[BCOVSessionProviderExtension oux_seekToTime:completionHandler:] oder wenn eine Anzeige wiedergegeben wird. Um zu testen, ob ein Suchversuch möglich ist, überprüfen Sie das -[BCOVSessionProviderExtension oux_canSeek] Eigentum. Weitere Informationen zu diesen beiden Methoden finden Sie im headerdoc.

Suche mit deaktivierten Anzeigen

Das BrightcovePlayerSDK bietet eine BOOL-Eigenschaft zum Deaktivieren von Anzeigen während der Suche. Die beabsichtigte Verwendung von OnceUX besteht darin, einer Anwendung zu ermöglichen, die Videowiedergabe nach dem Neustart fortzusetzen, ohne dass der Endbenutzer Anzeigen anzeigen muss, die sie’schon gesehen.

Es wird empfohlen, dem kBCOVPlaybackSessionLifecycleEventReady Handler der -playbackController:playbackSession:didReceiveLifecycleEvent: Methode von dir BCOVPlaybackController delegieren.

- (void)playbackController:(id<BCOVPlaybackController>)ControllerplaybackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVPlaybackSessionLifecycleEventReady isEqualToString:lifecycleEvent.eventType]) { // disable ads. _playbackController.adsDisabled = YES; // seek somewhere into the video content. [session.providerExtension oux_seekToTime:resumeTime completionHandler:^(BOOL finished) { // re-enable ads. _playbackController.adsDisabled = NO; // open the shutter. _playbackController.shutterFadeTime = 0.25; _playbackController.shutter = NO; }]; }

Beim Anrufen oux_seekTo:Abschluss: Um die Wiedergabe zu einem bestimmten Zeitpunkt fortzusetzen, wird das erste Bild des Videos angezeigt, bis die Suche abgeschlossen ist. Für eine sauberere Darstellung decken Sie die Videoansicht während der Suche vorübergehend ab, indem Sie BOOL . einstellen Verschluss Eigenschaft von BCOVPlabackController to JAWOHL bevor du anrufst -setVideos:. Wenn die Suche abgeschlossen ist, schließen Sie den Verschluss, indem Sie die Verschluss Eigentum an NEIN. Die ShutterFadeTime -Eigenschaft definiert die Dauer der Blendenanimation.

self.playbackController = [sdkManager createOUXPlaybackControllerWithViewStrategy:[sdkManager BCOVOUXdefaultControlsViewStrategy]]; // Fortsetzen der Wiedergabe, also aktivieren Sie den Verschluss, bevor Sie das Video laden. self.playbackController.shutterFadeTime = 0.0; self.playbackController.shutter = JA; NSArray *videos = @[[BCOVVideo videoWithURL: [NSURL URLWithString:onceUxUrl]]]; [self.playbackController setVideos:Videos];

Integrierte Bedienelemente der Player-Benutzeroberfläche

Das BrightcovePlayerSDK bietet einen integrierten Satz von UI-Steuerelementen, die mit dem OnceUX-Plugin sowohl für grundlegende Wiedergabe- als auch für Anzeigensteuerelemente verwendet werden können. Um die Steuerelemente zu verwenden, erstellen Sie a BCOVPUIPlayerView , und verknüpfen Sie es mit Ihrem OnceUX-Wiedergabe-Controller.

Erstellen Sie zunächst eine playerView-Eigenschaft in Ihrer Klasse.

@property (nichtatomar) BCOVPUIPlayerView *playerView;

Erstellen Sie die BCOVPUIPlayerView -Instanz und speichern Sie eine Referenz auf das Objekt.

 BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:nil options:nil controlView:controlView]; // Füge den PlayerView in deine eigene Videoansicht ein. [self.videoContainer addSubview:self.playerView];

Du’Sie müssen das Layout für die Player-Ansicht einrichten, Sie können dies mit Auto Layout oder der älteren Springs & Struts-Methode tun.

Federn & Streben

Stellen Sie den Rahmen so ein, dass er Ihrer Containeransicht entspricht, und fügen Sie dann die Playeransicht zur Containeransicht in Ihrer Ansichtshierarchie hinzu. Beachten Sie, dass VideoContainer ist Ihr eigenes Ansichtsobjekt in Ihrer App’s-Layout.

 self.playerView.frame = self.videoContainer.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth;

Automatisches Layout

self.playerView.translatesAutoresizingMaskIntoConstraints = NEIN; [NSLayoutConstraintactivateConstraints:@[ [self.playerView.topAnchor ConstraintEqualToAnchor:self.videoContainer.topAnchor], [self.playerView.rightAnchor ConstraintEqualToAnchor:self.videoContainer.rightAnchor], [self.playerView.leftAnchor ConstraintEqualToAnchor.self.Conchor] , [self.playerView.bottomAnchor ConstraintEqualToAnchor:self.videoContainer.bottomAnchor], ]];

Erstellen Sie nun die BCOVPlaybackController , weisen Sie es Ihrer Player-Ansicht zu und starten Sie dann die Wiedergabe von Videos.

// Companion-Slots initialisieren BCOVOUXAdComponentDisplayContainer *displayContainer = [[BCOVOUXAdComponentDisplayContainer alloc] initWithCompanionSlots:@[]]; // Erstellen Sie den Playback-Controller BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController>playbackController = [sdkManager createOUXPlaybackControllerWithViewStrategy:nil]; // Auf Display-/Companion-Anzeigennachrichten achten [playbackController addSessionConsumer:displayContainer]; // Sagen Sie der Player-Ansicht, dass dies der Playback-Controller ist, den wir verwenden self.playerView.playbackController =playbackController; // Video erstellen und abspielen BCOVVideo *video = [BCOVVideo videoWithURL:[NSURL URLWithString:<url-to-once-ux-video>]]; [playbackController setVideos:@[video]]; [WiedergabeController-Wiedergabe];

Weitere Informationen zur Verwendung und Anpassung der PlayerUI-Steuerelemente finden Sie in der README-Datei im BrightcovePlayerSDK.

Zugriff auf VMAP-Antwortdaten

Wenn Sie Zugriff auf die VMAP-Antwortdaten haben möchten, können Sie das . abonnieren kBCOVOUXVMAPResponseReceivedNotification Benachrichtigung. Nach Erhalt wird die Benachrichtigung’s userInfo-Wörterbuch enthält die VMAP-Antwort als NSData. Du kannst den ... benutzen kBCOVOUXVMAPResponseReceivedNotificationDataUserInfoKey konstant, um über userInfo darauf zuzugreifen. Da Sie möglicherweise mehrere Playback-Controller und damit mehrere VMAP-Antworten haben, können Sie die Benachrichtigung überprüfen’s Objekt, ID-WiedergabeController = notification.object um zu überprüfen, für welches Video die VMAP-Daten bestimmt sind. Wenn Sie die Benachrichtigung abonnieren, können Sie außerdem einen Wiedergabe-Controller als Objekt festlegen, sodass nur VMAP-Datenbenachrichtigungen zu diesem Wiedergabe-Controller empfangen werden. Zum Beispiel:

[[NSNotificationCenter defaultCenter] addObserver:self selector:@selector(vmapResponseReceived:) name:kBCOVOUXVMAPResponseReceivedNotification object:self.playbackController];

Bekannte Probleme

  • Sie können BrightcoveOUX nicht mit anderen Brightcove-Plug-Ins verwenden, mit Ausnahme des BrightcoveFairPlay-Plug-ins.

  • Das Plugin unterstützt derzeit nicht die Wiedergabe von OnceUX-Inhalten und Nicht-OnceUX-Inhalten im selben Wiedergabe-Controller. Wenn Sie beides abspielen müssen, müssen Sie einen separaten Wiedergabe-Controller für diesen Inhalt erstellen.

  • Beim Abspielen von OnceUX-Inhalten wird möglicherweise eine Meldung im Protokoll angezeigt, die besagt: “Keine Quelle mit DeliveryMethod von HLS oder MP4 gefunden, Rücksprung auf die erste Quelle.”. Diese Nachricht kann ignoriert werden und wird in einem zukünftigen Update behandelt.

  • Wenn die Anforderung des OnceUX VMAP-Dokuments nicht geladen werden kann, wird möglicherweise zusätzlich zu einem Fehlerereignis eine Meldung im Protokoll angezeigt, die besagt: “Cannot -forwardInvocation: zu einer nicht vorhandenen inneren Wiedergabesitzung.”. Diese Nachricht kann ignoriert werden und wird in einem zukünftigen Update behandelt.
