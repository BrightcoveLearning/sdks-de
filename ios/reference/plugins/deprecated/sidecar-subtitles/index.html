BCOVSidecarSubtitles Reference

Sidecar-Untertitel-Plug-in für Brightcove Player SDK für iOS und tvOS, Version 2.1.6.129

Unterstützte Plattformen

iOS 8.0 und höher.

tvOS 9.0 und höher.

Installation

Das Sidecar-Untertitel-Plug-in für Brightcove Player SDK bietet zwei Installationspakete für iOS; ein statisches Bibliotheks-Framework und ein dynamisches Framework. Für tvOS bietet das Sidecar Subtitles-Plugin nur ein dynamisches Framework.

Kakaoschoten

Sie können verwenden Kakaoschoten um das Sidecar Subtitles Plugin für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-SDK-SidecarSubtitles Podspec Hier.

Angeben des Standard-Pods Brightcove-Player-SDK-SidecarSubtitles installiert das statische Bibliotheksframework. Um das dynamische Framework zu installieren, deklarieren Sie den Pod mit dem dynamisch Unterspezifikation: Brightcove-Player-SDK-SidecarUntertitel/dynamisch

Beispiel für ein statisches Framework:

Pod 'Brightcove-Player-SDK-SidecarSubtitles'

Beispiel für ein dynamisches Framework:

Pod 'Brightcove-Player-SDK-SidecarUntertitel/dynamisch'

Die Pflege eines aktuellen Master-Podspec-Repositorys ist erforderlich, um sicherzustellen, dass Sie immer die neuesten Versionen der Brightcove-Software verwenden. Ab CocoaPods 1.0.0 sind Podspec-Repository-Updates keine automatische Funktion mehr. Um Ihr Master-Repository zu aktualisieren, führen Sie Folgendes auf der Befehlszeile aus:

Pod-Repository-Update

Handbuch

So fügen Sie Ihrem Projekt das Sidecar-Untertitel-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des BrightcoveSidecarSubtitles-Plugins von unserem . herunter Release-Seite.
  3. Hinzufügen BrightcoveSidecarSubtitles.framework zu Ihrem Projekt.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. (Nur dynamisches Framework) Auf dem “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu ‘BrightcoveSidecarSubtitles.framework’ zum “Eingebettete Binärdateien” Sektion.
  6. (Nur dynamisches Framework) Auf dem “Build-Phasen” Registerkarte, fügen Sie ein “Skript ausführen” Phase mit dem Befehl Schlag ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcoveSidecarSubtitles.framework/strip-frameworks.sh. Prüfen “Skript nur bei der Installation ausführen”. Dadurch werden nicht benötigte Architekturen aus dem Build entfernt, was für die Einreichung im App Store wichtig ist.
  7. (nur statisches Framework) Auf dem “Build-Einstellungen” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu -ObjC zum “Andere Linker-Flags” Einstellung bauen.

Importe

Das Brightcove Player SDK für iOS kann auf verschiedene Arten in Code importiert werden:

  • @import BrightcoveSidecarSubtitles;
  • #import <BrightcoveSidecarSubtitles/BrightcoveSidecarSubtitles.h>
  • #import <BrightcoveSidecarSubtitles/[spezifische Klasse].h>.

Schnellstart

 NSString *policyKey = <Ihr-Richtlinien-Schlüssel>; NSString *accountId = <Ihre-Konto-ID>; NSString *videoID = <Ihre-Video-ID>; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; [1] id<BCOVPlaybackController> controller = [playbackManager createSidecarSubtitlesPlaybackControllerWithViewStrategy:nil]; [self.view addSubview:controller.view]; BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithAccountId:accoundId policyKey:policyKey]; [2] [playbackService findVideoWithVideoID:videoID Parameter:nil Vervollständigung:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; }];

BCOVSidecarSubtitles fügt dem BCOVPlaybackManager einige Kategoriemethoden hinzu. Die erste davon ist -createSidecarSubtitlesPlaybackControllerWithViewStrategie:. Verwenden Sie diese Methode, um Ihren Wiedergabe-Controller zu erstellen. Alternativ (wenn Sie mehr als einen Sitzungsanbieter verwenden) können Sie einen BCOVSSSessionProvider erstellen und diesen an die Manager-Methode übergeben, die einen Wiedergabe-Controller mit Upstream-Sitzungsanbietern erstellt.* Wenn Sie für tvOS entwickeln, muss die an createSidecarSubtitlesPlaybackControllerWithViewStrategy übergebene ViewStrategy . sein Null.

* Beachten Sie, dass BCOVSSSessionProvider sollte vor allen Sitzungsanbietern in der Kette stehen, die beim Erstellen des Wiedergabe-Controllers an den Manager übergeben werden. Dieses Plugin ist nicht kompatibel mit dem Widevine-Plugin.

Wenn Sie Fragen haben oder Hilfe benötigen, besuchen Sie das Supportforum für Brightcove Native Player SDKs unter https://groups.google.com/forum/#!forum/brightcove-native-player-sdks .

Verwenden der integrierten PlayerUI-Steuerelemente

Das obige Code-Snippet stellt einen Videoplayer ohne Steuerelemente dar. Sie können Ihrem Code auf diese Weise Wiedergabesteuerelemente hinzufügen.

Fügen Sie eine Eigenschaft hinzu, um den Überblick zu behalten BCOVPUIPlayerView:

// Player-View von PlayerUI @property (nonatomic) BCOVPUIPlayerView *playerView;

Erstellen Sie die BCOVPUIBasicControlView , und dann die BCOVPUIPlayerView. Hier verbinden wir den Playback-Controller (und damit alle abgespielten Videos) mit den Steuerelementen. Stellen Sie die Player-Ansicht so ein, dass sie dem Video-Container aus Ihrem Layout entspricht (deinVideoView ) bei der Größenänderung.

BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:controller options:nil controlView:controlView]; // Größe der übergeordneten Ansicht anpassen self.playerView.frame = self.yourVideoView.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth; // BCOVPUIPlayerView zu Ihrer Videoansicht hinzufügen. [self.yourVideoView addSubview:self.playerView];

Wenn Sie den BCOVPUIPlayerView verwenden, müssen Sie auch eine Zeile darüber entfernen:

[self.view addSubview:controller.view]; // wird bei Verwendung von PlayerUI nicht mehr benötigt

Wenn Sie den Player mit einem anderen Playback-Controller wiederverwenden möchten, können Sie einfach eine neue Zuweisung vornehmen:

self.playerView.playbackController = otherPlaybackController;

Die Player-Ansicht fügt automatisch den Wiedergabe-Controller hinzu’s-Ansicht in eine eigene Ansichtshierarchie.

Bitte lesen Sie das Brightcove Native Player SDK’s README für weitere Informationen zum Hinzufügen und Anpassen von PlayerUI-Steuerelementen in Ihrer App.

Wenn Sie Fragen haben oder Hilfe benötigen, besuchen Sie die Brightcove Native Player SDK-Supportforum.

Untertiteldaten manuell ausfüllen

Das BCOVSidecarSubtitle-Plugin sucht nach einem Array von Untertitel-Metadaten in der BCOVVideo Objekteigenschaften, verschlüsselt von kBCOVSSVideoPropertiesKeyTextTracks. Wenn Sie verwenden BCOVPlaybackService um Videos abzurufen und diesen Videos sind Textspuren zugeordnet, dies wird automatisch ausgefüllt.

Wenn Sie Ihre eigenen Videos bereitstellen oder ein Perform-Kunde sind, müssen Sie die Daten wie folgt strukturieren:

NSArray *Untertitel = @[ @{ kBCOVSSTextTracksKeySource: ..., // required kBCOVSSTextTracksKeySourceLanguage: ..., // required kBCOVSSTextTracksKeyLabel: ..., // required kBCOVSSTextTracksKeyDuration: ..., // required/optional [1] kBCOVSSTextTracksKeyKind: kBCOVSSTextTracksKindSubtitles or kBCOVSSTextTracksKindCaptions, // required kBCOVSSTextTracksKeyDefault: ..., // optional kBCOVSSTextTracksKeyMIMEType: ..., // optional } , @{...} , // Wörterbuch der zweiten Textspur @{...} , // Wörterbuch der dritten Textspur ]; BCOVVideo *video = [BCOVVideo alloc] initWithSource:<source> cuePoints:<cuepoints> properties:@{ kBCOVSSVideoPropertiesKeyTextTracks:subtitles }];

Die kBCOVSSTextTracksKeySource key enthält die Quell-URL Ihres Untertiteltracks und kann entweder als WebVTT-URL oder als M3U8-Wiedergabelisten-URL bereitgestellt werden.

WebVTT-Dateien sollten a . haben “.vtt” Erweiterung und M3U8-Dateien sollten eine “.M3U8” Verlängerung. Wenn Sie diese Konventionen nicht einhalten können, müssen Sie ein kBCOVSSTextTracksKeySourceType Schlüssel, und geben Sie entweder kBCOVSSTextTracksKeySourceTypeWebVTTURL oder kBCOVSSTextTracksKeySourceTypeM3U8URL um den Dateityp anzugeben, auf den die URL verweist.

Wenn Sie einem aus Video Cloud abgerufenen Video Spuren bereitstellen, sollten Sie hinzufügen Ihre Untertitel zu allen vorhandenen Tracks anstatt überschreiben Sie. Dieser Code zeigt, wie Sie einem vorhandenen Video Tracks hinzufügen können:

BCOVVideo *updatedVideo = [Video-Update:^(id<BCOVMutableVideo> mutableVideo) { // Save the current tracks NSArray *originalTracks = video.properties[kBCOVSSVideoPropertiesKeyTextTracks]; // Create your text track dictionary NSArray *subtitles = @[ @{ kBCOVSSTextTracksKeySource: ..., // required kBCOVSSTextTracksKeySourceLanguage: ..., // required kBCOVSSTextTracksKeyLabel: ..., // required kBCOVSSTextTracksKeyDuration: ..., // required/optional [1] kBCOVSSTextTracksKeyKind: kBCOVSSTextTracksKindSubtitles or kBCOVSSTextTracksKindCaptions, // required kBCOVSSTextTracksKeyDefault: ..., // optional kBCOVSSTextTracksKeyMIMEType: ..., // optional } , @{...} , // Wörterbuch der zweiten Textspur @{...} , // Wörterbuch der dritten Textspur ]; // Neue Spuren an die Originalspuren anhängen, falls vorhanden NSArray *combinedTextTracks = ((originalTracks != nil) ? [originalTracks arrayByAddingObjectsFromArray:subtitles] : subtitles); // Aktualisieren Sie das aktuelle Wörterbuch (wir möchten die bereits enthaltenen Eigenschaften nicht verlieren) NSMutableDictionary *updatedDictionary = [mutableVideo.properties mutableCopy]; // Textspuren in der Eigenschaft Textspuren speichern updatedDictionary[kBCOVSSVideoPropertiesKeyTextTracks] = CombinedTextTracks; mutableVideo.properties = aktualisiertes Wörterbuch; }];

Anmerkungen

  • kBCOVSSTextTracksKeyDuration ist ein erforderlicher Schlüssel, wenn Sie Untertiteldateien mit der Erweiterung .vtt verwenden. kBCOVSSTextTracksKeyDuration ist ein optionaler Schlüssel, wenn Sie Untertiteldateien mit der Erweiterung .m3u8 verwenden.

Weitere Informationen zur Verwendung dieser Schlüssel finden Sie in der Codedokumentation in der Headerdatei BCOVSSComponent.h.

Bekannte Probleme

  • Untertitel werden beim Betrachten von 360-Grad-Videos nicht angezeigt.

  • Dieses Plug-in unterstützt derzeit nicht die Integration mit dem Widevine-Plug-in für Brightcove Brightcove Player SDK für iOS.

  • Beim Abrufen Ihrer Videos über die Brightcove-Wiedergabe-API müssen Ihre Wiedergaben eine Master-M3U8-Wiedergabeliste enthalten. Das Sidecar-Untertitel-Plugin funktioniert nicht mit M3U8-Wiedergabelisten mit einer einzigen Wiedergabe.

  • Wenn Sie dem Plugin eine Untertitel-Wiedergabeliste bereitstellen und diese Untertitel-Wiedergabeliste Links zu Web-VTT-Dateien enthält, die als 404 antworten, schlägt die Wiedergabe fehl. Dies ist ein Fehler bei Apple’s AVPlayer.
