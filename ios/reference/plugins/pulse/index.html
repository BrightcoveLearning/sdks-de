Installation

Pulse Plugin für Brightcove Player SDK bietet ein dynamisches Bibliotheksframework für die Installation.

Das Pulse-Plugin unterstützt INVIDI Technologies Pulse SDK Version 2.5.20.1.0 für iOS und Version 2.5.19.9.0 für tvOS, Pulse iOS- und tvOS-SDK-Referenz. Die Podspec für das Pulse-Plug-in für Brightcove Player SDK-Referenzen Pulse SDK-Versionsverlauf.

Kakaoschoten

Sie können verwenden Kakaoschoten um das Pulse-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-Puls Podspec Hier. Das PulseSDK muss zu Ihrem Projekt hinzugefügt werden, laden Sie die neueste Version herunter Hier. CocoaPods 1.0 oder neuer ist erforderlich.

Beispiel für eine CocoaPod-Poddatei:

Quelle 'https://github.com/CocoaPods/Specs' Quelle 'https://github.com/brightcove/BrightcoveSpecs.git' Plattform: ios, '11.0' use_frameworks! Ziel "Meine App"' do Pod 'Brightcove-Player-Pulse' Ende

Handbuch

So fügen Sie Ihrem Projekt das Pulse-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Laden Sie das neueste gezippte Brightcove Player SDK-Framework von der Veröffentlichungsseite.
  2. Laden Sie das neueste gezippte Pulse-Plug-in für das Brightcove Player SDK-Framework von der Veröffentlichungsseite.
  3. Laden Sie die . herunter PulseSDK Rahmen.
  4. Auf der “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie die dynamisch Framework, BrightcovePlayerSDK.framework, aus dem Brightcove Player SDK-Download zur Liste der Frameworks, Bibliotheken und eingebettete Inhalte. Das dynamische Framework BrightcovePlayerSDK.framework befindet sich im Verzeichnis ios/dynamic des Downloads. Die Einbetten Einstellung für den Rahmen muss “Einbetten & Signieren.”
  5. Auf der “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie BrightcovePulse.framework aus dem Pulse-Plug-in für den Brightcove Player SDK-Download zur Liste der Frameworks, Bibliotheken und eingebettete Inhalte. Die Einbetten Einstellung für den Rahmen muss “Einbetten & Signieren.”
  6. Auf der “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie PulseSDK.framework aus dem INVIDI Technologies-Download zur Liste der Frameworks, Bibliotheken und eingebettete Inhalte. Die Einbetten Einstellung für den Rahmen muss “Einbetten & Signieren.”
  7. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” die Pfade zu den Frameworks einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  8. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels:
    • Sicher gehen, dass -ObjC wurde hinzugefügt “Andere Linker-Flags” Einstellung bauen.

Importe

Das Pulse-Plug-in für Brightcove Player SDK kann auf verschiedene Weise in Code importiert werden. @import BrightcovePulse; , #import <BrightcovePulse/BrightcovePulse.h> oder #import <BrightcovePulse/[spezifische Klasse].h>. Sie können die importieren PulseSDK und BrightcovePlayerSDK Module in ähnlicher Weise.

Schnellstart

Das BrightcovePulse-Plugin ist eine Brücke zwischen PulseSDK und der Brightcove Player-SDK für iOS. Dieses Snippet zeigt seine grundlegende Verwendung mit serverseitigen Anzeigenregeln.

[1] OOContentMetadata *contentMetadata = [OOContentMetadata neu]; contentMetadata.category = <Kategorie>; contentMetadata.tags = @[ <tag1>, <tag2>, ..., <tagN> ]; contentMetadata.flags = @[ <flag1>, <flag2>, ..., <flagN> ]; OORequestSettings *requestSettings = [OORequestSettings neu]; requestSettings.linearPlaybackPositions = @[ @(position1), @(position2), ..., @(positionN) ]; NSString *pulseHost = <pulse domain>; [2] NSDictionary *pulseOptions = @{ kBCOVPulseOptionPulsePlaybackSessionDelegateKey: self, kBCOVPulseOptionPulsePersistentIdKey: [NSUUID.UUID UUIDString] } ; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController> controller = [3] [manager createPulsePlaybackControllerWithPulseHost:pulseHost contentMetadata:contentMetadata requestSettings:requestSettings adContainer:self.playerView.contentOverlayView CompanionSlots:nil viewStrategy:nil options:pulseOptions]; controller.delegate = selbst; [videoContainerView addSubview:controller.view]; NSString *policyKey = <Ihr-Richtlinien-Schlüssel>; NSString *accountId = <Ihre-Konto-ID>; NSString *videoID = <Ihre-Video-ID>; BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithAccountId:accountID policyKey:policyKey]; [playbackService findVideoWithVideoID:videoID Parameter:nil Vervollständigung:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; }];

Den Code in Schritte zerlegen:

  1. Erstelle das gleiche OOContentMetadaten , OORequestSettings die Sie erstellen würden, wenn Sie INVIDI Technologies verwenden würden’s Pulse iOS SDK direkt. Das Pulse-Plug-in erfordert eine gültige Domain und einen UIView-Container, um die Anzeigen anzuzeigen.
  2. Optional kann das Plugin ein NSDictionary mit Eigenschaften erhalten, die im Plugin verwendet werden sollen. Die verfügbaren Schlüssel sind kBCOVPulseOptionPulsePlaybackSessionDelegateKey , kBCOVPulseOptionPulseDeviceContainerKey und kBCOVPulseOptionPulsePersistentIdKey. Um die Pulse Session zu überschreiben, muss das PulsePlaybackSessionDelegate implementiert werden, wobei die kBCOVPulseOptionPulsePlaybackSessionDelegateKey mit der Klasse, die die Methode implementiert.
  3. BrightcovePulse fügt BCOVPlaybackManager einige Kategoriemethoden hinzu. Die erste davon ist -createPulsePlaybackControllerWithPulseHost:contentMetadata:requestSettings:adContainer:companionSlots:viewStrategy:Optionen:. Verwenden Sie diese Methode, um Ihren Wiedergabe-Controller zu erstellen.

Spielen und pausieren

Das Brightcove Pulse-Plug-in implementiert eine benutzerdefinierte Wiedergabe- und Pausenlogik, um ein möglichst reibungsloses Anzeigenerlebnis zu gewährleisten. Daher müssen Sie sicherstellen, dass Sie die Wiedergabemethode auf dem BCOVPlaybackController oder der [BCOVSessionProviderExtension pulse_play] oder [BCOVSessionProviderExtension pulse_pause] (BCOVSessionProviderErweiterung und nicht der AVPlayer.

Als Beispiel: Aufruf von Play zum ersten Mal auf BCOVPlaybackController ermöglicht BCOVPulse, Preroll-Anzeigen zu verarbeiten, ohne dass der Inhalt vor dem Preroll abgespielt wird. Weitere Informationen dazu, wie BCOVPulse den Standard überschreibt BCOVPlaybackController Methoden, bitte überprüfen Sie aus BCOVSessionProviderErweiterung.

Verwenden der integrierten PlayerUI

In deiner UIViewController , ein ... kreieren BCOVPUIPlayerView -Eigenschaft namens Player-Ansicht, die die Wiedergabesteuerelemente, die Videoinhaltsansicht und eine spezielle Ansicht enthält, in der Pulse seine Anzeigen anzeigen kann.

// Player-View von PlayerUI @property (nonatomic) BCOVPUIPlayerView *playerView;

Erstellen Sie dann Ihre Spieleransicht; liefern einen Null-Wiedergabe-Controller, der später hinzugefügt wird. Diese Playeransicht enthält sowohl die Videoinhaltsansicht als auch die Ansicht, die Wiedergabesteuerelemente und Anzeigensteuerelemente anzeigt. Dieses Setup ist gleich, egal welches Plugin Sie verwenden. Richten Sie die Player-Ansicht so ein, dass sie dem Video-Container aus Ihrem Layout entspricht (videoView ) bei der Größenänderung.

// Kontrollansicht erstellen und konfigurieren. BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; // Erstellen Sie die Player-Ansicht mit einem Null-Wiedergabe-Controller. self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:nil options:nil controlView:controlView]; // BCOVPUIPlayerView zu Ihrer Videoansicht hinzufügen. [self.videoView addSubview:self.playerView];

Du’Sie müssen das Layout für die Player-Ansicht einrichten, Sie können dies mit Auto Layout oder der älteren Springs & Struts-Methode tun.

Federn & Streben self.playerView.frame = self.videoView.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth;

Automatisches Layout self.playerView.translatesAutoresizingMaskIntoConstraints = NEIN; [NSLayoutConstraint activateConstraints:@[ [self.playerView.topAnchor ConstraintEqualToAnchor:self.videoView.topAnchor], [self.playerView.rightAnchor ConstraintEqualToAnchor:self.videoView.rightAnchor], [self.playerView.leftAnchor ConstraintEqualToAnchor:self.videoView.leftAnchor] , [self.playerView.bottomAnchor-EinschränkungEqualToAnchor:self.videoView.bottomAnchor], ]];

Das Erstellen des Playback-Controllers ist spezifisch für Pulse. Erstellen Sie Ihren Wiedergabe-Controller wie oben, aber übergeben Sie anstelle Ihrer Video-Container-Ansicht die contentOverlayView aus der Spieleransicht als dein AnzeigenContainer. Die contentOverlayView ist eine spezielle Ansicht, die zum Überlagern von Ansichten über den Hauptvideoinhalt verwendet wird. Sie sollten auch verwenden Null Anstatt von [Manager defaultControlsViewStrategy] wenn du das als dein benutzt hättest AnsichtStrategie (Dies war die ältere Methode zur Verwendung integrierter Steuerelemente).

// Erstellen Sie den Wiedergabecontroller. id<BCOVPlaybackController> controller = [Manager createPulsePlaybackControllerWithPulseHost:pulseHost contentMetadata:contentMetadata requestSettings:requestSettings adContainer:self.playerView.contentOverlayView CompanionSlots:nil viewStrategy:nil options:nil]; controller.delegate = selbst; // Weisen Sie der Player-Ansicht einen neuen Playback-Controller zu. // Dies verknüpft die Sitzung des PlayerControllers mit der PlayerUI. // Sie können diese Player-Ansicht beibehalten und ihm neue // Wiedergabe-Controller zuweisen, während sie erstellt werden. self.playerView.playbackController = self.playbackController;

Wenn Sie jetzt Videos mit Anzeigen abspielen, sehen Sie beim Abspielen von Videoinhalten die PlayerUI-Steuerelemente sowie Anzeigenmarkierungen auf dem Timeline-Scrubber.

Die PlayerUI ist hochgradig anpassbar. Weitere Informationen und Beispielcode finden Sie unter Benutzerdefinierte Layouts Abschnitt in der Datei README.md der Brightcove Native Player SDK-Repository.

Anpassen des Plugin-Verhaltens

Es gibt einige Konfigurationspunkte in BCOVPulse. Sie können BCOVPulse mit einem anderen Plug-in für das Brightcove Player SDK für iOS kombinieren, eine benutzerdefinierte Ansichtsstrategie erstellen und die aktuelle Anzeigenanfrage überschreiben.

BCOVPulsePlaybackSessionDelegate

Die BCOVPulsePlaybackSessionDelegate Protokoll bietet eine Möglichkeit für das BrightcovePulse-Plug-in, mit der Hostanwendung zu kommunizieren. Mit diesem Delegaten können Sie die Inhaltsmetadaten überschreiben und Einstellungen anfordern, bevor die Sitzung beginnt.

Die UIViewController muss das übernehmen BCOVPulsePlaybackSessionDelegate Protokoll.

@interface ViewController () <BCOVPulsePlaybackSessionDelegate>

Erstellen Sie ein NSDictionary, das die kBCOVPulseOptionPulsePlaybackSessionDelegateKey mit der Klasse, die die Methode zum Überschreiben der Sitzung implementiert.

NSDictionary *pulseOptions = @{ kBCOVPulseOptionPulsePlaybackSessionDelegateKey: self } ;

Ein ... kreieren BCOVPlaybackSessionProvider oder BCOVPlaybackController für Pulse mit den Methoden in BCOVPlaybackManager , denken Sie daran, das im vorherigen Schritt erstellte Wörterbuch zu übergeben. In diesem Beispiel wird ein Wiedergabecontroller verwendet.

BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController> controller = [manager createPulsePlaybackControllerWithPulseHost:pulseHost contentMetadata:[OOContentMetadata new] requestSettings:[OORequestSettings new] adContainer:self.playerView.contentOverlayView CompanionSlots:nil viewStrategy:nil options:pulseOptions];

Die createSessionForVideo:withPulseHost:contentMetadata:requestSettings: -Methode liefert das aktuelle Video, die Host-Domain, Inhaltsmetadaten und Einstellungen anfordern für die Sitzung. In diesem Beispiel waren die vorherigen Objekte leer und werden mit einem neuen überschrieben Kategorie und lineare Wiedergabepositionen Array.

- (id<OOPulseSession>)createSessionForVideo:(BCOVVideo *)video withPulseHost:(NSString *)pulseHost contentMetadata:(OOContentMetadata *)contentMetadata requestSettings:(OORequestSettings *)requestSettings { // Override the content metadata. contentMetadata.category = @"new_category_for_midrolls"; // Override the request settings. requestSettings.linearPlaybackPositions = @[ @(15) ]; return [OOPulse sessionWithContentMetadata:contentMetadata requestSettings:requestSettings]; }

Strategie anzeigen

Sie können dem BCOVPlaybackManager eine benutzerdefinierte Ansichtsstrategie bereitstellen, wenn Sie Ihren Wiedergabecontroller oder Sitzungsanbieter erstellen, anstatt die defaultControlsViewStrategy direkt anzugeben. Mit einer benutzerdefinierten Ansichtsstrategie können die Anzeigen-Container-Ansicht und die Anzeigen-Companion-Slots mit der Video-Content-Ansicht verknüpft werden. Dies ist ein Beispiel für eine benutzerdefinierte Ansichtsstrategie.

BCOVPlaybackControllerViewStrategy customViewStrategy = ^UIView* (UIView *view, id<BCOVPlaybackController>playbackController){ BCOVPlaybackControllerViewStrategy defaultControlsViewStrategy = [playbackManager defaultControlsViewStrategy]; UIView *contentAndDefaultControlsView = defaultControlsViewStrategy(view, playbackController); [<UIView of video container> addSubview:contentAndDefaultControlsView]; return <UIView of video container>; } ;

Sitzungsanbieter erstellen

Wenn Sie mehr als ein Plug-in für das Brightcove Player SDK für iOS verwenden, das einen benutzerdefinierten Wiedergabe-Controller erstellen muss, müssen Sie stattdessen eine Kette von Sitzungsanbietern zusammenstellen und den endgültigen Sitzungsanbieter an den [BCOVPlayerSDKManager createPlaybackControllerWithSessionProvider:viewStrategy:] Methode.

Beim Erstellen von Sitzungsanbietern kann das Vorladen der Sitzung aktiviert werden von BCOVBasicSessionProvider.

Häufig gestellte Fragen

Ich kann die Werbung hören, aber ich kann’Ich sehe sie nicht spielen.

Dies geschieht normalerweise, wenn sich die Anzeigencontaineransicht nicht in der Ansichtshierarchie befindet oder wenn die Anzeigenansicht (die eine Unteransicht der Anzeigencontaineransicht ist) von anderen Ansichten verdeckt wird.
