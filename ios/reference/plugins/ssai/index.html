Installation

Das SSAI-Plug-in für Brightcove Player SDK für iOS bietet zwei Installationspakete; ein statisches Bibliotheks-Framework und ein dynamisches Bibliotheks-Framework.

Kakaoschoten

Sie können verwenden Kakaoschoten um das SSAI-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-SSAI Podspec Hier.

Beispiel für ein dynamisches Framework:

```Bash-Quelle ‘https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks! Plattform: ios, ‘11,0’

Ziel ‘MyVideoPlayer’ mach pod ‘Brightcove-Player-SSAI’ Ende ```

Beispiel für einen statischen Rahmen:

```Bash-Quelle ‘https://github.com/brightcove/BrightcoveSpecs.git’

use_frameworks! Plattform: ios, 11.0'

Ziel ‘MyVideoPlayer’ mach pod ‘Brightcove-Player-SSAI-statisch’ Ende ```

Wenn Sie Ihre Installation aktualisieren,’Es ist eine gute Idee, die lokale Kopie Ihres BrightcoveSpecs-Repositorys zu aktualisieren, damit Sie lokal über die neuesten Podspecs verfügen, genau wie Sie Ihr CococaPods-Master-Repository aktualisieren würden.

Normalerweise, wenn Sie verwenden Pod-Update im Terminal geschieht dies automatisch, oder Sie können alternativ explizit mit aktualisieren Pod-Repo-Update Brightcove. (Ihr BrightcoveSpecs-Repository kann einen anderen Namen haben, wenn Sie es explizit zu Ihrer Liste der Podspecs-Repositorys hinzugefügt haben.)

Handbuch

So fügen Sie Ihrem Projekt das SSAI-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des BrightcoveSSAI-Plugins von unserer Release-Seite.
  3. Hinzufügen BrightcoveSSAI.framework zu Ihrem Projekt.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. (Nur dynamisches Framework) Auf dem “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu ‘BrightcoveSSAI.framework’ zum “Frameworks, Bibliotheken, eingebettete Inhalte” Sektion.
  6. (Nur dynamisches Framework) Auf dem “Build-Phasen” Registerkarte, fügen Sie ein “Skript ausführen” Phase mit dem Befehl Schlag ${BUILT_PRODUCTS_DIR}/${FRAMEWORKS_FOLDER_PATH}/BrightcoveSSAI.framework/strip-frameworks.sh. Prüfen “Skript nur bei der Installation ausführen”. Dadurch werden nicht benötigte Architekturen aus dem Build entfernt, was für die Einreichung im App Store wichtig ist.
  7. (nur statisches Framework) Auf dem “Build-Einstellungen” Registerkarte Ihres Anwendungsziels, fügen Sie hinzu -ObjC zum “Andere Linker-Flags” Einstellung bauen.

Importe

Das SSAI-Plug-in für Brightcove Player SDK für iOS kann auf verschiedene Weise in Code importiert werden. @import BrightcoveSSAI; , #import <BrightcoveSSAI/BrightcoveSSAI.h> oder #import <BrightcoveSSAI/[spezifische Klasse].h>.

BrightcoveSSAI ist ein Plugin für Brightcove Player-SDK für iOS die Unterstützung für die serverseitige Anzeigeneinfügung von Brightcove bietet.

Geben Sie ggf. die vorhandenen Companion-Slots ein. 1. BrightcoveSSAI fügt dem `BCOVPlaybackManager` einige Kategoriemethoden hinzu. Wenn Sie eine VMAP-URL im Unicorn Once-Stil verwenden, erstellen Sie das BCOVVideo-Objekt direkt mit `[BCOVVideo videoWithURL:<unicorn-style-url>]`. 1. Laden Sie das Video in den Wiedergabecontroller. Abrufen von Informationen zur Anzeigenwiedergabe

BrightcoveSSAI bietet Informationen zur Anzeigenwiedergabe über das BCOVPlaybackControllerAdsDelegate. Wenn Sie beispielsweise Ihre Steuerelemente während einer Anzeige ausblenden möchten, können Sie Folgendes implementieren: -[BCOVPlaybackControllerAdsDelegieren von PlaybackController:playbackSession:didEnterAdSequence:] sie zu verstecken.

Weitere Informationen zur Verwendung dieser Delegatmethoden finden Sie unter Brightcove Player-SDK für iOS.

Ich suche

Das BrightcoveSSAI-Plugin bietet eine Suchfunktion, die beim Implementieren von Steuerelementen verwendet werden sollte. Diese Suchfunktion wird in der Sitzung durch die AnbieterErweiterung Eigentum. So wird es verwendet:

```Objekt
CMTime contentTimeToSeekTo = <Berechnung-aus-Scrub-Leiste>; [self.currentSession.providerExtension ssai_seekToTime:contentTimeToSeekTo completeHandler:^(BOOL beendet) {

wenn (fertig) { [self.currentSession.player play]; }

}]; ```

Die FertigstellungHandler wird nach Abschluss einer erfolgreichen Suche ausgeführt. Es wird nicht ausgeführt, wenn eine Suche bereits durch einen vorherigen Aufruf an . eingeleitet wurde [BCOVSessionProviderExtension ssai_seekToTime:completionHandler:] oder wenn eine Anzeige wiedergegeben wird. Um zu testen, ob ein Suchversuch möglich ist, überprüfen Sie das [BCOVSessionProviderExtension ssai_canSeek] Eigentum. Weitere Informationen zu diesen beiden Methoden finden Sie im headerdoc.

Suche ohne Werbung

Das BrightcovePlayerSDK bietet die WerbungDeaktiviert BOOL-Eigenschaft zum Deaktivieren von Anzeigen während der Suche, sodass eine Anwendung die Wiedergabe fortsetzen kann, ohne dass der Endbenutzer zuvor wiedergegebene Anzeigen anzeigen muss.

In Vorbereitung auf die Suche deaktivieren automatisches Abspielen beim Einrichten der BCOVPlaybackController.

Die Logik zum Deaktivieren von Anzeigen sollte hinzugefügt werden kBCOVPlaybackSessionLifecycleEventReady Handler der -playbackController:playbackSession:didReceiveLifecycleEvent: Methode von dir BCOVPlaybackController delegieren.

```objc - (void)playbackController:(id .))Controller-WiedergabeSession:(id)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVPlaybackSessionLifecycleEventReady isEqualToString:lifecycleEvent.eventType]) { // Anzeigen deaktivieren. _playbackController.adsDisabled = JA;

// irgendwo im Videoinhalt suchen. [session.providerExtension ssai_seekToTime:resumeTime CompletionHandler:^(BOOL beendet) { // re-enable ads. _playbackController.adsDisabled = NO; // open the shutter. _playbackController.shutterFadeTime = 0.25; _playbackController.shutter = NO; }];

} ```

Beim Anrufen ssai_seekToTime:completionHandler: Um die Wiedergabe zu einem bestimmten Zeitpunkt fortzusetzen, kann der erste Frame des Videos sichtbar sein, bis die Suche abgeschlossen ist. Für eine sauberere Darstellung decken Sie die Videoansicht während der Suche vorübergehend ab, indem Sie die Verschluss Eigenschaft von BCOVPlabackController to JAWOHL bevor du anrufst -setVideos:. Wenn die Suche abgeschlossen ist, schließen Sie den Verschluss, indem Sie die Verschluss Eigentum an NEIN. Die ShutterFadeTime -Eigenschaft definiert die Dauer der Blendenanimation.

```objc self.playbackController = [sdkManager createSSAIPlaybackController];

// Aktivieren Sie den Verschluss, bevor Sie das Video laden. self.playbackController.shutterFadeTime = 0.0; self.playbackController.shutter = JA;

// Laden Sie das Video. ```

Beachten Sie, dass aus Leistungsgründen kleine Toleranzen in die Videosuche eingebaut sind. Eine Suche nach einer Content-Abspielkopfposition, die sich nahe am Anfang einer Anzeigensequenz befindet, kann zu einer Suche an den Anfang der Anzeigensequenz führen. Wenn autoPlay in diesem Fall NEIN ist, wird AVPlayer beim ersten Frame der Anzeigensequenz angehalten.

Integrierte Bedienelemente der Player-Benutzeroberfläche

Das BrightcovePlayerSDK bietet einen integrierten Satz von UI-Steuerelementen, die mit dem SSAI-Plug-in sowohl für grundlegende Wiedergabe- als auch für Anzeigensteuerelemente verwendet werden können. Um die Steuerelemente zu verwenden, erstellen Sie a BCOVPUIPlayerView , und verknüpfen Sie es mit Ihrem SSAI-Wiedergabe-Controller.

Erstellen Sie zunächst eine playerView-Eigenschaft in Ihrer Klasse.

objc @property (nichtatomar) BCOVPUIPlayerView *playerView;

Erstellen Sie die BCOVPUIPlayerView -Instanz und speichern Sie eine Referenz auf das Objekt.

objc BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:nil options:nil controlView:controlView]; // Füge den PlayerView in deine eigene Videoansicht ein. [self.videoContainer addSubview:self.playerView];

Du’Sie müssen das Layout für die Player-Ansicht einrichten, Sie können dies mit Auto Layout oder der älteren Springs & Struts-Methode tun.

objc self.playerView.frame = self.videoContainer.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth;

Zugriff auf VMAP-Antwortdaten ========================== Wenn Sie auf die VMAP-Antwortdaten zugreifen möchten, können Sie die Benachrichtigung `kBCOVSSAIVMAPResponseReceivedNotification` abonnieren. Nach dem Empfang enthält das userInfo-Wörterbuch der Benachrichtigung die VMAP-Antwort als NSData. Sie können die Konstante `kBCOVSSAIVMAPResponseReceivedNotificationDataUserInfoKey` verwenden, um von userInfo darauf zuzugreifen. Da Sie möglicherweise mehrere Playback-Controller und damit mehrere VMAP-Antworten haben, können Sie das Benachrichtigungsobjekt `idplaybackController = notification.object` überprüfen, um zu überprüfen, für welches Video die VMAP-Daten bestimmt sind. Wenn Sie die Benachrichtigung abonnieren, können Sie außerdem einen Wiedergabe-Controller als Objekt festlegen, sodass nur VMAP-Datenbenachrichtigungen zu diesem Wiedergabe-Controller empfangen werden. Bekannte Probleme

  • Da tvOS das Surfen im Internet nicht unterstützt, werden Companion Ads, Learn More und alle Anzeigenklicks auf dieser Plattform ignoriert.

  • Sie können BrightcoveSSAI nicht mit anderen Brightcove-Plug-ins verwenden, mit Ausnahme des BrightcoveFairPlay-Plug-ins oder des BrightcoveIMA-Plug-ins (nur für Pre-Roll-Anzeigen).

  • Wenn Sie das BrightcoveSSAI-Plugin zusammen mit dem BrightcoveFairPlay-Plugin verwenden möchten, muss das BrightcoveFairPlay-Plugin das UpstreamSessionProvider des SSAI-Sitzungsanbieters.

  • Das SSAI-Plugin unterstützt nicht das Abspielen von Nicht-SSAI-Inhalten. Wenn Sie Nicht-SSAI-Inhalte abspielen müssen, müssen Sie einen anderen Wiedergabe-Controller erstellen, der das SSAI-Plugin nicht verwendet.
