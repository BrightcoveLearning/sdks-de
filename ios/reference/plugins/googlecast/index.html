iOS 11.0 und höher.

Installation

Das Google Cast-Plug-in für Brightcove Player SDK bietet ein statisches Bibliotheksframework für die Installation.

Das Google Cast-Plug-in unterstützt Version 4.5.0 des Google Cast SDK für iOS. Die Versionshinweise zu Google Cast finden Sie Hier.

Kakaoschoten

Sie können verwenden Kakaoschoten um das Google Cast-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Hier finden Sie die neuesten Brightcove-Player-GoogleCast-Podspecs Hier. Der Pod integriert automatisch die richtige Version von GoogleCast. CocoaPods 1.0 oder neuer ist erforderlich.

Beispiel für eine CocoaPod-Poddatei:

```Bash-Quelle ‘https://github.com/CocoaPods/Specs' Quelle ’https://github.com/brightcove/BrightcoveSpecs.git'

Plattform: ios, ‘11,0’

use_frameworks!

Ziel ‘BeispielApp’ mach pod ‘Brightcove-Player-GoogleCast’ Ende ```

Bei der Installation von Brightcove-Player-GoogleCast wird automatisch die Abhängigkeit Brightcove-Player-Core installiert. Je nach Bedarf können Sie entweder das dynamische Brightcove-Player-Core-Framework oder das statische Framework einschließen. Standardmäßig ist das dynamische Core-Framework installiert. Um das statische Core-Framework auszuwählen, würde Ihr Podfile etwa so aussehen:

```Bash-Quelle ‘https://github.com/CocoaPods/Specs’ Quelle ‘https://github.com/brightcove/BrightcoveSpecs.git’

Plattform: ios, ‘11,0’

use_frameworks!

Ziel ‘BeispielApp’ mach pod ‘Brightcove-Player-GoogleCast-static’ Ende ```

Wenn Sie Ihre Installation aktualisieren,’Es ist eine gute Idee, die lokale Kopie Ihres BrightcoveSpecs-Repositorys zu aktualisieren, um sicherzustellen, dass Sie lokal über die neuesten Podspecs verfügen, genau wie Sie Ihr CococaPods-Master-Repository aktualisieren würden. Verwenden Pod-Repository-Update dies zu tun.

Handbuch

So fügen Sie Ihrem Projekt das Google Cast-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Folge dem Manuelle Einrichtung des Google Cast SDK Handbuch.
  2. Folge dem Manuelle Installation des Brightcove Player SDK Handbuch.
  3. Laden Sie die . herunter Google Cast-Plug-in für Brightcove Player SDK Rahmen.
  4. Auf der “Allgemein” Registerkarte Ihres Anwendungsziels, fügen Sie BrightcoveGoogleCast.framework aus dem Google Cast-Plug-in für den Brightcove Player SDK-Download zur Liste der . hinzu Frameworks, Bibliotheken, eingebettete Inhalte.
  5. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” die Pfade zu den Frameworks einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.

Importe

Das Google Cast-Plug-in für Brightcove Player SDK kann auf verschiedene Weise in Code importiert werden. @import BrightcoveGoogleCast; , #import <BrightcoveGoogleCast/BrightcoveGoogleCast.h> oder #import <BrightcoveGoogleCast/[spezifische Klasse].h>. Sie können die importieren GoogleCast und BrightcovePlayerSDK Module in ähnlicher Weise.

Bevor Sie beginnen

Bevor Sie versuchen, dieses Plugin zu verwenden, sollten Sie bereits mit Folgendem vertraut sein:

  • Registrieren Sie Ihre Bewerbung. Dies führt Sie durch die Erstellung Ihrer Empfängeranwendung und weist Ihnen eine Anwendungs-ID zu. Zu Beginn können Sie die Demo-Anwendungs-ID verwenden ‘4F8B3483’.
  • Setup für die Entwicklung mit dem Cast Application Framework (CAF) für iOS . Dies führt Sie durch die Einrichtung der erforderlichen Berechtigungen für Ihre App.
  • Den Besetzungskontext initialisieren. Dies führt Sie durch die Initialisierung des Besetzungskontexts mit Ihrer Empfängeranwendungs-ID.
  • Mini-Controller hinzufügen. Hier erfahren Sie, wie Sie das Google Cast SDK nutzen können’s Mini-Mediensteuerung.

Schnellstart

Das BrightcoveGoogleCast-Plug-in ist eine Brücke zwischen Google Cast iOS-SDK und der Brightcove Player-SDK für iOS.

Dieser Ausschnitt zeigt seine grundlegende Verwendung.

[1] self.googleCastManager = [BCOVGoogleCastManager neu]; [2] id<BCOVPlaybackController>playbackController = [BCOVPlayerSDKManager.sharedManager createPlaybackController]; [3] [playbackController addSessionConsumer:self.googleCastManager];

Den Code in Schritte zerlegen:

  1. Erstellen Sie einen Verweis auf die BCOVGoogleCastManager Singleton.
  2. Erstellen Sie eine zu verwendende Instanz von BCOVPlaybackController.
  3. Fügen Sie Ihre BCOVGoogleCastManager als Sitzungskonsumenten auf Ihre BCOVPlaybackController-Instanz verweisen.

Brightcove CAF-Empfänger

Die Anwendungs-ID für den Brightcove CAF-Empfänger lautet 341387A3 und wird der Konstanten kBCOVCAFReceiverApplicationID. Sie können die Anwendungs-ID überprüfen, indem Sie das CAF-Empfänger config.json.

Wenn Sie den Brightcove CAF-Empfänger verwenden,’Ich muss diese Variablen wie folgt initialisieren:

[1] BCOVReceiverAppConfig *receiverAppConfig = [BCOVReceiverAppConfig neu]; empfängerAppConfig.accountId = kAccountID; receiverAppConfig.policyKey = kServicePolicyKey; [2] self.googleCastManager = [[BCOVGoogleCastManager alloc] initForBrightcoveReceiverApp:receiverAppConfig];

Die folgenden Eigenschaften können auch aktiviert werden BCOVReceiverAppConfig wie benötigt:

  • splashScreen (zum Anpassen des Begrüßungsbildschirmbilds)
  • playerUrl (zur Verwendung eines benutzerdefinierten Players)
  • authToken (zur Verwendung mit PAS/EPA)
  • watermarkingToken (zur Verwendung mit Forensic Watermaking)
  • adConfigId (zur Verwendung mit SSAI)
  • userId (zur Verwendung mit Analytics-Tracking)
  • applicationId (zur Verwendung mit Analytics-Tracking)

Methoden delegieren

BCOVGoogleCastManagerDelegierter verfügt über Delegiertenmethoden, mit denen Sie beispielsweise benachrichtigt werden können, wenn wichtige Besetzungsereignisse aufgetreten sind oder bevorstehen

  • - (void)switchedToLocalPlayback:(NSTimeInterval)lastKnownStreamPosition;
  • - (void)switchedToRemotePlayback;
  • - (void)currentCastedVideoDidComplete;
  • - (void)castedVideoFailedToPlay;
  • - (ungültig)geeignetSourceNotFound;
  • - (void)willBuildMediaInformationBuilder:(GCKMediaInformationBuilder *_Nonnull);
  • - (void)willSendMediaLoadOptions:(GCKMediaLoadOptions *_Nonnull);

Eine vollständige Liste mit Beschreibungen finden Sie im BCOVGoogeCastManagerDelegate-Protokollreferenz.

Um von diesen Ereignissen zu profitieren, legen Sie einen Delegierten auf dem . fest BCOVGoogleCastManager Singleton.

 self.googleCastManager.delegate = selbst;

Quellenauswahl

Die BCOVGoogleCastManager wird versuchen, eine geeignete Quelle zu finden. Es sucht nach Quellen in dieser Reihenfolge:

  1. HTTPS-HLS v3
  2. HTTPS-DASH
  3. HTTPS MP4
  4. HTTP-HLS v3
  5. HTTP-DASH
  6. HTTP-MP4

Wenn keines davon gefunden wird, wird die Delegate-Methode geeigneteQuelleNotFound wird angerufen werden.

Ihr Video Cloud-Konto muss für die Unterstützung von HLS v3 eingerichtet sein, sicherstellen, dass DASH aktiviert ist oder für jedes Video eine MP4-Quelle verfügbar ist.

Anpassungen

Neben der Delegate-Eigenschaft gibt es zwei Eigenschaften, die Sie auf der Seite festlegen können BCOVGoogleCastManager Klasse. Diese sind:

  • GCKImage *fallbackPosterImage: Das GCKImage, das verwendet wird, wenn für ein Video kein Posterbild verfügbar ist.
  • CGGröße posterImageSize: Die Höhe und Breite, die Sie für das erstellte GCKImage-Objektbild verwenden möchten. Standardmäßig 480h x 720w.

Bekannte Probleme / Einschränkungen

Bei Verwendung eines standardmäßigen, nicht modifizierten Empfängers (einschließlich des Demo-Empfängers) gelten die folgenden Einschränkungen:

  • DRM wird nicht unterstützt.
  • Mehrere Audiospuren werden nicht unterstützt.
  • Clientseitige und serverseitige Werbung werden nicht unterstützt.
  • Live- und Live-DVR-Streams werden nicht unterstützt.

Bei Verwendung des Brightcove CAF-Empfängers gelten die folgenden Einschränkungen:

  • Clientseitige Werbung wird nicht unterstützt.
