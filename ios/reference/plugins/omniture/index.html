Omniture (Adobe Marketing Cloud – Adobe Analytics) Plugin für Brightcove Player SDK für iOS, Version 6.9.0.1697

Anforderungen

  • Xcode 11.0+
  • BOGEN

Unterstützte Plattformen

iOS 11.0 und höher.

Installation

Das Omniture-Plug-in ist ein statisches Bibliotheks-Framework, das Adobe Marketing Cloud v4.14.1 und Adobe Video Heartbeat v2.0.1 unterstützt. Die Marketing Cloud- und Video Heartbeat-Bibliotheken sind nicht in diesem SDK enthalten und muss Ihrem Projekt manuell hinzugefügt werden. Anweisungen zum Herunterladen der Bibliotheken finden Sie unten.

Kakaoschoten

Sie können verwenden Kakaoschoten Version 1.0 oder höher, um das Omniture-Plugin zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-Omniture Podspec Hier. Um diese Spezifikation zu verwenden, fügen Sie am Anfang Ihrer Poddatei Folgendes hinzu: Quelle 'https://github.com/brightcove/BCOVSpecs.git'.

Beispiel für ein statisches Framework:

```Bash-Quelle ‘https://github.com/brightcove/BrightcoveSpecs.git'

Ziel ‘MyOmniturePlayer’ mach pod ‘Brightcove-Player-Omniture’ Ende ```

Wenn Sie Ihre Installation aktualisieren,’Es ist eine gute Idee, die lokale Kopie Ihres BrightcoveSpecs-Repositorys zu aktualisieren, damit Sie lokal über die neuesten Podspecs verfügen, genau wie Sie Ihr CococaPods-Master-Repository aktualisieren würden.

Normalerweise, wenn Sie verwenden Pod-Update im Terminal geschieht dies automatisch, oder Sie können alternativ explizit mit aktualisieren Pod-Repository-Update.

Handbuch

So fügen Sie das Omniture-Plug-in für Brightcove Player SDK manuell zu Ihrem Projekt hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK.
  2. Laden Sie die neueste gezippte Version des Plugins von der . herunter Release-Seite.
  3. Ergänzen Sie die BrightcoveAMC.framework Projekt. Sie können dies tun, indem Sie mit der rechten Maustaste auf den Ordner Frameworks klicken und auswählen “Dateien hinzufügen zu” Option und wählen Sie die BrightcoveAMC.framework aus dem Pfad, in dem es gespeichert ist.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. Auf der “Build-Phasen” Registerkarte Ihres Anwendungsziels, fügen Sie Folgendes zum “Binärdateien mit Bibliotheken verknüpfen” Phase:
    • BrightcoveAMC.framework
  6. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels:
    • Sicher gehen, dass -ObjC wurde hinzugefügt “Andere Linker-Flags” Einstellung bauen.
  7. Laden Sie Adobe Marketing Cloud nach Adobe . herunter’S Anweisungen und füge hinzu AdobeMobileLibrary.a zu Ihrem Projekt.
  8. Laden Sie Adobe Video Heartbeat nach Adobe herunter’S Anweisungen und füge hinzu VideoHeartbeat.a zu Ihrem Projekt.
  9. Fügen Sie die Header-Ordner Marketing Cloud und Video Heartbeat zu den Header-Suchpfad-Einstellungen Ihres Projekts hinzu.

Schnellstart

BrightcoveAMC ist eine Brücke zwischen den Brightcove Player-SDK für iOS und Adobe-Video-Tracking, einschließlich Adobe Video Heartbeat v2 und Adobe Media-Tracking v4. BrightcoveAMC verfolgt grundlegendes Videoverhalten: Video bereit, Video abgeschlossen, Videowiedergabe und Videopause.

Kundenimplementierung

Um Adobe Video Heartbeat einzurichten, müssen Omniture-Plug-in-Clients Instanzen von BCOVAMCVideoHeartbeatConfigurationPolicy implementieren. Zum Einrichten von Adobe Media Tracking ist eine Instanz von BCOVAMCMediaSettingPolicy erforderlich. Diese Richtlinien ermöglichen die Anpassung von ADBMediaHeartbeatConfig- oder ADBMediaSettings-Objekten für jede Wiedergabesitzung.

Video Heartbeat v2

In diesem Beispiel wird Video-Heartbeat-Tracking verwendet.

[1] // Erstellen Sie das VHB-Konfigurationsrichtlinienobjekt. BCOVAMCVideoHeartbeatConfigurationPolicy videoHeartbeatConfigurationPolicy = ^ADBMediaHeartbeatConfig *(id<BCOVPlaybackSession> Sitzung) { ADBMediaHeartbeatConfig *configData = [[ADBMediaHeartbeatConfig alloc] init]; configData.trackingServer = <adobe-assigned_tracking_server>"; configData.channel = <cutomize_sample_channel>; configData.appVersion = <app_version>; configData.ovp = <online_video_platform>; configData.playerName = <player_name>; configData.ssl = <YES | NO>; // Set debugLogging to true to activate debug tracing. Remove it in production. configData.debugLogging = YES; return configData; } ; [2] // Erstellen Sie das Brightcove AMC-Analyserichtlinienobjekt aus w1w des VHB-Konfigurationsrichtlinienobjekts. BCOVAMCAnalyticsPolicy *heartbeatPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithHeartbeatConfigurationPolicy:videoHeartbeatConfigPolicy]; BCOVAMCSessionConsumer *sessionConsumer = [BCOVAMCSessionConsumer heartbeatAnalyticsConsumerWithPolicy:heartbeatPolicy Delegat:self]; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController> Controller = [Manager createPlaybackController]; controller.delegate = selbst; [self.view addSubview:controller.view]; [3] // Fügen Sie dem Wiedergabe-Controller den Brightcove AMC-Sitzungskonsumenten hinzu. [Controller addSessionConsumer:sessionConsumer]; [4] // Ein Video suchen und abspielen. NSString *policyKey = <Ihr-Richtlinien-Schlüssel>; NSString *accountId = <Ihre-Konto-ID>; NSString *videoID = <Ihre-Video-ID>; BCOVPlaybackService *service = [[BCOVPlaybackService alloc] initWithAccountId:accountId policyKey:policyKey]; [service findVideoWithVideoID:videoID Parameter:nil Vervollständigung:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; }];
  1. Erstellen Sie den Richtlinienblock für die Video-Heartbeat-Konfiguration, der zu Beginn jeder Wiedergabesitzung aufgerufen wird. Die Richtlinie ermöglicht die Anpassung von Konfigurationsdaten basierend auf der aktuellen Sitzung.
  2. Verwenden Sie den Konfigurationsrichtlinienblock, um eine BCOVAMCAnalyticsPolicy-Instanz mit . zu erstellen und zu initialisieren -[initWithHeartbeatConfigurationPolicy:]. Das BCOVAMCAnalyticsPolicy-Objekt wird verwendet, um den BCOVAMCSessionConsumer zu erstellen.
  3. Nachdem der Playback-Controller erstellt wurde, rufen Sie die Consumer-Methode add session auf, -[addSessionConsumer:] , um den AMC-Sitzungskonsumenten hinzuzufügen.

Marketing-Cloud

In diesem Beispiel wird Medienverfolgung verwendet.

[1] BCOVAMCMediaSettingPolicy mediaSettingPolicy = ^ADBMediaSettings *(id<BCOVPlaybackSession> Sitzung) { ADBMediaSettings *settings = [ADBMobile mediaCreateSettingsWithName:<cutomize_setting_name> [2] length:0 playerName:<cutomize_player_name> playerID:<cutomize_player_ID>]; [3] settings.milestones = @"25,50,75"; // a customization. return settings; } ; [4] BBCOVAMCAnalyticsPolicy *mediaPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithMediaSettingsPolicy:mediaSettingPolicy]; BCOVAMCSessionConsumer *sessionConsumer = [BCOVAMCSessionConsumer mediaAnalyticsConsumerWithPolicy:mediaPolicy Delegat:self]; BCOVPlayerSDKManager *manager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController> Controller = [Manager createPlaybackController]; controller.delegate = selbst; [self.view addSubview:controller.view]; [5] [Controller addSessionConsumer:sessionConsumer]; NSString *policyKey = <Ihr-Richtlinien-Schlüssel>; NSString *accountId = <Ihre-Konto-ID>; NSString *videoID = <Ihre-Video-ID>; BCOVPlaybackService *service = [[BCOVPlaybackService alloc] initWithAccountId:accountId policyKey:policyKey]; [service findVideoWithVideoID:videoID Parameter:nil Vervollständigung:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) { [controller setVideos:@[ video ]]; [controller play]; }];
  1. Erstellen Sie den Richtlinienblock für die Medieneinstellungen, der zu Beginn jeder Wiedergabesitzung aufgerufen wird. Die Richtlinie ermöglicht die Anpassung der Medieneinstellungen basierend auf der aktuellen Sitzung.
  2. Beim Erstellen der ADBMediaSettings-Instanz mit einer Klassenmethode von ADBMobile +[mediaCreateSettingsWithName:length:playerName:playerID:] , kannst du einstellen Videolänge zu 0. Das Omniture-Plugin wird es später aktualisieren.
  3. Fügen Sie optionale Einstellungen wie Meilensteine hinzu.
  4. Verwenden Sie den Richtlinienblock für Medieneinstellungen, um eine BCOVAMCAnalyticsPolicy-Instanz mit . zu erstellen und zu initialisieren -[initWithMediaSettingsPolicy:]. Das Richtlinienobjekt wird verwendet, um einen AMC-Sitzungskonsumenten für die Adobe-Medienverfolgung zu erstellen, als +[mediaAnalyticsConsumerWithPolicy:delegieren:].
  5. Nachdem der Playback-Controller erstellt wurde, rufen Sie die Consumer-Methode add session auf, -[addSessionConsumer:] , um den AMC-Sitzungskonsumenten hinzuzufügen.
