Überblick

Optionale Konfiguration für FW-Sitzungsanbieter.

  waitForAVPlayerDauer

Ob gewartet werden soll, bis das AVPlayer-Element eine Dauer meldet, bevor die adContextPolicy aufgerufen wird. Wenn NEIN eingestellt ist, wird die Dauer in die BCOVFWSessionProviderAdContextPolicy Block ist möglicherweise nicht genau. Standardmäßig ist Ja.

@property (nonatomic, Assign, getter=shouldWaitForAVPlayerDuration) BOOL waitForAVPlayerDuration

Diskussion

Ob gewartet werden soll, bis das AVPlayer-Element eine Dauer meldet, bevor die adContextPolicy aufgerufen wird. Wenn NEIN eingestellt ist, wird die Dauer in die BCOVFWSessionProviderAdContextPolicy Block ist möglicherweise nicht genau. Standardmäßig ist Ja.

  adTimeoutInterval

Die Anzahl der Sekunden, die der Sitzungsanbieter auf eine Anzeigenanfrage wartet, bevor eine Sitzung ohne Verwendung von Anzeigen bereitgestellt wird. Standardmäßig 3 Sekunden.

@property (nichtatomar, zuweisen) NSTimeInterval adTimeoutInterval

Diskussion

Die Anzahl der Sekunden, die der Sitzungsanbieter auf eine Anzeigenanfrage wartet, bevor eine Sitzung ohne Verwendung von Anzeigen bereitgestellt wird. Standardmäßig 3 Sekunden.

  cuePointProgressPolicy

Eine Richtlinie, die bestimmt, ob eine FreeWheel-Anzeige abgespielt werden soll, wenn ein bestimmtes Cue-Point-Ereignis auftritt, und wenn ja, wo die Wiedergabe fortgesetzt werden soll. Standardmäßig BCOVProgressPolicyProcessFinalCuePoint, BCOVProgressPolicyResumeFromContentPlayhead, YES.

@property (nichtatomar, stark) BCOVCuePointProgressPolicy *cuePointProgressPolicy

Diskussion

Eine Richtlinie, die bestimmt, ob eine FreeWheel-Anzeige abgespielt werden soll, wenn ein bestimmtes Cue-Point-Ereignis auftritt, und wenn ja, wo die Wiedergabe fortgesetzt werden soll. Standardmäßig BCOVProgressPolicyProcessFinalCuePoint, BCOVProgressPolicyResumeFromContentPlayhead, YES.

  PreloadSlots

Legt fest, ob Slots vorgeladen sind, damit sie bei einem Treffer spielbereit sind. Bei Einstellung auf YES werden die Preroll-Slots so schnell wie möglich geladen. Midrolls-Slots werden X Sekunden bevor der Slot gespielt werden soll geladen, bestimmt durch timeBeforeAdToPreload. Standardmäßig NEIN.

@property (nonatomic, Assign, getter=shouldPreloadSlots) BOOL preloadSlots

Diskussion

Legt fest, ob Slots vorgeladen sind, damit sie bei einem Treffer spielbereit sind. Bei Einstellung auf YES werden die Preroll-Slots so schnell wie möglich geladen. Midrolls-Slots werden X Sekunden bevor der Slot gespielt werden soll geladen, bestimmt durch timeBeforeAdToPreload. Standardmäßig NEIN.

  timeBeforeAdToPreload

Wenn das Vorabladen der Anzeigenfläche aktiviert ist, ist dies die Zeit in Sekunden, bevor die Anzeige vorab geladen wird. Standardmäßig 10 Sekunden.

@property (nichtatomar, zuweisen) NSTimeInterval timeBeforeAdToPreload

Diskussion
