Installation

Das FreeWheel-Plug-in für Brightcove Player SDK bietet ein statisches Bibliotheksframework für die Installation. Ein dynamisches Framework wird in Zukunft hinzugefügt, wenn FreeWheel eine Dylib-Version veröffentlicht.

Das FreeWheel-Plug-in für Brightcove Player SDK wurde in Version 6.38.0 des FreeWheel AdManager-Frameworks integriert, das über die Downloads zur AdManager SDK-Integration Seite.

Kakaoschoten

Sie können verwenden Kakaoschoten um das FreeWheel-Plug-in für Brightcove Player SDK zu Ihrem Projekt hinzuzufügen. Sie finden die neuesten Brightcove-Player-FreeWheel Podspec Hier. Um diese Spezifikation zu verwenden, fügen Sie oben in Podfile Folgendes hinzu: Quelle 'https://github.com/brightcove/BrightcoveSpecs.git'. CocoaPods-Version 1.0 oder höher ist erforderlich.

Das FreeWheel-SDK ist nicht in diesem Pod enthalten. Du muss fügen Sie das FreeWheel SDK AdManager.framework manuell zu Ihrem Projekt hinzu. Der Pod fügt jedoch alle Bibliotheken hinzu, die für das AdManager.framework-Framework erforderlich sind.

Beispiel für ein statisches Framework:

Quelle 'https://github.com/brightcove/BrightcoveSpecs.git' Pod 'Brightcove-Player-FreeWheel'

Wenn Sie Ihre Installation aktualisieren,’Es ist eine gute Idee, die lokale Kopie Ihres BrightcoveSpecs-Repositorys zu aktualisieren, damit Sie lokal über die neuesten Podspecs verfügen, genau wie Sie Ihr CococaPods-Master-Repository aktualisieren würden.

Normalerweise, wenn Sie verwenden Pod-Update im Terminal geschieht dies automatisch, oder Sie können alternativ explizit mit aktualisieren Pod-Repo-Update Brightcove. (Ihr BrightcoveSpecs-Repository kann einen anderen Namen haben, wenn Sie es explizit zu Ihrer Liste der Podspecs-Repositorys hinzugefügt haben.)

Handbuch

So fügen Sie Ihrem Projekt das FreeWheel-Plug-in für Brightcove Player SDK manuell hinzu:

  1. Installieren Sie die neueste Version des Brightcove Player-SDK und befolgen Sie die Anweisungen zur Installation des [Statischer Rahmen][https://github.com/brightcove/brightcove-player-sdk-ios/#ManuelleInstallation] .
  2. Laden Sie die neueste gezippte Version des Plugins von unserem . herunter Release-Seite.
  3. Ergänzen Sie die ‘BrightcoveFW.framework’ zu Ihrem Projekt. Sie können dies tun, indem Sie mit der rechten Maustaste auf den Ordner Frameworks klicken und auswählen “Dateien hinzufügen zu” Option und wählen Sie BrightcoveFW.framework aus dem Pfad, in dem es gespeichert ist.
  4. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels stellen Sie sicher, dass die “Framework-Suchpfade” den Pfad zum Framework einschließen. Dies sollte automatisch erfolgen, es sei denn, das Framework ist in einem anderen Stammverzeichnis als Ihr Projekt gespeichert.
  5. Auf der “Build-Phasen” Registerkarte Ihres Anwendungsziels, fügen Sie Folgendes zum “Binärdateien mit Bibliotheken verknüpfen” Phase:
    • BrightcoveFW.framework
  6. Auf der “Build-Einstellungen” Registerkarte Ihres Anwendungsziels:
    • Sicher gehen, dass -ObjC wurde hinzugefügt “Andere Linker-Flags” Einstellung bauen.
  7. Installieren Sie die FreeWheel-Bibliothek, die von Ihrem FreeWheel-Konto abgerufen werden muss.

Importe

Das FreeWheel-Plug-in für Brightcove Player SDK kann auf verschiedene Weise in Code importiert werden. @import BrightcoveFW; , #import <BrightcoveFW/BrightcoveFW.h> oder #import <BrightcoveFW/[spezifische Klasse].h>.

Der Code ist in Schritte unterteilt:

  1. Sie erstellen denselben Anzeigenmanager, den Sie erstellen würden, wenn Sie FreeWheel verwenden’s iOS SDK direkt; dies wird später benötigt.

Spielen und pausieren

Das Brightcove FreeWheel-Plug-in implementiert eine benutzerdefinierte Wiedergabe- und Pausenlogik, um ein möglichst reibungsloses Anzeigenerlebnis zu gewährleisten. Daher müssen Sie sicherstellen, dass Sie die Wiedergabemethode auf dem BCOVPlaybackController oder der [BCOVSessionProviderExtension fw_play] oder [BCOVSessionProviderExtension fw_pause] (BCOVSessionProviderErweiterung und nicht der AVPlayer.

Als Beispiel: Aufruf von Play zum ersten Mal auf BCOVPlaybackController ermöglicht es dem Brightcove FreeWheel-Plug-in, Preroll-Anzeigen zu verarbeiten, ohne dass der Inhalt vor dem Preroll abgespielt wird.

Verwenden der integrierten PlayerUI

Wenn Sie Version 5.1 oder höher des Brightcove Player SDK verwenden, können Sie die integrierten Anzeigensteuerelemente mit dem Brightcove FreeWheel-Plug-in nutzen.

Um die PlayerUI zu verwenden, erstellen Sie Ihren Playback-Controller wie im obigen Schnellstartcode.

BCOVPlayerSDKManager *sdkManager = [BCOVPlayerSDKManager sharedManager]; id<BCOVPlaybackController>playbackController = [sdkManager createFWPlaybackControllerWithAdContextPolicy:[self adContextPolicy] viewStrategy:nil];

Erstellen Sie dann Ihre Player-Ansicht. Diese Ansicht enthält sowohl die Videoplayer-Ansicht als auch die Ansicht, die Wiedergabe- und Anzeigensteuerelemente anzeigt. Dieses Setup ist gleich, egal welches Plugin Sie verwenden.

Notiz: Die BrightcovePlayerUI Modul wird nicht mehr benötigt und wurde entfernt. (Vor Version 5.1 des Brightcove Player SDK war das Brightcove PlayerUI-Plug-in ein separates Framework und Modul.) Sie können alle Importe entfernen, die auf das Brightcove PlayerUI-Modul verweisen. Alle PlayerUI-Header befinden sich jetzt im BrightcovePlayerSDK Modul.

Erstellen Sie eine Eigenschaft in Ihrem UIViewController, um die BCOVPUIPlayerView zu verfolgen. Der BCOVPUIPlayerView enthält sowohl den Wiedergabe-Controller’s-Ansicht und die Steuerelemente-Ansicht.

// Player-View von PlayerUI @property (nonatomic) BCOVPUIPlayerView *playerView;

Erstellen Sie BCOVPUIBasicControlView und dann BCOVPUIPlayerView. Hier verbinden wir den Playback-Controller (und damit alle abgespielten Videos) mit den Steuerelementen.

// Kontrollansicht erstellen und konfigurieren. BCOVPUIBasicControlView *controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlView:controlView]; // BCOVPUIPlayerView zu Ihrer Videoansicht hinzufügen. [self.videoView addSubview:self.playerView];

Du’Sie müssen das Layout für die Spieleransicht einrichten, Sie können dies mit Auto Layout oder dem älteren Springs & Struts-Ansatz tun.

Federn & Streben:

Stellen Sie die Player-Ansicht so ein, dass sie dem Video-Container aus Ihrem Layout entspricht (videoView ) bei der Größenänderung.

self.playerView.frame = self.videoView.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth;

Suche ohne Werbung

Verwenden -[BCOVPlaybackController seekWithoutAds:(CMTime)seekToTime completeHandler:(void (^)(BOOL beendet))completion] um die Wiedergabe zu einem bestimmten Zeitpunkt fortzusetzen, ohne den Benutzer zu zwingen, zuvor geplante Anzeigen anzusehen searchToTime.

In Vorbereitung für seekWithoutAds:completionHandler: , deaktivieren automatisches Abspielen beim Einrichten der BCOVPlaybackController.

Apple empfiehlt, zu warten, bis sich der Status eines AVPlayerItems in spielbereit ändert, bevor das AVPlayerItem verwendet wird. Rufen Sie daher an seekWithoutAds:completionHandler: in dem kBCOVPlaybackSessionLifecycleEventReady Handler der PlaybackController:playbackSession:didReceiveLifecycleEvent Methode von dir BCOVPlaybackControllerDelegate.

```objective-c - (void)playbackController:(NSObject)Controller-WiedergabeSession:(NSObject)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVPlaybackSessionLifecycleEventReady isEqualToString:lifecycleEvent.eventType]) {

// self.resumePlayback ist eine hypothetische Instanzvariable, die hier zur Veranschaulichung verwendet wird. if (self.resumePlayback) { __weak typeof(controller) weakController = controller; // seek without playing ads which are scheduled before the seek time, i.e. resume playback. [controller seekWithoutAds:CMTimeMake(seekWithoutAdsValue, seekWithoutAdsTimescale) completionHandler:^(BOOL finished) { if (!finished) { NSLog (@"seekWithoutAds failed to finish"); } typeof(Controller) starkerController = schwacherController; if (starker Controller) { // fade out the shutter to reveal the player view. strongController.shutterFadeTime = 0.25; strongController.shutter = NO; // turn off seek without ads - especially important if this player is being used with a playlist self.resumePlayback = NO; } }]; }

} } ```

Die Verschluss und ShutterFadeTime Eigenschaften der BCOVPlaybackController kann zusammen mit verwendet werden seekWithoutAds:completionHandler: um Frame-Flimmern auszublenden, das auftreten kann, wenn der AVPlayer Assets lädt. Aktivieren Sie in Ihrem BCOVPlaybackController-Setup-Code den Auslöser, um die Player-Ansicht auszublenden:

```Ziel-c NSObject *Wiedergabecontroller;

PlaybackController = [sdkManager createFWPlaybackControllerWithAdContextPolicy:nil viewStrategy:nil]; playController.delegate = self;

if (self.resumePlayback) { // Setze die Blenden-Überblendzeit auf Null, um die Player-Ansicht sofort auszublenden. playController.shutterFadeTime = 0.0; playController.shutter = JA;

// AutoPlay deaktivieren, wenn die Wiedergabe fortgesetzt wird. playController.autoPlay = NEIN;

} ```

Beachten Sie, dass Sie bei der Verwendung von Seek Without Ads in Ihrer App möglicherweise Netzwerkverkehr beobachten, der normalerweise bei der Einrichtung des IMA-Plug-ins auftritt. Dieser Datenverkehr ist für die ordnungsgemäße Einrichtung des Plugins erforderlich und hat keinen Einfluss auf die Funktion „Suchen ohne Werbung“.

Sie können das Standardverhalten des Plugins anpassen, indem Sie eine Instanz von . erstellen BCOVFWSessionProviderOptions und Überschreiben der Standardeigenschaften. Um a . zu verwenden BCOVFWSessionProviderOptions options-Instanz müssen Sie die BCOVFWSessionProvider mit [BCOVSDKManager createFWSessionProviderWithAdContextPolicy:upstreamSessionProvider:options:].

Vorladen von Slots

Abhängig von den Netzwerkbedingungen kann es zu einer Verzögerung bei der Wiedergabe von Anzeigen kommen, sobald die Anzeigenposition erreicht wurde. Um die Anzeigenleistung zu verbessern, ist es möglich, die FreeWheel-Slots vorab zu laden. Um das Vorladen zu aktivieren, aktivieren Sie PreloadSlots An BCOVFWSessionProviderOptions.

Wenn aktiviert, werden Pre-Roll-Slots so schnell wie möglich geladen. Für Mid-Rolls werden Slots eine konfigurierbare Anzahl von Sekunden vorgeladen, bestimmt durch timeBeforeAdToPreload An BCOVFWSessionProviderOptions.

Verwenden einer genauen Dauer

Abhängig von Ihrer FreeWheel-Konfiguration, Bereitstellung einer genauen Dauer bei Anrufen an setVideoId: ist der Unterschied zwischen dem Erhalten der richtigen Anzahl von Midrolls oder nicht.

Standardmäßig wartet das SDK mit dem Aufruf der BCOVFWSessionProviderAdContextPolicy bis AVPlayerItem eine genaue Dauer hat, und dieser Wert wird dem Blockaufruf bereitgestellt. Wenn Sie nicht warten möchten, bis die AVPlayerItem-Dauer fertig ist und Ihnen diese Informationen aus einer anderen Quelle zur Verfügung stehen (wie den BCOVVideo/BCOVSource-Eigenschaften oder Ihrem eigenen CMS), waitForAVPlayerDauer kann verwendet werden, um das Warten zu deaktivieren.

Umgang mit Suchvorgängen

Bei der Suche nach mehreren Anzeigen-Pods (z. B. zwei Mid-Roll-Slots an verschiedenen Positionen) bietet das SDK einen bequemen Mechanismus, um zu bestimmen, welche Pods abgespielt werden. Dies kann durch Ändern der Standardeinstellung geändert werden BCOVCuePointProgressPolicy An BCOVFWSessionProviderOptions. Für weitere Informationen zum BCOVCuePointProgressPolicy , bitte konsultieren Sie die BCOVFWSessionProvider.h Datei.

Nachverfolgung von FreeWheel-Fehlern

FreeWheel-Fehlerereignisse (FWAdErrorEvent ) gefunden über die FWAdEventNotification wird an die weitergereicht PlaybackController:playbackSession:didReceiveLifecycleEvent: Delegate-Methode mit einem Ereignistyp von kBCOVFWLifecycleEventAdError. Das gesamte userInfo-Objekt, das das FreeWheel SDK zusammen mit der Benachrichtigung sendet, ist in den Ereigniseigenschaften enthalten und kann mit dem kBCOVFWLifecycleEventPropertyKeyAdError Schlüssel.

Hier ist ein Beispiel, wie Sie diese Fehler abfangen können:

- (void)playbackController:(id<BCOVPlaybackController>)ControllerplaybackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent { if ([kBCOVFWLifecycleEventAdError isEqualToString:lifecycleEvent.eventType]) { NSDictionary *errorInfo = lifecycleEvent.properties[kBCOVFWLifecycleEventPropertyKeyAdError]; NSLog(@"FreeWheel Error Encountered: %@", errorInfo[FWInfoKeyErrorInfo]); } }
