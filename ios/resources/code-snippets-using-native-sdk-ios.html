    Benutzerdefinierte Steuerelemente für AirPlay

    Wenn Sie die Brightcove PlayerUI-Steuerelemente verwenden, sind Sie alle bereit. Die AirPlay-Funktionalität funktioniert sofort einsatzbereit und ermöglicht es Benutzern, Videos mit Apple TV auf hochauflösende Displays zu streamen.

    Wenn Sie benutzerdefinierte Steuerelemente verwenden, können Sie die folgenden Schritte ausführen:

    1. Erfahren Sie mehr über AirPlay in der Entwickler-Dokumentation von Apple für AirPlay.

    2. Verwenden Sie die allowsExternalPlayback Eigenschaft des Playback-Controllers, um die allowsExternalPlayback Eigenschaft des AVPlayer festzulegen:

      _playbackController.allowsExternalPlayback = YES;

    3. Richten Sie eine AirPlay-Router-Steuerung ein und übernehmen Sie die Auswahl. Einzelheiten finden Sie im AirPlay-Übersichtsdokument von Apple .

    Anpassen der Schaltfläche „Untertitel“

    Der BcovPlayerUI-Beispielcode zeigt Ihnen, wie Sie den Brightcove Player anpassen können, wenn Sie das Native SDK für iOS verwenden. Weitere Informationen finden Sie im Abschnitt PlayerUI-Steuerelemente anpassen des Referenzdokuments des Native SDK.

    Gehen Sie folgendermaßen vor, um die Schaltfläche für Untertitel mit PlayerUI anzupassen:

    1. Beginnen Sie mit der Basic Video Playback App.

    2. Ersetzen Sie Ihre Werte für den Richtlinienschlüssel, die Konto-ID und die Video-ID. Wählen Sie ein Video aus Ihrem Konto aus, das Textspuren enthält.

    3. Richten Sie die Spieleransicht mit einem Standard-VOD-Layout ein:

      // Set up our player view. Create with a standard VOD layout.
BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:
[BCOVPUIBasicControlView basicControlViewWithVODLayout] ];

    4. Der closedCaptionButton wird als a deklariert BCOVPUIButton , was eine Unterklasse von ist UIButton und drei zusätzliche Anpassungsmethoden hinzufügt. Wenn Sie die BcovPlayerUI-Steuerelemente anpassen, sollten Sie die Native Player-APIs überall dort verwenden, wo sie verfügbar sind. Ihr benutzerdefinierter Code sollte ähnlich wie folgt aussehen:

      // Customize the CC button.
BCOVPUIButton *ccButton = playerView.controlsView.closedCaptionButton;
ccButton.titleLabel.font = [UIFont systemFontOfSize:14.];
ccButton.primaryTitle = @"CC";
[ccButton showPrimaryTitle:YES];

    Anzeigen von FairPlay-Inhalten auf einem externen Bildschirm

    Wenn ein externes Display über einen AV-Adapter und ein HDMI-Kabel mit einem iOS-Gerät verbunden ist, besteht das Standardverhalten darin, den iOS-Bildschirm zu spiegeln. Die Ausnahme ist, wenn Sie FairPlay-geschütztes Video verwenden, das Apple vor dem Spiegeln verhindert (WWDC 2015, Session 502).

    Um FairPlay-geschützte Videos anzuzeigen, legen Sie die AVPlayer-Eigenschaften fest, die über den Brightcove Playback Controller bereitgestellt werden, damit FairPlay-Videos auf einem externen Display abgespielt werden können. Das Video wird im Vollbildmodus wiedergegeben. Hier ist ein Beispiel für das Festlegen dieser Eigenschaften:

    playbackController.allowsExternalPlayback = YES;
playbackController.usesExternalPlaybackWhileExternalScreenIsActive = YES;

    Google-Analytics

    Wenn Sie den Brightcove Player und die Katalogklasse verwenden, werden Videoanalysen automatisch erfasst und in Ihrem Video Cloud Analytics-Modul angezeigt. Für zusätzliche Metriken können Sie Ihrer App Google Analytics hinzufügen.

    Um Google Analytics in Ihre App zu integrieren, führen Sie die folgenden Schritte aus:

    1. Lesen Sie das Dokument von Google, um Analytics zu Ihrer iOS-App hinzuzufügen.

    2. Hier ist eine Möglichkeit, wie Sie Google Analytics verwenden können, um die Videowiedergabe mit dem Firebase SDK von Google zu verfolgen:

      // This snippet shows one way to track video playback
// using the Firebase SDK from Google Analytics with
// the lifecycle event playback controller delegate method.
- (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent
{
// Common parameters
NSString *video_name = session.video.properties[kBCOVVideoPropertyKeyName];
NSString *video_ID = session.video.properties[kBCOVVideoPropertyKeyId];

// Session is ready to play
if ([kBCOVPlaybackSessionLifecycleEventReady isEqualToString:lifecycleEvent.eventType])
{
	[FIRAnalytics logEventWithName:@"bcov_video_ready"
    parameters:@{
        @"bcov_video_name": video_name,
        @"bcov_video_id": video_ID
    }];
}

// Session encountered an error
if ([kBCOVPlaybackSessionLifecycleEventError isEqualToString:lifecycleEvent.eventType])
{
	NSError *error = lifecycleEvent.properties[kBCOVPlaybackSessionEventKeyError];
	int error_code = error.code;
	[FIRAnalytics logEventWithName:@"bcov_video_playback_error"
    parameters:@{
        @"bcov_video_name": video_name,
        @"bcov_video_id": video_ID,
        @"bcov_video_error_code": @(error_code)
    }];
}

// Session has completed
if ([kBCOVPlaybackSessionLifecycleEventTerminate isEqualToString:lifecycleEvent.eventType])
{
	[FIRAnalytics logEventWithName:@"bcov_video_terminate"
    parameters:@{
        @"bcov_video_name": video_name,
        @"bcov_video_id": video_ID
     }];
}
}

     

    Begrenzung der Bitrate

    Sie können nicht steuern, welche Quelle (Wiedergabeversion) im HLS-Manifest vom ausgewählt wird AVPlayer , aber Sie können bei der Wiedergabe eine Bitratenobergrenze setzen. Dies verhindert, dass der Spieler Quellen (Wiedergaben) mit einer Bitrate über der angegebenen Bitrate verwendet.

    Setzen Sie den preferredPeakBitRate auf das gewünschte Limit des Netzbandbreitenverbrauchs in Bit pro Sekunde für den gegebenen Wert AVPlayerItem .

    Verwenden Sie eine der folgenden Erklärungen:

    // Swift
var preferredPeakBitRate: Double
// Obj-C
@property(nonatomic) double preferredPeakBitRate

    Looping eines Videos

    In einigen Fällen möchten Sie möglicherweise, dass ein Video automatisch wiedergegeben wird. Um dies zu tun, können Sie das Lebenszyklusereignis „Ende des Videos“ abrufen, zum Anfang gehen und erneut spielen.

    Dieser Code geht davon aus, dass Sie den Delegaten des PlaybackControllers mit dieser Methode auf das Objekt gesetzt haben:

    - (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent
{
  if ([kBCOVPlaybackSessionLifecycleEventEnd isEqualToString:lifecycleEvent.eventType])
  {
    [controller seekToTime:kCMTimeZero completionHandler:^(BOOL finished) {
    if (finished)
    	{
        [controller play];
    	}
    }];
  }
}

    Verwalten von Videos in einer Playlist

    Eine Möglichkeit, eine Wiedergabeliste mit Videos zu verwalten, besteht darin, die Videoobjekte in einer Tabelle zu speichern. Wenn der Benutzer ein Video aus der Tabelle auswählt, enthält die Tabellenzeile das Videoobjekt.

    Hier ist eine Übersicht darüber, wie es funktioniert:

    1. Rufen Sie eine Playlist von Ihrem Konto ab.

      - (void)retrievePlaylist
{
[self.refreshControl beginRefreshing];

// Retrieve a playlist through the BCOVPlaybackService
BCOVPlaybackServiceRequestFactory *playbackServiceRequestFactory = [[BCOVPlaybackServiceRequestFactory alloc]         initWithAccountId:kDynamicDeliveryAccountID
		policyKey:kDynamicDeliveryPolicyKey];
BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithRequestFactory:playbackServiceRequestFactory];

[playbackService findPlaylistWithReferenceID:kDynamicDeliveryPlaylistRefID
		parameters:nil
		completion:^(BCOVPlaylist *playlist, NSDictionary *jsonResponse, NSError *error)
{
	 [self.refreshControl endRefreshing];

	 NSLog(@"JSON Response:\n%@", jsonResponse);

	 if (playlist)
	 {
      self.currentVideos = playlist.videos.mutableCopy;
      self.currentPlaylistTitle = playlist.properties[@"name"];
      self.currentPlaylistDescription = playlist.properties[@"description"];

      NSLog(@"Retrieved playlist containing %d videos", (int)self.currentVideos.count);

      [self usePlaylist:self.currentVideos];
	 }
	 else
	 {
      NSLog(@"No playlist for ID %@ was found.", kDynamicDeliveryPlaylistRefID);
	 }

}];
}

    2. Initialisieren Sie die Container erneut, die Informationen zu den Videos in der aktuellen Wiedergabeliste speichern.

    3. Wenn die Tabellensicht ausgewählt ist, wird der Index der Zeile verwendet, um eine neue zu erstellen videoDictionary. Fragen Sie als Nächstes das Wörterbuch nach dem Video. Wenn das Video nicht Null ist, laden Sie das Video in den playbackController.

      // Play the video in this row when selected
- (IBAction)tableView:(UITableView *)tableView didSelectRowAtIndexPath:(nonnull NSIndexPath *)indexPath
{
NSDictionary *videoDictionary = self.videosTableViewData[ (int)indexPath.row ];
BCOVVideo *video = videoDictionary[@"video"];

  if (video != nil)
  {
  	[self.playbackController setVideos:@[ video ]];
  }
}

    Ein vollständiges Codebeispiel finden Sie im Beispiel des Offline Players .

    Werte des Medienfortschritts

    Während der Medienwiedergabe können die Werte, die der Fortschrittsmethode des Player SDK gemeldet werden, einen Anfangswert von negativer Unendlichkeit und einen Endwert der positiven Unendlichkeit enthalten. Diese Werte werden bei der Verarbeitung von Pre-Roll- und Postroll-Anzeigen verwendet.

    Wenn diese Werte für Sie nicht wichtig sind oder Ihre eigene Fortschrittsverfolgung stören, können sie mit einer bedingten Aussage wie dieser leicht ignoriert werden:

    - (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didProgressTo:(NSTimeInterval)progress
{
if (progress < 0.0 || progress >= INFINITY)
return;
// Your code here
}

    Programmatisches Hinzufügen von Cue-Points

    Video Cloud-Kunden können einem Video mit Video Cloud Studio Cue-Points hinzufügen, wie im Dokument Cue-Points zu Videos hinzufügen gezeigt.

    Sie können Ihrem Video auch programmgesteuert Cue-Points hinzufügen. Der folgende Code fügt dem von der Playback-API zurückgegebenen Video vierteljährliche Intervall-Cue-Points hinzu:

    // programmatically add cue points to a video
- (void)requestContentFromPlaybackService
{
[self.service findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {
if (video)
{
// Get the video duration from the properties dictionary
NSNumber *durationNumber = video.properties[@"duration"]; // milliseconds
float duration = durationNumber.floatValue / 1000.0; // convert to seconds
video = [video update:^(id<BCOVMutableVideo> mutableVideo)
{
// Add quarterly interval cue points of your own type
BCOVCuePoint *cp1 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 250, 1000)];
BCOVCuePoint *cp2 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 500, 1000)];
BCOVCuePoint *cp3 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 750, 1000)];
BCOVCuePoint *cp4 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 1000, 1000)];
// Create new cue point collection using existing cue points and new cue points
NSMutableArray *newCuePoints = [[NSMutableArray alloc] initWithArray:mutableVideo.cuePoints.array];
[newCuePoints addObject:cp1];
[newCuePoints addObject:cp2];
[newCuePoints addObject:cp3];
[newCuePoints addObject:cp4];
mutableVideo.cuePoints = [[BCOVCuePointCollection alloc] initWithArray:newCuePoints];
}];
[self.playbackController setVideos:@[ video ]];
}
else
{
NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
}
}];
}

    Beachten Sie, dass der Wert von ein beliebiger String-Wert sein your cue point type kann, den Sie möchten, solange Sie keines der iOS-Pluginsverwenden. Einzelheiten finden Sie im Dokument BCovCuePoint Protocol Reference .

    Wenn Sie Cue-Points mit dem IMA-Plugin verwenden, erfahren Sie mehr darüber im Abschnitt VAST und vMap/Server Side Ad Rules des IMA-Plugins für das native SDK für iOS-Notizen. Die IMA-Beispielanwendung zeigt Ihnen den für IMA-Wert-Cue-Points erforderlichen Wert.

    Der folgende Code lauscht auf Ihre Cue-Points und zeigt eine Meldung an:

    // listen for cue points and display them
-(void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didPassCuePoints:(NSDictionary *)cuePointInfo {
BCOVCuePointCollection *cpc = cuePointInfo[@"kBCOVPlaybackSessionEventKeyCuePoints"];
for (BCOVCuePoint *cp in cpc) {
  if ([cp.type isEqualToString:@"your cue point type"])
    {
    NSLog(@"Found your cue point at %f", CMTimeGetSeconds(cp.position));
    }
  }
}

    Entfernen des Spielers

    Es kann Fälle geben, in denen Sie den Spieler und die Ansicht entfernen möchten.

    Wenn Sie den View-Controller freigeben, der Eigentümer eines ist, BCOVPlaybackController wird auch der Wiedergabecontroller freigegeben. Entfernen Sie dazu die Player-Ansicht aus der Superview und stellen Sie den Zeiger für den Wiedergabe-Controller auf nil.

    Hier ist ein Codebeispiel: 

    playerView?.removeFromSuperview()
playerView = nil
playbackController = nil

    Einstellen der Wiedergaberate

    Um die Wiedergaberate zu steuern, können Sie die rate Eigenschaft für die AVPlayer Klasse festlegen, die in der Sitzung verfügbar ist.

    Standardmäßig kann die Spielrate nur in regelmäßigen Intervallen (0,50, 0,67, 1,0, 1,25, 1,50 und 2,0) eingestellt werden. Durch Festlegen des AudioTimePitchAlgorithmus können Sie detailliertere Ratenwerte (wie 1,7) verwenden. Weitere Details finden Sie in dieser Stackoverflow-Diskussion.

    avPlayerItem.audioTimePitchAlgorithm = AVAudioTimePitchAlgorithmVarispeed;

    Obwohl in diesem Beispiel der varispeed Eigenschaftstyp verwendet wird, verfügt die AVAudioTimepitchAlgorithmus-Klasse über 4 Eigenschaftstypen, die Sie verwenden können:

    • NiedrigQualityZerolatenz
    • spektral
    • TimeDomäne
    • varispeed

    Beachten Sie, dass die varispeed Option keine Tonhöhenkorrektur bietet. Dies kann dazu führen, dass Audio mit einer hohen Wiedergaberate wie 1,7 unnatürlich hoch ist. Wählen Sie die Immobilienoption, die Ihren Anforderungen am besten entspricht.

    Für a BCOVPlaybackSession würde Ihr Code ähnlich aussehen wie folgt:

    - (void)playbackController:(id<BCOVPlaybackController>)controller
  playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent {

  if ( [lifecycleEvent.eventType isEqualToString:kBCOVPlaybackSessionLifecycleEventReady] ) {
    NSLog(@"%@ at time %f", kBCOVPlaybackSessionLifecycleEventReady, CMTimeGetSeconds([session.player currentTime]));
    AVPlayerItem * avPlayerItem = [session.player currentItem];
    avPlayerItem.audioTimePitchAlgorithm = AVAudioTimePitchAlgorithmVarispeed;
    session.player.rate = 1.7;
  }
}
    Das native SDK für iOS unterstützt keine negativen Werte für die Wiedergaberate (dh wir unterstützen das Rückwärtsspielen von Videos nicht).

    Einstellen des VR-Schutzbrillen-Modus für 360°-Videos

    Wenn Sie ein 360°-Video abspielen, können Benutzer die Taste Video 360 in der Steuerungsleiste auswählen, um in den VR-Goggle-Modus zu wechseln. Möglicherweise möchten Sie dies auch programmgesteuert tun, bevor die Wiedergabe beginnt. Sie können dies tun, indem Sie die aktualisieren BCOVPlaybackController Protokolle viewProjection Eigenschaft wie folgt:

    - (void)playbackController:(id<BCOVPlaybackController>)controller didAdvanceToPlaybackSession:(id<BCOVPlaybackSession>)session
{
BCOVVideo360ViewProjection *viewProjection = [self.playbackController.viewProjection copy];
[viewProjection setProjectionStyle:BCOVVideo360ProjectionStyleVRGoggles];
[self.playbackController setViewProjection:viewProjection];
}
    Beachten Sie, dass Sie den Projektionsstil des Controllers in der didAdvanceToPlaybackSession Delegate-Methode festlegen müssen.

    Ändern der Hintergrundfarbe

    Wenn Sie ein Video im Hochformat wiedergeben, bemerken Sie möglicherweise einen schwarzen Rand oben und unterhalb des Players. Die Player-Ansicht hat die Größe des Bildschirms, aber das Video nimmt nur einen kleinen Teil der Mitte der Player-Ansicht ein. Die sichtbaren Teile rund um das Video sind der Hintergrund der Player-Ebene.

    Das ist normales AVPlayer Verhalten. Es verkleinert Ihr Video so, dass es in die Player-Ebene passt, und der Rest ist der Hintergrund der Player-Ebene.

    Sie können den Hintergrund der Player-Ebene mit dem folgenden Code ändern:

    (void)playbackController:(id<BCOVPlaybackController>)controller didAdvanceToPlaybackSession:(id<BCOVPlaybackSession>)session
{
session.playerLayer.backgroundColor = UIColor.whiteColor.CGColor;
}
