    Schritt für Schritt: Einfache Video-App mit Objective-C

    In diesem Thema erfahren Sie, wie Sie mit dem Brightcove Player SDK für iOS eine einfache Videowiedergabe-App erstellen. Sie werden es mit der Programmiersprache Objective-C erstellen.

    Übersicht

    In diesem Beispiel wird ein Video wiedergegeben, das von der Brightcove Playback API zurückgegeben wurde, der neuesten und empfohlenen API, um Inhalte aus Ihrer Video Cloud-Bibliothek abzurufen.

    Für diese Option benötigen Sie einen Richtlinienschlüssel. Wenn Sie mit Richtlinienschlüsseln nicht vertraut sind, finden Sie weitere Informationen unter Überblick über die Richtlin

    Dieses Beispiel ist in Objective-C geschrieben, aber Sie können auch mit Swift experimentieren. Siehe folgendes:
    • Schritt für Schritt: Einfache Video-App mit Swift
    • Swift Version des VideoCloudBasicPlayer

    In diesem Beispiel werden Videos aus einem internen Array von Videodateien wiedergegeben.

    3. Führen Sie den pod install Befehl aus.
    4. Öffnen und starten Sie den neu erstellten Arbeitsbereich.

    Erstellen Sie das Projekt

    Richten Sie ein Projekt in Xcode ein. Fügen Sie dann das Brightcove Player SDK zusammen mit allen Abhängigkeiten zum Projekt hinzu.

    Richten Sie das Xcode-Projekt ein

    Erstellen Sie ein neues Xcode-Projekt für die App.

    1. Beginnen Sie mit der Erstellung eines neuen iOS-Projekts in Xcode. Wählen Sie Neues Xcode-Projekterstellen aus. Alternativ können Sie im Xcode-Hauptmenü Datei, Neu, Projektauswählen.

    2. Wählen Sie Einzelansichtsanwendung für die Vorlage und wählen Sie Weiteraus.

    3. Legen Sie die Projektinformationen wie folgt fest:

      • Bezeichnung des Produkts: Einfaches Video-Wiedergeben
      • Mannschaft: keine

        Das Feld Team ist optional. Wenn Sie Ihre App im App Store verteilen möchten, müssen Sie über das Apple Developer Program ein Team auswählen, zu dem Sie gehören. Der Teamname ist für das Signieren von Code erforderlich. In diesem Beispiel wählen wir keine aus.

      • Name der Organisation: Ihr Firmenname
      • Organisations-ID: com.beispiel-company
        Dies macht den Produktnamen für den App Store einzigartig.

    4. Wählen Sie den Speicherort aus, an dem Sie Ihr Projekt speichern und erstellen möchten.

    5. Schließen Sie nun das Projekt (ja, schließen Sie es — das ist wichtig!)

    Fügen Sie das SDK und seine Abhängigkeiten zu Ihrem Projekt hinzu

    Die einfachste Möglichkeit, das SDK und seine Abhängigkeiten zu Ihrem Projekt hinzuzufügen, ist die Verwendung von CocoaPods.

    CocoaPods ist ein Abhängigkeitsmanager, der Ihrem Projekt Bibliotheken hinzufügt. Dies ist nicht erforderlich, erleichtert aber die Installation. Informationen zur Installation von CocoaPods finden Sie in den Anweisungen auf der CocoaPods Website.

    • CocoaPods Version 1.0 oder neuer ist erforderlich.
    • Durch die Verwendung von CocoaPods (dem pod Tool) in einem bestehenden Xcode-Projekt wird die Projektdatei geändert. Vielleicht möchten Sie vorher ein Backup erstellen.

    1. Erstellen Sie in Ihrem Projektordner eine Nur-Text-Datei namens Podfile (keine Dateierweiterung).

    2. Fügen Sie der Podfile mit einem Texteditor die folgenden Codezeilen hinzu und speichern Sie sie. Dieser Code macht folgendes:

      • 1. Zeile: Verweist auf den GitHub-Speicherort für die Brightcove-Pod-Spezifikationen.
      • 3. Zeile: Legt fest, dass Pods statt statischer Bibliotheken Frameworks verwenden.
      • Zeile 5: Definiert die iOS-Plattformversion.

      • Zeilen 7-9: Installieren Sie das Brightcove Native Player SDK.

        1. Holen Sie sich die aktuelle Native SDK-Version aus dem Brightcove Native SDK for iOS Overview Dokument.
        2. Ersetzen Sie den Wert '6.3.11' durch die aktuelle SDK-Versionsnummer.

        3. Weitere Informationen zum Verwalten von Abhängigkeiten finden Sie im Artikel So verwalten Sie Abhängigkeiten mit CocoaPods .

         
        Die target Eigenschaft wird für CocoaPods Version 1.0 und höher verwendet.
      source 'https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks!

platform :ios, '11.0' # (or whatever version you want)

target 'Simple-Video-Playback' do
  pod 'Brightcove-Player-Core', '6.3.11'
end

    3. Öffnen Sie eine Terminalsitzung und navigieren Sie zu Ihrem Xcode-Projektordner für Simple-Video-Playure

    4. Geben Sie in der Terminalsitzung den folgenden Befehl ein:

      pod install

      und drücke die Eingabetaste , um es auszuführen. Sie sollten eine Reihe von Nachrichten im Terminal sehen, die darauf hinweisen, dass das Brightcove Player SDK Ihrem Projekt hinzugefügt wurde.

    5. Beachten Sie die letzte Zeile, was wichtig ist - ab diesem Zeitpunkt müssen Sie die Datei Simple-Video-Playback.XCWorkspace in Xcode öffnen, nicht die Simple-Video-Playback.xcodeproj Datei. Der pod install Befehl erstellt diese .xcworkspace Datei für Ihr Projekt.

    • Wenn Sie regelmäßig Podfiles erstellen und bearbeiten, können Sie versuchen, die vorherigen Schritte mit der CocoaPods App abzuschließen.
    • Wenn Sie CocoaPods nicht verwenden möchten, können Sie die neueste Version des SDK herunterladen und sie manuell zu Ihrem Projekt hinzufügen.

    Festlegen des Audioverhaltens für die App

    Die Audiositzung behandelt das Audioverhalten auf App-Ebene. Erfahren Sie mehr über die AvAudioSession Klasse.

    Sie können aus mehreren Kategorien und Einstellungen für Audiositzungen wählen, um das Audioverhalten Ihrer App anzupassen. Wählen Sie die beste Kategorie für Audio-Sessions für Ihre App aus.

    Für dieses Beispiel werden wir verwenden AVAudioSessionCategoryPlayback. Dadurch wird Audio wiedergegeben, selbst wenn der Bildschirm gesperrt ist und der Schalter Ring/Silent auf stumm eingestellt ist. Um es einfach zu halten, werden wir den Code dafür in den App-Delegaten einfügen.

    1. Öffnen Sie in Ihrem Projekt die App Delegate-Implementierungsdatei (AppDelegate.M).

    2. Fügen Sie in der didFinishLaunchingWithOptions Methode Code hinzu, um die Kategorie der Audiositzungen festzulegen. Achten Sie darauf, das AVFoundation Framework zu importieren.

      • Wir benötigen den folgenden Code, um sicherzustellen, dass das Audio wiedergegeben wird, wenn wir es erwarten. Zum Beispiel hören wir das Video nicht, ohne diesen Code festzulegen, wenn der Stummschaltung eingeschaltet ist.
      • Zur Vereinfachung in der Stichprobe legen wir dies in den App-Delegaten. Weitere Informationen zur Verwendung in Ihrer eigenen App finden Sie im Audio-Sitzungsprogrammierhandbuch.

    Passen Sie das Projekt mit Ihren Werten an

    Richten Sie den Spieler ein

    1. Erstellen Sie unter der init Funktion eine Funktion namens setup , die beim Laden der App aufgerufen wird.

      • Zeile 40: Verwendet den gemeinsamen Manager, um einen Wiedergabe-Controller zu erstellen. Die BCOVPlayerSDKManager Klasse ist ein Singleton, mit dem Sie andere Objekte im SDK-Ökosystem erstellen können.

      • Zeile 42: Optional: Wenn Sie die BCOVVideo Klasse überschreiben oder den Brightcove Player und den Wiedergabe-Service oder -Katalog nicht verwenden, müssen Sie Ihre Video Cloud-Konto-ID an Video Cloud Analytics senden. Auf diese Weise können Sie Daten für diese App in Video Cloud Analytics anzeigen.

        Weitere Informationen zum Senden von Daten an Video Cloud Analytics finden Sie im Handbuch zur Verwendung von Brightcove Analytics .
      • Zeilen 44-46: Stellen Sie den Vertreter ein und schalten Sie die Funktionen für den automatischen Vorsprung und die automatische Wiedergabe ein.

      • Zeile 48: Initialisiert den Wiedergabe-Dienst mit Ihrer Konto-ID und Ihrem Richtlinienschlüssel.

      - (void)setup
{
_playbackController = [BCOVPlayerSDKManager.sharedManager createPlaybackController];

_playbackController.analytics.account = kViewControllerAccountID; // Optional

_playbackController.delegate = self;
_playbackController.autoAdvance = YES;
_playbackController.autoPlay = YES;

_playbackService = [[BCOVPlaybackService alloc] initWithAccountId:kViewControllerAccountID policyKey:kViewControllerPlaybackServicePolicyKey];
}

    Konfigurieren Sie den Player

    1. In der viewDidLoad Funktion machen Sie folgendes:

      • Zeile 56: Erstellen und legen Sie die Brightcove-Spielersteuerelemente mithilfe des standardmäßigen VOD-Layouts fest.
      • Zeile 58: Fügt die Spieleransicht als Unteransicht der Hauptansicht hinzu.
      • Zeile 59: Deaktivieren Sie die automatische Größenänderung der Maske.
      • Zeilen 60-65: Verwenden Sie Auto Layout , um dynamische Contraints für die Spieleransicht zu definieren.
      • Zeilen 66: Weist die Spieleransicht der zugehörigen globalen Variable zu.
      • Zeilen 69: Verbindet die Player-Ansicht mit dem Wiedergabe-Controller.

      • Zeile 71: Ruft die requestContentFromPlaybackService Funktion auf, die Sie im nächsten Schritt definieren werden.

      - (void)viewDidLoad {
  [super viewDidLoad];
  // Do any additional setup after loading the view, typically from a nib.

  // Set up our player view. Create with a standard VOD layout.
  BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:[BCOVPUIBasicControlView basicControlViewWithVODLayout] ];

  _videoContainer addSubview:playerView];
  playerView.translatesAutoresizingMaskIntoConstraints = NO;
  [NSLayoutConstraint activateConstraints:@[
      [playerView.topAnchor constraintEqualToAnchor:_videoContainer.topAnchor],
      [playerView.rightAnchor constraintEqualToAnchor:_videoContainer.rightAnchor],
      [playerView.leftAnchor constraintEqualToAnchor:_videoContainer.leftAnchor],
      [playerView.bottomAnchor constraintEqualToAnchor:_videoContainer.bottomAnchor],
  ]];
  _playerView = playerView;

  // Associate the playerView with the playback controller.
  _playerView.playbackController = _playbackController;

  [self requestContentFromPlaybackService];
}

    Fordern Sie Inhalte aus der Brightcove-Bibliothek an

    Um Videoinhalte wiederzugeben, fordern Sie ein Video aus Ihrer Video Cloud-Bibliothek an.

    1. Erstellen Sie unter der viewDidLoad Funktion eine Funktion mit dem Namen requestContentFromPlaybackService wie folgt:

      - (void)requestContentFromPlaybackService
{
[self.playbackService findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {

	if (video)
	{
		[self.playbackController setVideos:@[ video ]];
	}
	else
	{
		NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
	}

}];
}

    Verwalten der Mediensicherheit

    App Transport Security (ATS) setzt sichere Verbindungen zwischen Ihrer App und Webdiensten durch. Mit der Veröffentlichung des iOS 9 SDK durch Apple wurde eine neue Funktion namens App Transport Security (ATS) hinzugefügt.

    1. Eine der folgenden Situationen trifft auf Ihre App zu:

      • Standardmäßig verwendet das Brightcove Native SDK für iOS eine Quellenauswahlrichtlinie, um HTTPS über HTTP-Quellen zu wählen, damit Sie Ihre Apps mit aktiviertem ATS erstellen können.

        Das war's, du bist bereit, deine App zu starten.

         

      • Wenn Ihre Remote-Assets HTTPS anstelle von HTTP-Quellen verwenden, können Sie Ihre Apps mit aktiviertem ATS erstellen.

        Das war's, du bist bereit, deine App zu starten.

         

      • Wenn Sie HTTP-Quellen verwenden oder andere HTTP-Anrufe in Ihrer App haben, kann folgende Fehlermeldung auftreten:

        App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

        Dies bedeutet, dass ATS aktiviert ist, Ihr System jedoch nicht so konfiguriert ist, dass es die ATS-Anforderungen erfüllt. Informationen zur Behebung dieser Situation finden Sie im Dokument Working with App Transport Security (ATS) .
