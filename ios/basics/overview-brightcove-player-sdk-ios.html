    Übersicht: Brightcove-Player-SDK für iOS

    Willkommen bei der Player SDK für iOS-Entwicklerdokumentation. Hier finden Sie Informationen zum Erstellen nativer Apps zum Abspielen von Videoinhalten auf iOS-Geräten.

    Aufbau der Dokumentation

    Bei Brightcove bemühen wir uns, die Dokumentation strukturiert, sauber und intuitiv zu halten. Daher nutzen wir Github ReadMes für Details zu all unseren Plugins und Integrationen. Zu Ihrer Bequemlichkeit haben wir diese Plugins unten aufgelistet und verknüpft.

    Unterstützte Versionen

    Das Native Player SDK für iOS bietet Unterstützung für die folgenden Versionen von iOS:

    Aktive Unterstützung

    Brightcove bietet aktive Unterstützung für das neueste iOS SDK in den folgenden iOS-Versionen:

    • iOS-14
    • iOS-13
    • ios 12,4,1

    Passive Unterstützung

    Brightcove bietet passive Unterstützung für die folgenden iOS-Versionen:

    • iOS 11,4,1

    Support-Level verstehen

    Die Unterstützung ist in Bezug auf die [Hauptversion]definiert.[ Nebenversion].[ Patchversion] eines Betriebssystems (zum Beispiel: iOS 11.4.1). Um die neuesten technologischen Verbesserungen von Apple nutzen zu können, bieten wir zwei Supportstufen an:

    • Active (Aktiv): Brightcove wird Fehler auf diesen Betriebssystemversionen testen und beheben.
    • Passiv: Brightcove führt unter diesen Betriebssystemversionen Tests aus, Bugs werden allerdings nur fallabhängig behoben.

    Brightcove leistet im Allgemeinen aktiven Support für die Betriebssystemversionen, die von 80 % der Benutzer verwendet werden. Brightcove leistet passiven Support für alle anderen Versionen, die von mindestens 5 % der Benutzer verwendet werden.

    Erste Schritte

    Bevor Sie mit der Entwicklung mit dem Player SDK für iOS beginnen, sollten Sie einige Konzepte lernen:

    1. Medien

      Die aktuelle Version des Native Player SDK unterstützt das Abspielen von Inhalten von Ihrem Video Cloud-Konto über die Playback-API oder als Remote-Assets mit Internet-zugänglichen URLs.

    2. Analytics

      Mit dem Native Player SDK erhalten Video Cloud-Kunden automatisch robuste Analysedaten, die in Video Cloud gemeldet werden.

    3. Kern-SDK und Abhängigkeiten

      Das Brightcove Player SDK für iOS und seine Plugins können Ihrem Projekt mit dem CocoaPods Dependency Manager hinzugefügt werden. Die Podspecs für alle Brightcove Player SDK-Komponenten werden im BrightcoveSpecs-Repositorygehostet.

      CocoaPods Version 1.0 oder neuer ist erforderlich.
    4. Native SDK-Referenz

      Um Ihnen das Verständnis des SDK zu erleichtern, finden Sie eine detaillierte Aufschlüsselung der Architektur im Player SDK für iOS. Erfahren Sie, wie kritische Komponenten interagieren, um mit der Komplexität der Steuerung des nativen Videoplayers umzugehen. Player SDK für iOS-Installation, Schnellstart und Architektur

      Hier finden Sie auch die neueste Beta-Version des SDK, falls eine existiert.

    5. Überblick über das native SDK

      Für einen schnellen Überblick erfahren Sie, wie die Komponenten des Brightcove Player SDK mit den nativen iOS-Komponenten interagieren. Übersicht über das Spieler-SDK für iOS

    6. Fragen und Antworten

      Begleiten Sie uns im SDK-Entwicklerforum , in dem Fragen zu SDK von Brightcove Engineers und der Community beantwortet werden. Suche nach zuvor beantworteten Fragen oder poste eine neue. Als öffentliche Gruppe können Sie Zugriff erhalten, indem Sie Ihre E-Mail-Adresse angeben.

    7. Video-Training

      Zeigen Sie eine Reihe von Schulungsvideos an, die Sie durch die Schritte zum Erstellen einer Swift-App in Xcode führen, die ein Video aus Ihrer Video Cloud-Bibliothek abspielt.

    Tipps:

    • PlayerUI Ansichten und Steuerelemente (einschließlich Werbeteuerung) verwenden den iOS Safe Layout-Guide, um eine unsachgemäße Platzierung in Ecken, Kerben oder unter dem Hausindikator zu vermeiden.
    • Die Sidecar Subtitles Komponente unterstützt die Wiedergabe über AirPlay mit Untertiteln. Einzelheiten dazu, wie Sie Ihre Untertitel synchron halten, finden Sie im Dokument Synchronisieren von WebVTT Captions .

    Hinweise:

    • Um die CocoaPods Compliance zu verbessern, hat das Native SDK für iOS 6.7.7 einige Podspec-Dateien umbenannt. Einzelheiten finden Sie in der Native SDK-Referenz.
    • Das minimale Bereitstellungsziel für alle Brightcove Native Player für iOS-Frameworks ist jetzt iOS 11.0 und tvOS 11.0
    • Das Brightcove Native SDK für iOS 6.5.0+ erfordert Xcode 11.0 oder höher.
    • Die automatische Referenzzählung (ARC) ist in Ihrem Projekt erforderlich. Dies ist der aktuelle Standard für Xcode-Projekte.
    • Mit der Veröffentlichung von iOS 9 und Xcode 7 wurde eine neue Funktion namens App Transport Security (ATS) eingeführt. Erfahren Sie mehr über die Arbeit mit App Transport Security.
    • Ein bekanntes Apple-Problem mit FairPlay-Streaming-Inhalten besteht darin, dass es nicht im AirPlay-Spiegelungsmodus gerendert wird. Weitere Einzelheiten finden Sie in der WWDC-Präsentation: Content Protection for HLS (scrollen Sie etwa zur Hälfte des Transkripts).

    Integration

    Die folgende Tabelle enthält Links, um Plugins in Ihre SDK-Implementierung zu integrieren. Die Plugins können von ihren jeweiligen Github-Repos heruntergeladen werden, und jedes Repo enthält eine detaillierte README mit Anweisungen zur Einrichtung des Plugins.

    Die Beispielanwendungen demonstrieren die Grundfunktionalität jedes in das SDK integrierten Plugins. Diese Beispiel-Apps sind eine gute Möglichkeit, sich über jedes Plugin zu informieren und dienen als Bezugspunkt beim Debuggen.

    Die aktuelle Version des Native SDK für iOS enthält alle Komponenten mit derselben Versionsnummer. Bei der Integration von Komponenten müssen das SDK und die Versionsnummern der Komponenten übereinstimmen. Wenn Sie beispielsweise die IMA-Plugin-Komponente 6.7.0 verwenden, müssen Sie das Native SDK 6.7.0 verwenden.

    [ 1] Die aktuelle Version des IMA-Plugins unterstützt Googles IMA iOS SDK v3, das iOS-Versionen 9.0+ unterstützt. Einzelheiten finden Sie in der Dokumentation von Google.

    Die Brightcove Native SDKs unterstützen VMAP XML-Vorlagen, die das VAST-Protokoll einschließlich VAST 3.0 und eingeschränkte Unterstützung für VAST 4.0enthalten. Weitere Informationen zur VAST Versionskompatibilität finden Sie im Support- und Kompatibilitätsdokument von Google für das IMA- SDK für iOS. Die Native SDKs unterstützen auch das IAB Open Measurement SDK.

    [ 2] Wenn Sie Swift verwenden, müssen Sie die IMA-Bibliotheken von Google manuell hinzufügen, anstatt CocoaPods zu verwenden. Dies liegt daran, dass die IMA-Bibliothek von Google keine dynamischen Frameworks unterstützt und Swift den Aufbau statischer Bibliotheken nicht unterstützt. Einzelheiten finden Sie im Abschnitt „ Manuell “ in der Dokumentation zum IMA-Plugin für Brightcove Player SDK für iOS . Obwohl das IMA-Plugin von Brightcove ein dynamisches Framework verwendet, wird die Google-IMA-Bibliothek nur als statisches Framework bereitgestellt.

    Kombinieren von Plugins

    Mit dem Native SDK für iOS werden die BCOVSessionProviders aus jedem Plugin erstellt und miteinander verkettet. Diese Kette wird dann verwendet, um das zu erstellen BCOVPlaybackController . Einzelheiten finden Sie im Abschnitt Kombinierende Plugins der SDK-Referenz.

    IMA-Plugin

    Die Platzierung von VAST Ads wird normalerweise durch ein VMAP-Dokument festgelegt. VAST beschreibt eine Anzeige, während VMAP die Platzierung von Anzeigen in einem Video beschreibt. Informationen zum Code finden Sie im BasiciMapLayer-iOS-Beispiel .
