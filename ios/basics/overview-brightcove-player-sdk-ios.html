Dokumentationsstruktur

Wir bei Brightcove bemühen uns, die Dokumentation strukturiert, sauber und intuitiv zu halten. Daher nutzen wir github READMEs für Details zu all unseren Plugins und Integrationen. Der Einfachheit halber haben wir diese Plugins unten aufgelistet und verlinkt.

Unterstützte Versionen

Das Native Player SDK für iOS bietet Unterstützung für die folgenden iOS-Versionen:

Aktive Unterstützung

Brightcove bietet aktiven Support für das neueste iOS SDK in der neuesten öffentlichen Version der folgenden iOS-Versionen:

  • iOS 14
  • iOS 13.7
  • iOS 12.5.3

Passive Unterstützung

Brightcove bietet passiven Support für die folgenden iOS-Versionen:

  • iOS 11.4.1

Unterstützungsstufen verstehen

Unterstützung wird definiert in Bezug auf die [Hauptversion].[Nebenversion].[Patch-Version] eines Betriebssystems (zum Beispiel: iOS 11.4.1). Um von den neuesten Technologieverbesserungen von Apple zu profitieren, bieten wir zwei Supportstufen an:

  • Aktiv: Brightcove testet und behebt Fehler in diesen Betriebssystemversionen.
  • Passiv: Brightcove wird diese Betriebssystemversionen testen, aber Fehlerkorrekturen werden von Fall zu Fall behandelt.

Im Allgemeinen bietet Brightcove aktiven Support für die Versionen des Betriebssystems, die von 80 % der Benutzerbasis verwendet werden, und passiven Support für andere Versionen, die von mindestens 5 % der Benutzerbasis verwendet werden.

Einstieg

Bevor Sie mit der Entwicklung mit dem Player SDK für iOS beginnen, sollten Sie sich mit einigen Konzepten vertraut machen:

  1. Medien

    Die aktuelle Version des Native Player SDK unterstützt die Wiedergabe von Inhalten aus Ihrem Video Cloud-Konto über die Wiedergabe-API oder als Remote-Assets mit über das Internet zugänglichen URLs.

  2. Analytik

    Mit dem Native Player SDK erhalten Video Cloud-Kunden automatisch robuste Analysedaten, die in Video Cloud gemeldet werden.

  3. Core SDK und Abhängigkeiten

    Das Brightcove Player SDK für iOS und seine Plug-ins können Ihrem Projekt mithilfe der Kakaoschoten Abhängigkeitsmanager. Die Podspecs für alle Brightcove Player SDK-Komponenten werden im BrightcoveSpecs-Repository.

    CocoaPods-Version 1.0 oder neuer ist erforderlich.
  4. Referenz zum nativen SDK

    Um das SDK besser zu verstehen, finden Sie eine detaillierte Aufschlüsselung der Architektur im Player SDK für iOS. Erfahren Sie, wie kritische Komponenten interagieren, um die Komplexität der Steuerung des nativen Videoplayers zu bewältigen. Player SDK für iOS-Installation, Schnellstart und Architektur

    Hier finden Sie auch die neueste Beta-Version des SDK, falls vorhanden.

  5. Übersicht über das native SDK

    Für einen schnellen Überblick erfahren Sie, wie die Brightcove Player SDK-Komponenten mit den nativen iOS-Komponenten interagieren. Übersicht über das Player-SDK für iOS

  6. Fragen und Antworten

    Begleiten Sie uns in der SDK-Entwicklerforum Hier werden SDK-bezogene Fragen von Brightcove-Ingenieuren und der Community beantwortet. Suchen Sie nach bereits beantworteten Fragen oder stellen Sie eine neue. Als öffentliche Gruppe erhalten Sie Zugang, indem Sie Ihre E-Mail-Adresse angeben.

  7. Videoschulung

    Sehen Sie sich eine Reihe von Schulungsvideos an, die Sie durch die Schritte zum Erstellen einer Swift-App in Xcode führen, die ein Video aus Ihrer Video Cloud-Bibliothek abspielt.

Tipps:

  • PlayerUI-Ansichten und -Steuerelemente (einschließlich Werbesteuerelementen) verwenden die iOS-Anleitung für sicheres Layout, um eine falsche Platzierung in Ecken, im Kerbbereich oder unter der Startanzeige zu vermeiden.
  • Die Sidecar-Untertitelkomponente unterstützt die Wiedergabe über AirPlay mit Untertiteln. Weitere Informationen zum Synchronisieren Ihrer Untertitel finden Sie im Synchronisieren von WebVTT-Untertiteln dokumentieren.

Anmerkungen:

  • Um die Compliance mit CocoaPods zu verbessern, hat das Native SDK für iOS 6.7.7 einige Podspec-Dateien umbenannt. Einzelheiten finden Sie im Referenz zum nativen SDK.
  • Das Mindestbereitstellungsziel für alle Brightcove Native Player für iOS-Frameworks ist jetzt iOS 11.0 und tvOS 11.0
  • Das Brightcove Native SDK für iOS 6.5.0+ erfordert Xcode 11.0 oder höher.
  • In Ihrem Projekt ist die automatische Referenzzählung (ARC) erforderlich. Dies ist die aktuelle Standardeinstellung für Xcode-Projekte.
  • Mit der Veröffentlichung von iOS 9 und Xcode 7 wurde eine neue Funktion namens App Transport Security (ATS) eingeführt. Lerne mehr über Arbeiten mit App-Transportsicherheit.
  • Ein bekanntes Apple-Problem mit FairPlay-Streaming-Inhalten ist, dass diese nicht im AirPlay-Spiegelungsmodus gerendert werden. Weitere Informationen finden Sie im WWDC-Präsentation: Inhaltsschutz für HLS (Blättern Sie etwa zur Hälfte durch das Transkript).

Plugins kombinieren

Mit dem Native SDK für iOS ist die BCOVSessionProviders von jedem Plugin werden erstellt und miteinander verkettet. Diese Kette wird dann verwendet, um die BCOVPlaybackController . Einzelheiten finden Sie im Abschnitt Plugins kombinieren der SDK-Referenz.

Werbung mit dem IMA-Plugin

Die Platzierung von VAST-Anzeigen wird normalerweise durch ein VMAP-Dokument festgelegt. VAST beschreibt eine Anzeige, während VMAP die Platzierung von Anzeigen in einem Video beschreibt. Codedetails finden Sie im BasicIMAPlayer iOS Stichprobe.

Wenn Sie mit IMA-Anzeigen arbeiten, sollten Sie vor der Implementierung prüfen, ob Ihre Anzeigen außerhalb des SDK funktionieren. Folgende Links sind dabei hilfreich:

  • Google VAST Ad Response Validator
  • Leitfaden für digitale Video-Wiedergabelisten (Multiple Ad Playlist, VMAP)
  • Technisches Google IMA-Forum

Hinweise zur Plattform

Die folgenden Hinweise beziehen sich auf die iOS-Plattform.

Spiegeln Ihres Bildschirms mit Airplay

Die AirPlay-Funktionalität funktioniert sofort mit dem Native SDK für iOS und ermöglicht es Benutzern, Videos mit Apple TV auf High-Definition-Displays zu streamen.

Sie können die Brightcove-Steuerelemente oder Ihre eigenen benutzerdefinierten Steuerelemente für AirPlay verwenden. Einzelheiten finden Sie im Benutzerdefinierte Steuerelemente für AirPlay Code-Auszug.

Die Verwendung von AirPlay mit älteren Geräten (Gen 3) kann zu unerwartetem Verhalten führen. Das Zielgerät weiß möglicherweise nicht, welche Textspuren auf dem Sendergerät ausgewählt wurden.
